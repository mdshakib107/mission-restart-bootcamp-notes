# `this` কী?

তোমরা জানো, JavaScript-এর `this` keyword অনেকটা এমন এক বন্ধুর মতো, যে কোনো argument-এর সময় বারবার পক্ষ বদলায়। এক মুহূর্তে সে এই ব্যক্তির পক্ষে, পরের মুহূর্তে অন্য ব্যক্তির পক্ষে।

`this`-এর ক্ষেত্রেও তুমি একই রকম কিছু দেখতে পাবে।

কখনো দেখবে, এটি একটি object-কে refer করছে।  
পরের মুহূর্তে দেখবে, এটি `undefined`-কে refer করছে।  
আবার অন্য কোনো মুহূর্তে দেখবে, এটি `window` object-কে refer করছে।

তখন তোমার মনে হতে পারে—আসলে এখানে কী হচ্ছে?

এই lesson-এর উদ্দেশ্য হলো বিষয়টিকে এতটা ভেঙে ব্যাখ্যা করা, যাতে তুমি বুঝতে পারো:

- কোন situation-এ `this` কী refer করছে,
- এবং কেন ঠিক সেই value-টাই refer করছে।

তাহলে আর তোমার মনে হবে না যে `this` ঠিকভাবে কাজ করছে না, অথবা `this` misbehave করছে।

চলো, কিছু core example দিয়ে শুরু করা যাক।

`this` হলো JavaScript-এর একটি keyword।

JavaScript-এ `this` keyword থাকার মূল উদ্দেশ্য হলো—একটি নির্দিষ্ট execution context-এ আসলে কী ঘটছে, তা বুঝতে আমাদের সাহায্য করা।

আমরা যখন “this” বলি, literal অর্থে আমরা কোনো একটি জিনিসের দিকে নির্দেশ করি:

> “এই জিনিসটি”  
> “এটাই”  
> “এই object-টি”

JavaScript programming context-এ আমরা যখন `this` বলি, তখন আমরা বোঝার চেষ্টা করি:

> এই মুহূর্তে আমরা কোন object-এর context-এর মধ্যে আছি, এবং code কোন context-এ execute হচ্ছে?

এটাই JavaScript-এর `this` keyword-এর মূল উদ্দেশ্য।

`this` keyword-এর value দেখে তুমি নির্ধারণ করতে পারো, এই মুহূর্তে তুমি কোন context-এর মধ্যে আছ। তারপর সেই অনুযায়ী decision নিতে পারো, business logic লিখতে পারো এবং প্রয়োজনীয় implementation করতে পারো।

`this` কখনো object-কে point করছে, কখনো `undefined`, আবার কখনো `window` object-কে point করছে—এই সব confusion নির্ভর করে `this` কোথায় ব্যবহার করা হচ্ছে তার উপর।

কিন্তু এখানে একটি খুব গুরুত্বপূর্ণ বিষয় আছে:

> `this` source code-এর কোথায় লেখা আছে—শুধু সেটি দেখে answer দেবে না।  
> Code execution-এর সময় `this` কোন function-এর মধ্যে এবং সেই function কীভাবে ব্যবহার বা invoke করা হচ্ছে—সেটিই বেশি গুরুত্বপূর্ণ।

এই lesson-এ আমরা সেটাই দেখব।

# Global Scope-এ `this`

মনে করো, তুমি browser environment-এ একটি JavaScript file load করলে। এমনকি file-টি blank হলেও, JavaScript একটি **Global Execution Context** তৈরি করে।

Browser environment-এর global execution context-এ আমরা সাধারণত দুইটি গুরুত্বপূর্ণ জিনিস পাই:

- `this` keyword
- `window` object

আর server-side environment-এ, যেমন Node.js-এ, তুমি `this` এবং global environment-এর object পাবে।

Initial global execution context-এ browser-এর ক্ষেত্রে `this` keyword-এর value `window` object-এর সমান হয়।

অর্থাৎ:

```js
this === window
```

এখানে `this`, `window` object-কে point করছে।

তাই global level-এ যদি আমরা লিখি:

```js
console.log(this);
```

তাহলে আমরা আশা করব output-এ `window` object আসবে।

অর্থাৎ, browser-এর global scope-এ:

```js
console.log(this);
```

Output হবে:

```js
Window { ... }
```

এখন অন্তত আমরা একটি situation জানি যেখানে `this` `window` object-কে refer করে।

> Global level-এ `this` access করলে, browser-এর classic JavaScript context-এ এটি `window` object-কে refer করে।

কিন্তু এটিই সব নয়। আরও অনেক situation আছে যেখানে `this`-এর value আলাদা হবে।

তোমার আসল কাজ হলো—প্রতিটি situation-এ `this`-এর value কী হবে, সেটি বুঝতে শেখা।

যদি তুমি situation দেখে নিজের মাথায় হিসাব করতে পারো:

> “এই situation-এ `this`-এর value এই object হবে।”

তাহলেই `this` keyword-এর বড় অংশ তুমি আয়ত্ত করে ফেলেছ।

এখন বিষয়টি আরও ভালোভাবে বুঝতে হলে আমাদের **binding**-এর rules শিখতে হবে।

# Binding কী?

Binding বললে প্রশ্ন আসে:

- কে bind হচ্ছে?
- কিসের সঙ্গে bind হচ্ছে?
- কোন value কোথায় associate হচ্ছে?

Global aspect ছাড়াও `this` বোঝার সময় আরও দুইটি বিষয় খুব গুরুত্বপূর্ণ:

- object
- function

কোনো function-এর ভিতরে `this` থাকলে, `this`-এর exact value বুঝতে প্রথমে দেখতে হবে:

> Function-টি কোথায় call হয়েছে?  
> Function-টি কোথায় invoke হয়েছে?

Function কোথায় লেখা আছে, শুধু সেটি দেখে answer দেবে না।

Function কোথায় invoke হয়েছে, সেটি বুঝতে পারলে `this`-এর value অনেক পরিষ্কারভাবে নির্ধারণ করা যায়।

চলো, example দিয়ে বুঝি।

# Implicit Binding

আমরা একটি object তৈরি করি:

```js
const employee = {
  id: 1,
  firstName: "Alex",
  lastName: "B"
};
```

আমরা জানি, object-এর ভিতরে method থাকতে পারে।

Method হলো এমন একটি property, যার value একটি function।

চলো একটি method তৈরি করি:

```js
const employee = {
  id: 1,
  firstName: "Alex",
  lastName: "B",

  returnThis: function () {
    return this;
  }
};
```

এখন object-এর property access করতে আমরা লিখতে পারি:

```js
console.log(employee.id);
```

এতে employee-এর `id` পাওয়া যাবে।

এখন আমরা object-এর ভিতরের `this` keyword-এর value জানতে চাই।

তাই লিখি:

```js
console.log(employee.returnThis());
```

`returnThis()` method `this` return করছে।

Output-এ আমরা দেখব `this` পুরো `employee` object-কেই refer করছে।

অর্থাৎ:

```js
this === employee
```

কেন?

আমরা method call করেছি এভাবে:

```js
employee.returnThis();
```

এখানে method-টি `employee` object-এর উপর invoke হয়েছে।

এই rule-টি খুব ভালোভাবে মনে রাখো:

> যখন কোনো object-এর উপর তার method invoke করা হয়, তখন method-এর ভিতরের `this` সেই object-এর সঙ্গে bind হয়, যে object-এর উপর method-টি invoke করা হয়েছে।

আবার বলছি:

> যখন আমরা কোনো object-এর উপর method invoke করি, তখন `this`-এর value সেই object-ই হয়।

এখানে:

```js
employee.returnThis();
```

`returnThis` একটি method, এবং এটি `employee` object-এর উপর invoke হয়েছে।

তাই:

```js
this === employee
```

এই behavior-কে বলা হয় **implicit binding**।

## Implicit Binding-এর Definition

কেউ যদি তোমাকে জিজ্ঞেস করে implicit binding কী, তুমি বলতে পারো:

> যখন dot notation ব্যবহার করে কোনো object-এর উপর একটি method call করা হয়, তখন method-এর ভিতরের `this` সেই object-এর সঙ্গে bind বা associate হয়, যে object-এর উপর method-টি invoke করা হয়েছে।

Pattern:

```js
object.method();
```

এখানে regular method-এর ভিতরে:

```js
this === object
```

এখন যেহেতু `this` `employee` object-কে point করছে, তাই আমরা লিখতে পারি:

```js
this.id
```

যা equivalent:

```js
employee.id
```

একইভাবে:

```js
this.firstName
```

Equivalent:

```js
employee.firstName
```

এবং:

```js
this.lastName
```

Equivalent:

```js
employee.lastName
```

তাই আমরা আরেকটি method লিখতে পারি:

```js
const employee = {
  id: 1,
  firstName: "Alex",
  lastName: "B",

  getFullName: function () {
    return `${this.firstName} ${this.lastName}`;
  }
};
```

এখন call করি:

```js
console.log(employee.getFullName());
```

এখানে `getFullName()` method-টি `employee` object-এর উপর call হয়েছে।

তাই implicit binding rule অনুযায়ী:

```js
this === employee
```

ফলে:

```js
this.firstName
```

মানে:

```js
employee.firstName
```

যার value `"Alex"`।

এবং:

```js
this.lastName
```

মানে:

```js
employee.lastName
```

যার value `"B"`।

তাই final output:

```text
Alex B
```

যখনই object method-এর ভিতরে `this` দেখবে, তখন method invocation দেখো:

```js
employee.getFullName();
```

Method regular function হলে `this` সেই object-কে refer করবে, যার উপর method call হয়েছে।

এই rule-টি মনে রেখো, কারণ বাস্তবে `this`-এর অনেক example implicit binding-এর সঙ্গে সম্পর্কিত।

# Tom and Jerry Example

এখন একটু complex example দেখি।

Tom and Jerry cartoon-এর reference ব্যবহার করি।

```js
const tom = {
  name: "Tom",
  age: 7
};

const jerry = {
  name: "Jerry",
  age: 3
};
```

এখন একটি function বানাই:

```js
function greetMe(obj) {
  obj.logMessage = function () {
    console.log(`${this.name} is ${this.age} years old`);
  };
}
```

এই function একটি object গ্রহণ করছে।

আমরা জানি, object-এর মধ্যে নতুন property যোগ করা যায়। আর property-এর value যদি function হয়, তাহলে সেটি method হয়ে যায়।

এখানে `obj`-এর মধ্যে নতুন property যোগ করা হয়েছে:

```js
obj.logMessage
```

এবং তার value একটি function।

এখন যদি আমরা লিখি:

```js
greetMe(tom);
```

তাহলে `tom` object function-এর ভিতরে `obj` হিসেবে যাবে।

আগে `tom` object ছিল:

```js
{
  name: "Tom",
  age: 7
}
```

এখন তার সঙ্গে `logMessage` method যোগ হবে:

```js
{
  name: "Tom",
  age: 7,
  logMessage: function () { ... }
}
```

এখন call করি:

```js
tom.logMessage();
```

`logMessage()` method-টি `tom` object-এর উপর call হয়েছে।

Implicit binding rule অনুযায়ী:

```js
this === tom
```

তাই:

```js
this.name
```

হবে `"Tom"`।

এবং:

```js
this.age
```

হবে `7`।

Output:

```text
Tom is 7 years old
```

এখন যদি আমরা একই কাজ Jerry-এর জন্য করি:

```js
greetMe(jerry);
jerry.logMessage();
```

এখানে method call হয়েছে:

```js
jerry.logMessage();
```

তাই:

```js
this === jerry
```

ফলে output:

```text
Jerry is 3 years old
```

এখন ভালো করে লক্ষ্য করো।

Function code একই ছিল:

```js
function () {
  console.log(`${this.name} is ${this.age} years old`);
}
```

কিন্তু `this`-এর value বদলে গেছে call site-এর উপর ভিত্তি করে।

```js
tom.logMessage();
```

এখানে `this` → `tom`

আর:

```js
jerry.logMessage();
```

এখানে `this` → `jerry`

তাই `this` কোথায় লেখা আছে দেখে answer দেবে না।

সবসময় দেখবে:

> Function-টি কোথায় call হয়েছে?  
> কোন object-এর উপর invoke হয়েছে?

Invocation point দেখেই implicit binding apply করবে।

যদি object-এর মধ্যে `name` property না থাকে, তাহলে:

```js
this.name
```

হবে `undefined`।

এখানে অন্য কোনো জাদু নেই। Object method-এর ক্ষেত্রে rule খুব straightforward:

> `object.method()` হলে regular method-এর ভিতরে `this` সেই object।

এই rule মাথায় রাখতে পারলে `this`-এর একটি বড় অংশ তুমি শিখে ফেলেছ।

---

# Standalone Function-এর ভিতরে `this`

এখন object method-এর context থেকে বেরিয়ে আমরা standalone function দেখি।

Standalone function মানে function কোনো object-এর method নয়।

```js
function sayName() {
  console.log("this inside function:", this);
}

sayName();
```

এখন প্রশ্ন:

> এখানে `this` কী হবে?

Non-strict browser context-এ output হবে:

```text
Window { ... }
```

অর্থাৎ `this` আবার `window` object-কে refer করছে।

এখানে `this`, `sayName` function-কে refer করছে না।

Function-টি global scope-এ declare করা হয়েছে এবং standaloneভাবে call হয়েছে:

```js
sayName();
```

কোনো object-এর উপর call হয়নি।

তাই non-strict browser environment-এ default behavior অনুযায়ী `this` global `window` object-এর দিকে যায়।

এখন মনে হতে পারে:

> তাহলে কি function-এর `this` সবসময় outer scope-এর `this` নেয়?

চলো nested function দিয়ে পরীক্ষা করি।

```js
function outer(a) {
  console.log("this inside outer:", this);

  function inner(b) {
    console.log("this inside inner:", this);
  }

  return inner;
}
```

এখন:

```js
const outerResult = outer(2);
outerResult(3);
```

`outer()` global level-এ standaloneভাবে call হয়েছে, তাই non-strict mode-এ `this` → `window`।

কিন্তু inner function তো outer function-এর ভিতরে declare করা। তাহলে কি inner-এর `this`, outer function-এর context নেবে?

Output দেখে বোঝা যাবে:

```text
this inside outer: Window
this inside inner: Window
```

অর্থাৎ inner function-এর `this` outer function-কে refer করছে না।

এখানে regular inner function standaloneভাবে call হয়েছে:

```js
outerResult(3);
```

তাই regular function-এর invocation rule প্রযোজ্য হয়েছে।

এটি lexicalভাবে outer function-এর `this` inherit করেনি।

এখন আমরা আরেকটি গুরুত্বপূর্ণ concept দেখি: **strict mode**।

---

# Strict Mode

JavaScript file-এর শুরুতে লিখতে পারো:

```js
"use strict";
```

Strict mode JavaScript code-কে কিছু stricter rule অনুসরণ করতে বাধ্য করে। এর উদ্দেশ্য হলো accidental mistake এবং confusing behavior কমানো।

Strict mode চালু করলে কিছু code-এর output বদলে যেতে পারে।

এখন আগের standalone function লিখি:

```js
"use strict";

function sayName() {
  console.log(this);
}

sayName();
```

এবার output হবে:

```text
undefined
```

Strict mode JavaScript-কে standalone function-এর `this` automatically `window` object-এ bind করতে দেয় না।

তাই:

> Strict mode-এ standalone regular function-এর ভিতরে `this` হলো `undefined`।

Nested standalone function-এর ক্ষেত্রেও একই:

```js
"use strict";

function outer() {
  console.log(this);

  function inner() {
    console.log(this);
  }

  inner();
}

outer();
```

Output:

```text
undefined
undefined
```

তাই rule:

```text
Standalone regular function:
non-strict browser mode → window
strict mode → undefined
```

বাস্তব project-এ বেশিরভাগ সময় strict behavior-এর মধ্যে code চালানো হয়। তাই plain standalone regular function-এর ভিতরে `this` সাধারণত `undefined` হবে—এটি মনে রাখা গুরুত্বপূর্ণ।

`this` সেই বন্ধুর মতো, যে situation অনুযায়ী পক্ষ বদলায়।

এখন পর্যন্ত আমরা শিখলাম:

- Global scope-এ `this`
- Implicit binding
- Standalone regular function-এর ভিতরে `this`
- Strict mode-এ `this`

কিন্তু এখানেই শেষ নয়।

এখন Arrow Function-এর কথা বলি।

---

# Arrow Function-এর ভিতরে `this`

আমরা একটি arrow function লিখি:

```js
const getFood = () => {
  return this;
};
```

এখন:

```js
console.log(getFood());
```

ধরা যাক arrow function global scope-এ define করা হয়েছে।

এবং আমরা strict mode-এ code চালাচ্ছি।

Standalone regular function strict mode-এ `undefined` দিয়েছিল। কিন্তু arrow function global scope-এ define থাকলে এটি surrounding lexical scope-এর `this` নেয়।

Browser-এর classic script context-এ global `this` হলো `window`।

তাই arrow function return করতে পারে:

```text
Window { ... }
```

এখানেই regular function এবং arrow function-এর বড় পার্থক্য।

## Arrow Function-এর সবচেয়ে গুরুত্বপূর্ণ Rule

> Arrow function-এর নিজস্ব `this` নেই।

Arrow function-এর ভিতরে `this` থাকলে, JavaScript দেখে:

> Arrow function-টি lexically কোথায় define করা হয়েছে?

তারপর surrounding scope-এর `this` ব্যবহার করে।

একে lexical `this` বলা যায়।

Global scope-এ arrow function define হলে:

```js
const getFood = () => this;
```

এখানে arrow function-এর surrounding scope global scope।

তাই browser context-এ:

```js
this === window
```

---

## Regular Function বনাম Arrow Function

Regular standalone function:

```js
"use strict";

function regularFn() {
  return this;
}
```

Output:

```text
undefined
```

Arrow function:

```js
const arrowFn = () => {
  return this;
};
```

এটি lexical surrounding `this` নেয়।

তাই মনে রাখো:

> Regular function-এর `this` বুঝতে call site দেখো।  
> Arrow function-এর `this` বুঝতে definition location বা lexical scope দেখো।

# Object-এর মধ্যে Arrow Function

এখন বিষয়টিকে একটু complex করি।

```js
const food = {
  name: "Mango",
  color: "Yellow",

  getDescription: () => {
    return `${this.name} is ${this.color}`;
  }
};
```

এখন call করি:

```js
console.log(food.getDescription());
```

অনেকে ভাববে output হবে:

```text
Mango is Yellow
```

কিন্তু তা হবে না।

কেন?

`getDescription` হলো arrow function।

Arrow function-এর নিজস্ব `this` নেই।

Function-টি object literal-এর মধ্যে লেখা হয়েছে ঠিকই, কিন্তু object literal নিজে arrow function-এর জন্য `this` তৈরি করে না।

Arrow function যেখানে lexically define করা হয়েছে, তার surrounding scope হলো global scope।

তাই `this` global context-কে refer করবে।

Browser context-এ:

```js
this === window
```

`window` object-এর মধ্যে আমাদের food object-এর `name` এবং `color` values নেই।

তাই:

```js
this.name
```

এবং:

```js
this.color
```

expected `"Mango"` এবং `"Yellow"` হবে না।

ফলে output হতে পারে:

```text
undefined is undefined
```

এটি খুব common mistake।

Object-এর property হিসেবে arrow function লিখলেই `this` object-কে refer করবে—এটি ভুল ধারণা।

---

# Regular Method দিয়ে Fix

একটি fix হলো arrow function-এর পরিবর্তে regular function ব্যবহার করা।

```js
const food = {
  name: "Mango",
  color: "Yellow",

  getDescription: function () {
    return `${this.name} is ${this.color}`;
  }
};
```

এখন:

```js
console.log(food.getDescription());
```

এখানে `getDescription()` regular method, এবং এটি `food` object-এর উপর invoke হয়েছে।

Implicit binding rule অনুযায়ী:

```js
this === food
```

তাই output:

```text
Mango is Yellow
```

Modern method shorthand-এও লেখা যায়:

```js
const food = {
  name: "Mango",
  color: "Yellow",

  getDescription() {
    return `${this.name} is ${this.color}`;
  }
};
```

# কিন্তু Arrow Function ব্যবহার করেই Fix করতে হবে

এখন challenge:

> Regular function-এ পরিবর্তন করা যাবে না। Arrow function ব্যবহার করেই `Mango is Yellow` print করতে হবে।

আগের code-এ সমস্যা ছিল arrow function object-এর সরাসরি property value হিসেবে define ছিল।

আমাদের arrow function-এর lexical scope এক level নিচে নিয়ে যেতে হবে।

আমরা একটি regular method-এর ভিতরে arrow function তৈরি করতে পারি:

```js
const food = {
  name: "Mango",
  color: "Yellow",

  getDescription: function () {
    return () => {
      return `${this.name} is ${this.color}`;
    };
  }
};
```

এখন:

```js
const descriptionFn = food.getDescription();
console.log(descriptionFn());
```

চলো reasoning করি।

প্রথমে call হয়েছে:

```js
food.getDescription();
```

`getDescription` regular method।

Implicit binding অনুযায়ী:

```js
this === food
```

এই method-এর ভিতরে arrow function তৈরি হয়েছে:

```js
() => {
  return `${this.name} is ${this.color}`;
}
```

Arrow function-এর নিজস্ব `this` নেই।

তাই এটি lexical parent-এর `this` নেয়।

Lexical parent হলো `getDescription` method-এর execution context।

সেখানে:

```js
this === food
```

তাই arrow function-এর ভিতরেও:

```js
this.name
```

হবে `"Mango"`।

এবং:

```js
this.color
```

হবে `"Yellow"`।

ফলে output:

```text
Mango is Yellow
```

তুমি যদি formula জানো, rule জানো, তাহলে `this`-এর প্রশ্ন ঘুমের মধ্যেও solve করতে পারবে।

এটি কঠিন নয়। দরকার শুধু সঠিক mental model।

JavaScript-এর execution context, scope, closure, object এবং `this`—সব concept একে অন্যের সঙ্গে connected।

JavaScript এক ঘণ্টায় পুরো শেখানো যায় না। Deepভাবে একবার শিখলে এবং concept-এর connection বুঝলে সেটি দীর্ঘদিন মনে থাকে।

# Closure এবং `this`

এখন লক্ষ্য করো:

```js
const descriptionFn = food.getDescription();
```

`getDescription()` execution শেষ হয়ে গেছে।

কিন্তু পরে:

```js
descriptionFn();
```

call করলে arrow function এখনো outer method-এর `this` access করতে পারছে।

এটি closure-এর behavior-এর সঙ্গে যুক্ত।

আমরা closure lesson-এ শিখেছি:

> Outer function execution শেষ হয়ে গেলেও inner function outer scope-এর data ধরে রাখতে পারে।

এখানে inner arrow function outer method-এর `this` lexicalভাবে capture করেছে।

তাই outer method execution শেষ হওয়ার পরও arrow function `food` object-এর context ব্যবহার করতে পারছে।

# এখন পর্যন্ত Rules-এর Recap

চলো দ্রুত revise করি।

## Global Scope

Browser environment-এর classic script context-এ:

```js
this === window
```

## Standalone Regular Function

Non-strict mode:

```js
this === window
```

Strict mode:

```js
this === undefined
```

## Implicit Binding

```js
object.method();
```

Method regular function হলে:

```js
this === object
```

## Arrow Function

Arrow function-এর নিজস্ব `this` নেই।

এটি দেখে:

> কোথায় lexically define করা হয়েছে?

তার surrounding scope-এর `this` নেয়।

## Arrow Function সরাসরি Object Property হলে

```js
const obj = {
  method: () => this
};
```

এখানে `this` সাধারণত `obj` নয়।

## Regular Method-এর ভিতরে Arrow Function হলে

```js
const obj = {
  method() {
    return () => this;
  }
};
```

`obj.method()` হিসেবে call হলে outer method-এর `this` হবে `obj`, এবং arrow function সেই `this` capture করবে।

এখন আমরা explicit binding শিখব।

---

# Explicit Binding

Implicit binding-এ method object-এর অংশ হিসেবে call হয়।

কিন্তু কখনো function এবং object সম্পূর্ণ আলাদা থাকে।

উদাহরণ:

```js
function greeting() {
  console.log(`Hello ${this.name} belongs to ${this.address}`);
}
```

আর আলাদা object:

```js
const user = {
  name: "TapasScript",
  address: "all of you"
};
```

`greeting` function এবং `user` object-এর মধ্যে কোনো direct relationship নেই।

`greeting` object-এর method নয়।

কিন্তু আমরা চাই:

> `greeting` function-টি `user` object-এর context-এ execute হোক।

অর্থাৎ function-এর ভিতরের `this` যেন `user` object-কে refer করে।

এই situation-এ explicit binding ব্যবহার করা হয়।

## Explicit Binding-এর Definition

> Function এবং object স্বাভাবিকভাবে related না হলেও, function-এর `this`-কে ইচ্ছাকৃতভাবে কোনো external object-এর সঙ্গে bind করাকে explicit binding বলা হয়।

Explicit binding করা যায়:

- `call()`
- `apply()`
- `bind()`

---

# `call()` Method

আগের function:

```js
function greeting() {
  console.log(`Hello ${this.name} belongs to ${this.address}`);
}
```

Object:

```js
const user = {
  name: "TapasScript",
  address: "all of you"
};
```

এখন লিখি:

```js
greeting.call(user);
```

এখানে `call()` function-এর ভিতরের `this`-কে `user` object-এর সঙ্গে bind করবে।

তারপর function সঙ্গে সঙ্গে execute হবে।

অর্থাৎ execution-এর সময়:

```js
this === user
```

তাই:

```js
this.name
```

হবে `"TapasScript"`।

এবং:

```js
this.address
```

হবে `"all of you"`।

Output:

```text
Hello TapasScript belongs to all of you
```

এটাই `call()` দিয়ে explicit binding।

---

# `call()`-এ Additional Arguments

একটি function লিখি:

```js
function likes(hobby1, hobby2) {
  console.log(`${this.name} likes ${hobby1}, ${hobby2}`);
}
```

একটি object:

```js
const person = {
  name: "Tapas"
};
```

এখন explicit binding:

```js
likes.call(person, "teaching", "blogging");
```

এখানে:

- প্রথম argument `person` → `this`
- দ্বিতীয় argument `"teaching"` → `hobby1`
- তৃতীয় argument `"blogging"` → `hobby2`

Function execution-এর সময়:

```js
this === person
```

তাই output:

```text
Tapas likes teaching, blogging
```

`call()`-এর syntax:

```js
functionName.call(thisArg, arg1, arg2, arg3);
```

---

# `apply()` Method

`call()`-এর একটি limitation হলো—arguments comma-separated দিতে হয়।

যদি অনেক argument থাকে, তাহলে line বড় হয়ে যায়।

ধরা যাক arguments একটি array-এ আছে:

```js
const hobbiesToApply = ["sleeping", "eating"];
```

এখন লিখি:

```js
likes.apply(person, hobbiesToApply);
```

`apply()`-ও `this`-কে `person` object-এর সঙ্গে bind করবে এবং function সঙ্গে সঙ্গে execute করবে।

Output:

```text
Tapas likes sleeping, eating
```

`call()` এবং `apply()`-এর মূল difference:

## `call()`

```js
likes.call(person, "teaching", "blogging");
```

Arguments comma-separated।

## `apply()`

```js
likes.apply(person, ["sleeping", "eating"]);
```

Arguments array হিসেবে।

বাকি behavior একই:

- দুটোই explicit binding করে
- দুটোই function immediately execute করে

মনে রাখার সহজ নিয়ম:

> Apply → Arguments as Array

---

# `bind()` Method

এখন `bind()` শিখি।

`call()` এবং `apply()` function-কে সঙ্গে সঙ্গে execute করে।

কিন্তু `bind()` সঙ্গে সঙ্গে execute করে না।

`bind()` একটি নতুন function return করে।

উদাহরণ:

```js
function newHobbies(hobby1, hobby2) {
  console.log(`${this.name} likes ${hobby1}, ${hobby2}`);
}
```

Object:

```js
const officer = {
  name: "Bob"
};
```

`call()` ব্যবহার করলে:

```js
newHobbies.call(officer, "dancing", "singing");
```

সঙ্গে সঙ্গে output:

```text
Bob likes dancing, singing
```

কিন্তু `bind()` ব্যবহার করলে:

```js
const newFn = newHobbies.bind(
  officer,
  "dancing",
  "singing"
);
```

এখানে `newHobbies` এখনই execute হবে না।

একটি নতুন bound function return হবে, যা `newFn` variable-এ থাকবে।

পরে আমরা call করতে পারি:

```js
newFn();
```

তখন output:

```text
Bob likes dancing, singing
```

তাই rule:

> `bind()` function-কে এখন bind করে, কিন্তু execution পরে করা যায়।

এটি useful যখন:

- এখন function run করতে চাই না
- পরে কোনো condition অনুযায়ী run করতে চাই
- function callback হিসেবে pass করতে চাই
- context preserve করতে চাই

`call()` হলে function তখনই execute হয়ে যাবে।

`bind()` হলে নতুন function পাওয়া যাবে, যা পরে invoke করা যাবে।

---

# `call()`, `apply()`, `bind()`-এর পার্থক্য

| Method | `this` Bind করে | সঙ্গে সঙ্গে Execute করে | Arguments |
|---|---:|---:|---|
| `call()` | হ্যাঁ | হ্যাঁ | Comma-separated |
| `apply()` | হ্যাঁ | হ্যাঁ | Array |
| `bind()` | হ্যাঁ | না | নতুন function return করে |

# `new` Keyword এবং `this`

আগের Day 12 object session-এ আমরা constructor function এবং `new` keyword শিখেছি।

একটি constructor function:

```js
function Cartoon(name, animal) {
  this.name = name;
  this.animal = animal;

  this.log = function () {
    console.log(`${this.name} is a ${this.animal}`);
  };
}
```

এখন object instance তৈরি করি:

```js
const tomCartoon = new Cartoon("Tom", "cat");
```

আরেকটি:

```js
const jerryCartoon = new Cartoon("Jerry", "mouse");
```

Constructor function-এর নাম capital letter দিয়ে শুরু করা একটি common convention।

`new Cartoon(...)` call করলে একটি নতুন object তৈরি হয়।

Constructor execution-এর সময় `this` সেই নতুন object-কে refer করে।

Tom-এর ক্ষেত্রে:

```js
this.name = "Tom";
this.animal = "cat";
```

Jerry-এর ক্ষেত্রে:

```js
this.name = "Jerry";
this.animal = "mouse";
```

এখন:

```js
tomCartoon.log();
```

এখানে method call হয়েছে `tomCartoon` object-এর উপর।

তাই:

```js
this === tomCartoon
```

Output:

```text
Tom is a cat
```

একইভাবে:

```js
jerryCartoon.log();
```

এখানে:

```js
this === jerryCartoon
```

Output:

```text
Jerry is a mouse
```

তাই `new` keyword-এর সঙ্গে `this` নতুন instance-কে refer করে।

# Quick Recap

আমরা `this` keyword-কে বিভিন্ন জায়গায় দেখলাম:

- Global scope-এ `this`
- Standalone function-এ `this`
- Strict mode-এ standalone function-এর `this`
- Object method-এ implicit binding
- Arrow function-এ lexical `this`
- Object-এর মধ্যে arrow function
- Regular function-এর ভিতরে arrow function
- `call()`, `apply()`, `bind()` দিয়ে explicit binding
- `new` keyword দিয়ে তৈরি object-এর context

`this` বিভিন্ন জায়গায় behavior বদলায়, কিন্তু প্রতিটি জায়গার জন্য নির্দিষ্ট rule আছে।

Primary task হিসেবে তোমাদের একটি table তৈরি করতে হবে।

একটি column-এ situation লিখবে।

আরেকটি column-এ লিখবে সেই situation-এ `this` কীভাবে behave করে।

যেমন:

| Situation | `this` |
|---|---|
| Global scope | `window` |
| Standalone function, strict mode | `undefined` |
| Object method | সেই object |
| Arrow function | Lexical parent-এর `this` |
| Explicit binding | Pass করা object |
| Constructor with `new` | নতুন instance |

এই table progress tracker-এ রাখবে এবং practice করবে।

এখন দুইটি tricky interview question solve করি।

# Interview Question 1: Nested Function

ধরা যাক code:

```js
const user = {
  name: "Tapas",

  greet: function () {
    function inner() {
      console.log(`Hello ${this.name}`);
    }

    inner();
  }
};

user.greet();
```

প্রশ্ন:

> Output কি `Hello Tapas` হবে?

প্রথমে `user.greet()` call হয়েছে।

`greet` regular method এবং `user` object-এর উপর call হয়েছে।

তাই `greet` method-এর ভিতরে:

```js
this === user
```

যদি আমরা সরাসরি `greet` method-এর ভিতরে লিখতাম:

```js
console.log(this.name);
```

তাহলে output হতো:

```text
Tapas
```

কিন্তু actual code-এ `this` ব্যবহার করা হয়েছে inner function-এর ভিতরে।

```js
function inner() {
  console.log(`Hello ${this.name}`);
}
```

এবং inner function call হয়েছে:

```js
inner();
```

এটি কোনো object method call নয়।

এটি standalone regular function call।

তাই non-strict browser mode-এ:

```js
this === window
```

Strict mode-এ:

```js
this === undefined
```

অর্থাৎ inner function-এর `this`, `user` object-কে refer করছে না।

তাই `Hello Tapas` পাওয়া যাবে না।

---

## Arrow Function দিয়ে Fix

Inner regular function-কে arrow function-এ পরিবর্তন করি:

```js
const user = {
  name: "Tapas",

  greet: function () {
    const inner = () => {
      console.log(`Hello ${this.name}`);
    };

    inner();
  }
};

user.greet();
```

এখন কী হবে?

`user.greet()` → implicit binding

তাই outer method-এর ভিতরে:

```js
this === user
```

`inner` arrow function-এর নিজস্ব `this` নেই।

Arrow function lexically `greet` method-এর ভিতরে define হয়েছে।

তাই এটি outer method-এর `this` capture করে।

ফলে:

```js
this.name
```

হবে `"Tapas"`।

Output:

```text
Hello Tapas
```

Arrow function শুধু syntax ছোট করার জন্য নয়। Lexical `this` behavior JavaScript-এর বহু পুরোনো nested function problem সহজ করে দেয়।

# Interview Question 2: Detached Method

Code:

```js
const obj = {
  name: "John",

  greet: function () {
    console.log(`Hello ${this.name}`);
  }
};
```

যদি আমরা লিখি:

```js
obj.greet();
```

তাহলে implicit binding হবে।

```js
this === obj
```

Output:

```text
Hello John
```

কিন্তু code যদি হয়:

```js
const greetFn = obj.greet;
greetFn();
```

এখানে কী হবে?

এই line:

```js
const greetFn = obj.greet;
```

শুধু function reference নিয়েছে।

Function execute হয়নি।

পরে:

```js
greetFn();
```

standaloneভাবে execute হয়েছে।

এখন call site-এ `obj.` নেই।

Function object থেকে detached হয়ে গেছে।

তাই implicit binding হারিয়েছে।

Strict mode-এ:

```js
this === undefined
```

Non-strict browser context-এ:

```js
this === window
```

তাই `Hello John` পাওয়া যাবে না।

---

## `call()` দিয়ে Fix

```js
greetFn.call(obj);
```

এখানে explicit binding হবে:

```js
this === obj
```

Output:

```text
Hello John
```

---

## `bind()` দিয়ে Fix

```js
const boundGreet = obj.greet.bind(obj);
boundGreet();
```

এখানে `bind()` একটি নতুন function return করবে, যার `this` `obj`-এর সঙ্গে bound থাকবে।

পরে call করলে:

```text
Hello John
```

# Appendix: `this` Rules Table

| Situation | `this`-এর Value |
|---|---|
| Browser global scope | `window` |
| Standalone regular function, non-strict | `window` |
| Standalone regular function, strict mode | `undefined` |
| `obj.method()` regular method | `obj` |
| Arrow function | Lexical surrounding scope-এর `this` |
| Arrow function directly inside object literal | সাধারণত object নয় |
| Arrow inside regular object method | Outer method-এর `this` capture করে |
| `fn.call(obj)` | `obj`, এবং function immediately execute হয় |
| `fn.apply(obj, args)` | `obj`, এবং function immediately execute হয় |
| `fn.bind(obj)` | `obj`-bound নতুন function return করে |
| `new Constructor()` | নতুন তৈরি হওয়া instance |
| Detached method | Original object binding হারায় |
