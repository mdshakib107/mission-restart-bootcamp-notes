# 2. JavaScript Object কী এবং কেন দরকার

শুরুতে আমরা data সংরক্ষণ করেছি simple variable-এ। যেমন:

```js
const age = 40;
const isAdmin = true;
const name = "Tapas";
```

এগুলো primitive data type:

- number
- string
- boolean
- null
- undefined
- bigint
- symbol

কিন্তু real-world application-এ অনেক সময় complex data সংরক্ষণ করতে হয়। যেমন:

- User data
- Employee data
- Company data
- Product data
- Student data

একজন user-এর শুধু name নয়; age, address, email, role, preferences—অনেক তথ্য থাকতে পারে। আলাদা আলাদা variable ব্যবহার করলে data ছড়িয়ে যায় এবং manage করা কঠিন হয়।

## Instructor-এর definition

> **Objects are nothing but key-value pairs.**

আরও নির্দিষ্টভাবে:

> Object হলো একটি **keyed collection**, যেখানে প্রতিটি value একটি unique key ব্যবহার করে চিহ্নিত ও retrieve করা যায়।

উদাহরণ:

```js
const user = {
  name: "Tapas",
  age: 40,
};
```

এখানে:

| Property | Key | Value |
|---|---|---|
| প্রথম property | `name` | `"Tapas"` |
| দ্বিতীয় property | `age` | `40` |

## সহজ ব্যাখ্যা

Object-কে একটি labelled box-এর মতো ভাবা যায়। প্রতিটি label একটি key এবং label-এর সঙ্গে থাকা তথ্যটি value।

## Common mistake

শুধু value দেখে object-এর data বোঝার চেষ্টা করা। Object-এর মূল শক্তি হলো key-এর মাধ্যমে value সংগঠিত করা।

## মনে রাখার নিয়ম

> Object = `{ key: value }`

# 3. Object Literal Syntax দিয়ে Object তৈরি

Object তৈরির সবচেয়ে সরাসরি ও সবচেয়ে common উপায় হলো **object literal syntax**।

```js
const user = {};
```

শুধু `{}` লিখলেই একটি object তৈরি হয়। কোনো property না থাকলে একে empty object বলা হয়।

```js
const user = {
  name: "Tapas",
  age: 40,
};
```

এখানে `name` ও `age` হলো properties। প্রতিটি property-তে থাকে:

```text
key: value
```

একাধিক property comma দিয়ে আলাদা করা হয়।

## গুরুত্বপূর্ণ ধারণা

Object নিজেও একটি value। তাই object-কে variable-এ assign করা যায়।

```js
const user = {
  name: "Tapas",
};
```

এখানে `user` variable একটি object value ধরে রেখেছে।

## Common mistake

```js
const user = {
  name = "Tapas",
};
```

এটি ভুল, কারণ object property-তে assignment operator `=` নয়, colon `:` ব্যবহার করতে হয়।

সঠিক:

```js
const user = {
  name: "Tapas",
};
```

# 4. Object Property Access: Dot Notation ও Bracket Notation

Object থেকে value retrieve করতে প্রধানত দুইটি syntax আছে:

1. Dot notation
2. Bracket notation / Subscript notation

## 4.1 Dot Notation

```js
const user = {
  name: "Tapas",
  age: 40,
};

console.log(user.name); // Tapas
console.log(user.age);  // 40
```

Instructor-এর ভাষায়, dot operator ব্যবহার করে object-এর একটি key-এর value query করা যায়।

```text
object.key
```

## 4.2 Bracket Notation

```js
console.log(user["name"]); // Tapas
console.log(user["age"]);  // 40
```

```text
object["key"]
```

## Dot বনাম Bracket

| বিষয় | Dot notation | Bracket notation |
|---|---|---|
| সাধারণ key | ব্যবহার করা যায় | ব্যবহার করা যায় |
| Space-যুক্ত key | ব্যবহার করা যায় না | ব্যবহার করা যায় |
| Hyphen/special character | সাধারণত ব্যবহার করা যায় না | ব্যবহার করা যায় |
| Dynamic variable key | ব্যবহার করা যায় না | ব্যবহার করা যায় |
| Syntax | ছোট ও পরিষ্কার | flexible |

## মনে রাখার নিয়ম

> Static ও valid identifier key হলে dot notation। Special বা dynamic key হলে bracket notation।

# 5. নতুন Property যোগ, পরিবর্তন ও মুছে ফেলা

Object তৈরি হওয়ার পরেও property যোগ, modify এবং delete করা যায়।

## 5.1 নতুন Property যোগ করা

```js
const user = {
  name: "Tapas",
  age: 40,
};

user.isSeniorCitizen = false;
```

এখন object:

```js
{
  name: "Tapas",
  age: 40,
  isSeniorCitizen: false
}
```

## 5.2 Existing Property Modify করা

```js
user.age = 34;
```

এখন `age` 40 থেকে 34 হয়েছে।

Bracket notation দিয়েও modify করা যায়:

```js
user["movie lover"] = false;
```

## 5.3 Property Delete করা

JavaScript-এর `delete` keyword ব্যবহার করা হয়।

```js
delete user.age;
```

Special key হলে:

```js
delete user["movie lover"];
```

## Common mistake

```js
user.age = undefined;
```

এটি property delete করে না। Property object-এর মধ্যে থাকে, শুধু value `undefined` হয়।

সত্যিকারের delete:

```js
delete user.age;
```

## মনে রাখার নিয়ম

- Add/Update: `object.key = value`
- Delete: `delete object.key`

# 6. Special Character-যুক্ত Key

সাধারণ key quote ছাড়াই লেখা যায়:

```js
const user = {
  name: "Tapas",
  age: 40,
};
```

JavaScript internally key-কে string হিসেবে treat করে।

কিন্তু key-এর মধ্যে space বা special character থাকলে quote দিতে হবে।

```js
const user = {
  "is admin": true,
};
```

এটি access করতে হবে bracket notation দিয়ে:

```js
console.log(user["is admin"]); // true
```

নিচেরটি ভুল:

```js
console.log(user.is admin);
```

কারণ JavaScript `user.is` পর্যন্ত একটি expression হিসেবে বুঝে, এরপর `admin` দেখে syntax error দেয়।

## নতুন special key যোগ করা

```js
user["movie lover"] = true;
```

## Common mistake

```js
user.movie lover = true;
```

এটি invalid syntax।

## মনে রাখার নিয়ম

> Key-তে space থাকলে quote + bracket notation বাধ্যতামূলক।

# 7. Dynamic Key দিয়ে Property Access

Real-world programming-এ key hardcoded নাও হতে পারে। এটি আসতে পারে:

- User input থেকে
- API response থেকে
- Form field থেকে
- Function argument থেকে
- Runtime computation থেকে

ধরা যাক:

```js
const user = {
  name: "Tapas",
  age: 34,
};

const someKey = "age";
```

ভুল:

```js
console.log(user.someKey);
```

এখানে JavaScript `someKey`-কে variable হিসেবে নয়, literal property name হিসেবে খুঁজবে। অর্থাৎ `user` object-এ `someKey` নামে property আছে কি না দেখবে।

সঠিক:

```js
console.log(user[someKey]); // 34
```

Runtime-এ `someKey`-এর value `"age"` বসে যায়। Expression কার্যত হয়:

```js
user["age"]
```

## Common mistake

Dynamic key-এর ক্ষেত্রে dot notation ব্যবহার করা।

## মনে রাখার নিয়ম

> Variable-এর value-কে key হিসেবে ব্যবহার করতে হলে bracket notation।

# 8. Computed/Dynamic Property Name দিয়ে Object তৈরি

শুধু dynamic key access নয়, dynamic key দিয়ে object property তৈরি করাও যায়।

```js
const car = prompt("Which is your favorite car?");

const favoriteCars = {
  [car]: 5,
};
```

User যদি `BMW` লেখে, object হবে:

```js
{
  BMW: 5
}
```

User যদি `Audi` লেখে, object হবে:

```js
{
  Audi: 5
}
```

এখানে `[car]`-কে **computed property name** বলা হয়।

## সহজ ব্যাখ্যা

Square bracket-এর মধ্যে expression দিলে JavaScript আগে expression evaluate করে, তারপর result-কে key বানায়।

## আরও উদাহরণ

```js
const prefix = "user";
const id = 101;

const data = {
  [`${prefix}_${id}`]: "Active",
};

console.log(data); // { user_101: "Active" }
```

# 9. Object তৈরির বিভিন্ন Pattern

লেকচারে object তৈরির কয়েকটি উপায় দেখানো হয়েছে:

| Pattern | `new` দরকার? | Custom type পাওয়া যায়? | Common usage |
|---|---:|---:|---|
| Object literal | না | না | সবচেয়ে common |
| Constructor function | হ্যাঁ | হ্যাঁ | Blueprint থেকে instance |
| `new Object()` | হ্যাঁ | না, generic Object | সাধারণত কম ব্যবহার |
| Factory function | না | না | Function থেকে নতুন object return |

Prototype এবং OOP-এর গভীর আলোচনা পরে হবে। এই lesson-এ syntax ও practical difference বোঝানো হয়েছে।

# 10. Constructor Function

Constructor function দেখতে সাধারণ function-এর মতো, কিন্তু এর উদ্দেশ্য object instance তৈরি করা।

## Convention

Constructor function-এর নাম capital letter দিয়ে শুরু করা হয়।

```js
function Car(name, model) {
  this.name = name;
  this.model = model;
}
```

এখানে:

- `name`, `model` হলো parameter
- `this.name`, `this.model` হলো নতুন object-এর property
- `this` keyword পরবর্তী lesson-এ বিস্তারিত শেখানো হবে

Object তৈরি করতে `new` keyword ব্যবহার করতে হয়:

```js
const bmwCar = new Car("BMW", "X1");
const audiCar = new Car("Audi", "A8");
```

Output ধারণা:

```js
Car { name: "BMW", model: "X1" }
Car { name: "Audi", model: "A8" }
```

## Blueprint analogy

Instructor constructor function-কে একটি blueprint হিসেবে ব্যাখ্যা করেছেন। একটি blueprint থেকে একাধিক instance তৈরি করা যায়।

- `Car` = blueprint
- `bmwCar` = একটি instance
- `audiCar` = আরেকটি instance

## `instanceof`

```js
console.log(bmwCar instanceof Car); // true
```

এটি যাচাই করে object-টি কোন constructor/blueprint থেকে তৈরি হয়েছে।

## Common mistakes

### ভুল ১: Capital letter convention না মানা

```js
function car(name, model) {}
```

Technically চলতে পারে, কিন্তু regular function ও constructor function আলাদা বোঝা কঠিন হয়।

### ভুল ২: `new` না ব্যবহার করা

```js
const car = Car("BMW", "X1");
```

Constructor function invoke করার সময় `new` না দিলে expected object তৈরি হবে না এবং `this`-এর behavior ভিন্ন হতে পারে।

## মনে রাখার নিয়ম

> Constructor = Capital name + `this` + `new`

# 11. Object Constructor

JavaScript-এর built-in `Object` constructor দিয়েও object তৈরি করা যায়।

```js
const person = new Object();

person.name = "Alex";
person.age = 76;
```

এটি একটি generic object তৈরি করে।

```js
console.log(person);
```

## সীমাবদ্ধতা

Custom constructor-এর মতো `Car`, `User`, `Employee` ইত্যাদি custom type বোঝা যায় না। এটি শুধু generic `Object`।

## Practical advice

সাধারণ object তৈরির জন্য object literal বেশি clean:

```js
const person = {
  name: "Alex",
  age: 76,
};
```

# 12. Factory Function

Factory function হলো এমন একটি regular function, যা call করলে নতুন object return করে।

Instructor closure lesson-এর সঙ্গে factory function-এর connection মনে করিয়ে দেন। Input অনুযায়ী factory function বিভিন্ন object তৈরি করতে পারে।

```js
function createUser(name, age) {
  return {
    name,
    age,
  };
}
```

ব্যবহার:

```js
const user1 = createUser("Tapas", 39);
const user2 = createUser("Bob", 32);
```

এখানে `new` keyword লাগে না।

## Constructor বনাম Factory Function

| বিষয় | Constructor Function | Factory Function |
|---|---|---|
| Naming convention | Capital letter | সাধারণ function naming |
| `new` keyword | আবশ্যক | প্রয়োজন নেই |
| Property setup | `this.property` | Object return |
| Custom instance type | পাওয়া যায় | সাধারণত পাওয়া যায় না |
| Call syntax | `new Car(...)` | `createUser(...)` |

## Common mistake

Factory function call করার সময় অপ্রয়োজনীয়ভাবে `new` ব্যবহার করা।

# 13. Object Property Shorthand

যখন property key এবং variable/parameter-এর নাম একই হয়, তখন shorthand ব্যবহার করা যায়।

পূর্ণ syntax:

```js
function createUser(name, age) {
  return {
    name: name,
    age: age,
  };
}
```

Shorthand:

```js
function createUser(name, age) {
  return {
    name,
    age,
  };
}
```

দুটোর output একই।

## শর্ত

Key name ও variable name একই হতে হবে।

```js
const userName = "Tapas";

const user = {
  name: userName,
};
```

এখানে শুধু `name` লিখলে হবে না, কারণ variable-এর নাম `userName`।

## মনে রাখার নিয়ম

> `name: name` → `name`

# 14. Object Method

Object-এর property value primitive-ই হতে হবে—এমন নয়। Value হতে পারে:

- Object
- Array
- Function

Object-এর ভিতরে function থাকলে সেই function-কে **method** বলা হয়।

## Instructor-এর terminology

> Whenever you have a function inside an object, you call that function a method.

উদাহরণ:

```js
const profile = {
  name: "Tapas",
  company: "Creo",
  message: function () {
    console.log(`${this.name} works at ${this.company}`);
  },
};
```

Method invoke:

```js
profile.message();
```

Output:

```text
Tapas works at Creo
```

## Function access বনাম invoke

```js
profile.message;   // function reference
profile.message(); // function execute/invoke
```

## Template Literal

```js
`${this.name} works at ${this.company}`
```

Dynamic value `${...}` syntax-এর মধ্যে লেখা হয়। Runtime-এ value replace হয়।

## Common mistake

Parentheses ছাড়া method execute হবে না।

```js
profile.message; // শুধু function reference
```

# 15. Nested Object

একটি object-এর property value যদি আরেকটি object হয়, তাকে nested object বলা হয়।

## Instructor-এর definition

> An object can have a property whose value can be another object.

উদাহরণ:

```js
const profile = {
  name: "Tapas",
  company: "Creo",
  address: {
    city: "Bangalore",
    zip: 560032,
    state: "Karnataka",
    country: "India",
  },
};
```

এখানে `address` হলো nested object।

## Nested property access

```js
console.log(profile.address.country); // India
```

আরও nested হলে chain বাড়বে:

```js
object.level1.level2.level3
```

## Nested method

```js
const profile = {
  address: {
    country: "India",
    greeting: function () {
      console.log("Welcome to India");
    },
  },
};

profile.address.greeting();
```

## গুরুত্বপূর্ণ ধারণা

Nested object complex data structure represent করতে সাহায্য করে। Depth প্রয়োজন অনুযায়ী যত খুশি হতে পারে, তবে অতিরিক্ত গভীর nesting code maintain করা কঠিন করতে পারে।

# 16. `in` Operator দিয়ে Property Existence Check

কোনো object-এ property আছে কি না জানতে অনেকেই value check করে:

```js
if (!profile.salary) {
  console.log("The salary property does not exist");
}
```

কিন্তু এটি reliable নয়। কারণ property থাকতে পারে, কিন্তু value হতে পারে:

- `undefined`
- `0`
- `false`
- `""`
- `null`

উদাহরণ:

```js
const profile = {
  salary: undefined,
};
```

এখানে `profile.salary` হলো `undefined`, কিন্তু property আসলে আছে।

## সঠিক উপায়: `in` operator

```js
console.log("salary" in profile); // true
```

Property না থাকলে:

```js
console.log("address" in profile); // false
```

## Syntax

```js
"propertyName" in object
```

## Common mistake

```js
salary in profile
```

যদি `salary` নামে variable না থাকে, error হবে। Property name string হিসেবে দিতে হবে:

```js
"salary" in profile
```

## মনে রাখার নিয়ম

> Value আছে কি না নয়; key আছে কি না জানতে `in` ব্যবহার করো।

# 17. `for...in` Loop

Object-এর সব key iterate করতে `for...in` loop ব্যবহার করা যায়।

```js
const profile = {
  name: "Tapas",
  company: "Creo",
  age: 39,
};

for (let key in profile) {
  console.log(key);
}
```

Output:

```text
name
company
age
```

Value পেতে dynamic bracket notation:

```js
for (let key in profile) {
  console.log(key, profile[key]);
}
```

## কেন `profile[key]`?

কারণ `key` runtime-এ বদলায়। তাই dot notation নয়, bracket notation দরকার।

ভুল:

```js
profile.key
```

এটি প্রতিবার `key` নামের property খুঁজবে।

## `for...in` বনাম `for...of`

| Loop | সাধারণত কোথায় | কী দেয় |
|---|---|---|
| `for...in` | Object | key/property name |
| `for...of` | Array/iterable | element/value |

# 19. Object Reference ও Object Comparison

Object হলো non-primitive data type। Non-primitive value memory-তে একটি location-এ store হয় এবং variable সেই memory location-এর reference ধরে।

Instructor execution context lesson-এর memory diagram মনে করিয়ে দেন। Object heap memory-তে store হয় এবং reference-এর মাধ্যমে access করা হয়।

## একই value, ভিন্ন object

```js
const fruit = { name: "Mango" };
const anotherFruit = { name: "Mango" };

console.log(fruit === anotherFruit); // false
console.log(fruit == anotherFruit);  // false
```

কেন false?

- `fruit` এক memory location নির্দেশ করে
- `anotherFruit` অন্য memory location নির্দেশ করে
- value দেখতে একই হলেও reference আলাদা

## একই reference

```js
let fruit = { name: "Mango" };
let anotherFruit = fruit;

console.log(fruit === anotherFruit); // true
```

এখন দুইটি variable একই object reference করছে।

## গুরুত্বপূর্ণ নিয়ম

Object comparison value দিয়ে নয়, reference দিয়ে হয়।

## `const` প্রসঙ্গ

```js
const fruit = { name: "Mango" };
fruit.name = "Apple"; // allowed
```

Object-এর property modify করা যায়। কিন্তু variable reassign করা যায় না:

```js
fruit = { name: "Banana" }; // error
```

Instructor reassignment দেখানোর জন্য `let` ব্যবহার করেছেন।

## Common mistake

দুইটি object-এর content একই দেখে `===` true হবে মনে করা।

# 20. `Object.assign()`

## Instructor-এর definition

> `Object.assign()` copies all of an object's own properties from a source object to a target object.

Syntax:

```js
Object.assign(target, source)
```

উদাহরণ:

```js
const target = {
  p: 1,
  q: 2,
};

const source = {
  a: 3,
  b: 5,
};

const returnedObject = Object.assign(target, source);

console.log(returnedObject);
```

Output:

```js
{
  p: 1,
  q: 2,
  a: 3,
  b: 5
}
```

## Key conflict হলে override

```js
const target = {
  p: 1,
  a: 2,
};

const source = {
  a: 3,
  b: 5,
};

Object.assign(target, source);
```

Final target:

```js
{
  p: 1,
  a: 3,
  b: 5
}
```

Source-এর `a: 3`, target-এর `a: 2`-কে override করেছে।

## Clone-এর জন্য

```js
const obj = {
  name: "Tapascript",
};

const obj2 = Object.assign({}, obj);
```

এখানে empty object target এবং `obj` source।

```js
console.log(obj === obj2); // false
```

Values একই, reference আলাদা।

# 21. Shallow Copy বনাম Deep Copy

`Object.assign()` একটি **shallow copy** তৈরি করে।

## Instructor-এর definition

> `Object.assign()` performs a shallow copy. For nested objects, it copies the reference instead of creating a new nested copy.

উদাহরণ:

```js
const obj3 = {
  a: 1,
  b: {
    c: 2,
  },
};

const obj4 = Object.assign({}, obj3);
```

Top-level property `a` স্বাধীনভাবে copy হয়। কিন্তু nested object `b`-এর reference shared থাকে।

```js
obj4.b.c = 3;

console.log(obj3.b.c); // 3
console.log(obj4.b.c); // 3
```

এটি bug নয়; shallow copy-এর expected behavior।

Top-level property modify করলে:

```js
obj4.a = 100;

console.log(obj3.a); // 1
console.log(obj4.a); // 100
```

## Diagram ধারণা

```text
obj3.b ─────┐
            ├──> একই nested object
obj4.b ─────┘
```

## Shallow copy

- Top-level নতুন object
- Nested object/array reference shared হতে পারে

## Deep copy

- Top-level নতুন object
- Nested object/array-ও নতুন reference

## Common mistake

Nested object clone করতে `Object.assign()` ব্যবহার করে সম্পূর্ণ independent copy আশা করা।

# 22. `structuredClone()`

Deep cloning-এর জন্য instructor `structuredClone()` ব্যবহারের পরামর্শ দেন।

```js
const obj3 = {
  a: 1,
  b: {
    c: 2,
  },
};

const obj5 = structuredClone(obj3);
```

এখন:

```js
obj5.a = 300;
obj5.b.c = 30;

console.log(obj5.a);   // 300
console.log(obj3.a);   // 1
console.log(obj5.b.c); // 30
console.log(obj3.b.c); // 2
```

Nested object-ও independent reference পেয়েছে।

## কখন ব্যবহার করবেন

- Nested object clone
- Nested array clone
- Immutable update workflow
- Source data পরিবর্তন না করে copy নিয়ে কাজ

## সতর্কতা

`structuredClone()` সব ধরনের value clone করতে পারে না। Function সাধারণত clone করা যায় না। তবে এই lesson-এর মূল ফোকাস nested object deep copy বোঝা।

# 23. `Object.entries()`

`Object.entries()` object-কে key-value pair-এর array-তে convert করে।

```js
const person = {
  a: "Tapas",
  b: 32,
};

const myArray = Object.entries(person);
```

Output:

```js
[
  ["a", "Tapas"],
```

প্রতিটি inner array-তে:

- index 0 = key
- index 1 = value

## Real-world use case

```js
Object.entries(apiResponse)
```

তারপর `for...of`, `map()`, ইত্যাদি ব্যবহার করা যায়।

# 24. `Object.fromEntries()`

`Object.fromEntries()` হলো `Object.entries()`-এর বিপরীত কাজ। Key-value pair-এর array বা Map থেকে object তৈরি করে।

```js
const entries = [
  ["foo", "bar"],
  ["baz", 42],
];

const obj = Object.fromEntries(entries);
console.log(obj);
```

Output:

```js
{
  foo: "bar",
  baz: 42
}
```

## Relationship

```js
const object = { a: 1, b: 2 };
const entries = Object.entries(object);
const rebuiltObject = Object.fromEntries(entries);
```

# 25. `Object.freeze()`

## Instructor-এর definition

> Immutable means something that you cannot change. Mutable means something that you can change.

`Object.freeze()` object-কে এমনভাবে freeze করে যাতে:

- Existing property value modify করা যায় না
- নতুন property add করা যায় না
- Existing property delete করা যায় না

```js
const employee = {
  salary: 100,
};

Object.freeze(employee);

employee.salary = 200;
employee.name = "Alex";
delete employee.salary;

console.log(employee);
```

Output remains:

```js
{
  salary: 100
}
```

## Frozen কি না পরীক্ষা

```js
console.log(Object.isFrozen(employee)); // true
```

## গুরুত্বপূর্ণ সীমাবদ্ধতা

`Object.freeze()` shallow। Nested object থাকলে inner object আলাদাভাবে freeze না করলে modify হতে পারে। এই specific nuance transcript-এ গভীরভাবে দেখানো হয়নি, কিন্তু production code-এ গুরুত্বপূর্ণ।

# 26. `Object.seal()`

`Object.seal()`-ও object-এর structure রক্ষা করে, কিন্তু `freeze()`-এর তুলনায় কিছুটা flexible।

Seal করলে:

- নতুন property add করা যায় না
- Existing property delete করা যায় না
- Existing property-এর value modify করা যায়

```js
const department = {
  name: "Finance",
};

Object.seal(department);

department.address = "Bangalore"; // add হবে না
delete department.name;           // delete হবে না
department.name = "HR";           // modify হবে

console.log(department);
```

Output:

```js
{
  name: "HR"
}
```

## Freeze বনাম Seal

| Operation | `Object.freeze()` | `Object.seal()` |
|---|---:|---:|
| Existing value modify | না | হ্যাঁ |
| New property add | না | না |
| Property delete | না | না |
| Structure lock | হ্যাঁ | হ্যাঁ |

## মনে রাখার নিয়ম

- Freeze = value + structure lock
- Seal = শুধু structure lock; existing value বদলানো যায়

# 27. `Object.hasOwn()`

`Object.hasOwn()` নির্দিষ্ট object-এর own property আছে কি না true/false return করে।

```js
const department = {

};

console.log(Object.hasOwn(department, "name"));    // true
console.log(Object.hasOwn(department, "address")); // false
```

## Syntax

```js
Object.hasOwn(object, "propertyName")
```

## `in` বনাম `Object.hasOwn()`

| Method | Own property | Inherited property |
|---|---:|---:|
| `"key" in object` | true | true হতে পারে |
| `Object.hasOwn(object, "key")` | true | false |

Prototype lesson-এ inherited property আরও পরিষ্কার হবে।

# 28. Object Destructuring পরিচিতি

## Instructor-এর ব্যাখ্যা

`destructure` মানে একটি structure ভেঙে তার অংশগুলো আলাদা করে নেওয়া। Object-এর ক্ষেত্রে property value-গুলো বের করে variable-এ নেওয়া হয়।

ধরা যাক student object:

```js
const student = {
  name: "John Williamson",
  age: 9,
  standard: 3,
  subjects: ["Math", "English", "Science"],
  parents: {
    father: "Mr. Williamson",
    mother: "Mrs. Williamson",
    email: "john.parents@abcd.com",
  },
  address: {
    street: "1st Main Road",
    city: "John City",
    country: "USA",
    zip: 5791,
  },
};
```

Traditional access:

```js
const name = student.name;
const age = student.age;
```

Destructuring:

```js
const { name, age } = student;
```

JavaScript automatically:

- `name` নামে variable তৈরি করে
- `student.name` assign করে
- `age` নামে variable তৈরি করে
- `student.age` assign করে

## মূল সুবিধা

- কম code
- পরিষ্কার syntax
- Repeated access কমে
- Function parameter clean হয়
- API response handling সহজ হয়

---

# 29. Destructuring Use Case 1: Default Value

Object-এ property না থাকলেও destructuring-এর সময় default value দেওয়া যায়।

```js
const { name, age, meal = "bread" } = student;
```

যদি `student.meal` না থাকে, তাহলে:

```js
meal === "bread"
```

যদি object-এ property থাকে:

```js
const student = {
  meal: "rice",
};

const { meal = "bread" } = student;
console.log(meal); // rice
```

Default value শুধু property `undefined` হলে ব্যবহৃত হয়।

## Traditional দীর্ঘ code

```js
const meal = student.meal ? student.meal : "bread";
```

Destructuring syntax অনেক ছোট।

## Common mistake

Default value property-এর existing valid falsy value-কে replace করবে মনে করা।

```js
const obj = { count: 0 };
const { count = 10 } = obj;
console.log(count); // 0
```

---

# 30. Destructuring Use Case 2: Dynamic Default Value

Default value static string/number হতে হবে—এমন নয়। Expression থেকেও আসতে পারে।

```js
const { subjects, numberOfSubjects = subjects.length } = student;
```

যদি `numberOfSubjects` property না থাকে, তাহলে `subjects.length` evaluate হবে।

```js
console.log(numberOfSubjects); // 3
```

## গুরুত্বপূর্ণ ordering

একই destructuring pattern-এর মধ্যে আগে destructure করা variable পরের default expression-এ ব্যবহার করা যায়।

```js
const { subjects, count = subjects.length } = student;
```

কিন্তু উল্টো করলে সমস্যা হতে পারে:

```js
const { count = subjects.length, subjects } = student;
```

কারণ default expression evaluate হওয়ার সময় `subjects` initialize না-ও হয়ে থাকতে পারে।

# 31. Destructuring Use Case 3: Alias

Property name conflict এড়াতে alias ব্যবহার করা যায়।

```js
const { standard: studentStandard } = student;
```

এখানে:

- Object property = `standard`
- নতুন variable = `studentStandard`

```js
console.log(studentStandard); // 3
```

এখন `standard` নামে variable তৈরি হয়নি।

```js
console.log(standard); // ReferenceError
```

## Real-world context

Instructor React, Angular, API, GraphQL, REST, webhook-এর উদাহরণ দেন। একাধিক response object-এ একই property যেমন `loading`, `data`, `error` থাকতে পারে। Alias দিয়ে conflict এড়ানো যায়।

```js
const { loading: graphQLLoading } = graphQLResponse;
const { loading: restLoading } = restResponse;
const { loading: webhookLoading } = webhookResponse;
```

## মনে রাখার নিয়ম

```text
propertyName: newVariableName
```

# 32. Destructuring Use Case 4: Nested Object Destructuring

Nested property সরাসরি destructure করা যায়।

Traditional:

```js
const address = student.address;
const zip = address.zip;
```

Destructuring:

```js
const {
  address: { zip },
} = student;
```

এখন `zip` variable-এর value `5791`।

## গুরুত্বপূর্ণ বিষয়

উপরের syntax-এ `address` নামে variable তৈরি হয় না; শুধু `zip` তৈরি হয়।

`address` variable-ও চাইলে:

```js
const { address, address: { zip } } = student;
```

## Deep nesting pattern

```js
const {
  level1: {
    level2: {
      level3: { value },
    },
  },
} = object;
```

## Common mistake

Colon দেখে alias ভেবে confusion হওয়া। Nested destructuring-এ:

```js
address: { zip }
```

মানে `address` property-এর object থেকে `zip` বের করা।

Alias হলে:

```js
zip: postalCode
```

# 33. Destructuring Use Case 5: Function Parameter

Function-এর ভিতরে পুরো object নিয়ে repeated property access না করে parameter level-এ destructure করা যায়।

Traditional:

```js
function sendEmail(student) {
  console.log(`Sent an email to ${student.parents.email}`);
}

sendEmail(student);
```

Destructured parameter:

```js
function sendEmail({ parents: { email } }) {
  console.log(`Sent an email to ${email}`);
}

sendEmail(student);
```

## সুবিধা

- Function কোন data ব্যবহার করছে, signature দেখেই বোঝা যায়
- Repeated long chain লেখা লাগে না
- Code clean হয়

## Common mistake

Function-এ destructuring আছে, কিন্তু argument হিসেবে nested property পাঠানো:

```js
sendEmail(student.parents.email);
```

উপরের function পুরো student-shaped object আশা করছে। তাই call হবে:

```js
sendEmail(student);
```

# 34. Destructuring Use Case 6: Function Return Value

Function object return করলে return value সরাসরি destructure করা যায়।

```js
const getStudent = () => student;
```

Traditional:

```js
const anotherStudent = getStudent();
const anotherName = anotherStudent.name;
const anotherSubjects = anotherStudent.subjects;
```

Destructuring with alias:

```js
const {
  name: anotherName,
  subjects: anotherSubjects,
} = getStudent();
```

এখন:

```js
console.log(anotherName);
console.log(anotherSubjects);
```

## কেন alias দরকার হয়েছিল

Instructor-এর code-এ `name` ও `subjects` variable আগেই declare করা ছিল। একই scope-এ আবার একই `const` name declare করলে error হবে। তাই alias নেওয়া হয়েছে।

# 35. Destructuring Use Case 7: Loop-এর ভিতরে

Array-এর প্রতিটি element যদি object হয়, `for...of` loop-এর variable অংশে destructuring করা যায়।

```js
const students = [
  { name: "William", grade: "A" },
  { name: "Tom", grade: "A+" },
  { name: "Bob", grade: "B" },
];

for (let { name, grade } of students) {
  console.log(name, grade);
}
```

Output:

```text
William A
Tom A+
Bob B
```

প্রতিটি iteration-এ:

1. একটি student object নেওয়া হয়
2. `name` ও `grade` destructure হয়
3. ব্যবহার করা হয়

## `for...in` বনাম `for...of` পুনরায়

- Object properties iterate: `for...in`
- Array elements iterate: `for...of`

# 36. Optional Chaining

Optional chaining ES2020-এ introduce হওয়া syntax। Nested property access করার সময় intermediate value `null` বা `undefined` হলে runtime error এড়াতে এটি ব্যবহার করা হয়।

## Problem

```js
const employee = {
  salary: 100,
};

console.log(employee.department); // undefined
console.log(employee.department.name);
```

শেষ লাইন error দেবে:

```text
Cannot read properties of undefined (reading 'name')
```

কারণ `employee.department` নিজেই `undefined`। `undefined.name` access করা যায় না।

## Optional Chaining Syntax

```js
const name = employee.department?.name;
console.log(name); // undefined
```

`?.` হলো optional chaining operator।

এখানে logic:

- `department` যদি `null` বা `undefined` হয়, পরের property access বন্ধ
- Error না দিয়ে `undefined` return

## পুরনো conditional approach

```js
const name = employee.department
  ? employee.department.name
  : undefined;
```

Optional chaining:

```js
const name = employee.department?.name;
```

## কখন ব্যবহার করবেন

- API response incomplete হতে পারে
- Nested optional property
- User profile-এর optional fields
- Backend সব data নাও পাঠাতে পারে
- Graceful fallback প্রয়োজন

## কখন ব্যবহার না করাও যুক্তিযুক্ত

Instructor উল্লেখ করেন, কিছু ক্ষেত্রে error report হওয়া দরকার। যদি property অবশ্যই থাকার কথা, কিন্তু API দিচ্ছে না, optional chaining error লুকিয়ে দিতে পারে। তখন regular access ব্যবহার করে error detect করা যেতে পারে।

## Common mistake

Optional chaining সব error solve করবে মনে করা। এটি শুধু `null` বা `undefined` intermediate access handle করে।

## আরও উদাহরণ

```js
const city = user.address?.city;
const firstItem = order.items?.[0];
const result = service.run?.();
```

# 37. Common Mistakes

## 1. Object property-তে `=` ব্যবহার

ভুল:

```js
const user = { name = "Tapas" };
```

সঠিক:

```js
const user = { name: "Tapas" };
```

## 2. Space-যুক্ত key dot notation দিয়ে access

ভুল:

```js
user.is admin;
```

সঠিক:

```js
user["is admin"];
```

## 3. Dynamic key-তে dot notation

ভুল:

```js
const key = "age";
user.key;
```

সঠিক:

```js
user[key];
```

## 4. Constructor function-এ `new` ভুলে যাওয়া

```js
const car = Car("BMW", "X1");
```

সঠিক:

```js
const car = new Car("BMW", "X1");
```

## 5. Method invoke না করা

```js
profile.message;
```

সঠিক:

```js
profile.message();
```

## 6. Property existence value দিয়ে check

```js
if (!obj.salary) {}
```

এটি `0`, `false`, `""`, `undefined` ইত্যাদি ক্ষেত্রে ভুল result দিতে পারে।

সঠিক:

```js
"salary" in obj
```

অথবা own property:

```js
Object.hasOwn(obj, "salary")
```

## 7. Same-value object-কে equal ভাবা

```js
{ a: 1 } === { a: 1 } // false
```

কারণ reference আলাদা।

## 8. `Object.assign()`-কে deep clone ভাবা

Nested reference shared থাকে।

## 9. `freeze()` এবং `seal()` একই ভাবা

- Freeze: modify/add/delete কিছুই নয়
- Seal: existing value modify করা যায়

## 10. Alias নেওয়ার পর original property name variable ব্যবহার

```js
const { standard: studentStandard } = student;
console.log(standard); // error
```

## 11. Nested destructuring-এর syntax ভুল করা

সঠিক:

```js
const { address: { zip } } = student;
```

## 12. Optional chaining দিয়ে required data error লুকানো

সব জায়গায় `?.` ব্যবহার করলে data contract bug ধরা নাও পড়তে পারে।

# 38. Interview Questions

## Basic

### 1. JavaScript object কী?

Object হলো key-value pair-এর একটি keyed collection, যেখানে unique key ব্যবহার করে value সংরক্ষণ ও retrieve করা হয়।

### 2. Dot notation ও bracket notation-এর পার্থক্য কী?

Dot notation valid static property name-এর জন্য। Bracket notation special-character key ও dynamic key-এর জন্য।

### 3. Object method কী?

Object-এর কোনো property-এর value function হলে সেই function-কে method বলা হয়।

### 4. Nested object কী?

একটি object-এর property value যদি আরেকটি object হয়, তাকে nested object বলা হয়।

### 5. `delete` কী করে?

Object থেকে একটি property remove করে। Property-তে `undefined` assign করা delete নয়।

## Intermediate

### 6. Constructor function ও factory function-এর পার্থক্য কী?

Constructor function সাধারণত capital letter দিয়ে শুরু হয়, `this` ব্যবহার করে এবং `new` দিয়ে invoke করতে হয়। Factory function regular function, object return করে এবং `new` লাগে না।

### 7. Object comparison কেন value দিয়ে হয় না?

Object reference type। Equality operator reference compare করে, nested content নয়।

### 8. `Object.assign()` কী করে?

Source object-এর own enumerable properties target object-এ copy করে। একই key থাকলে source target-এর value override করে।

### 9. Shallow copy কী?

Top-level properties copy হলেও nested object/array-এর reference shared থাকে।

### 10. Deep copy কী?

Nested structure সহ সম্পূর্ণ independent copy, যেখানে nested reference-ও আলাদা।

### 11. `Object.freeze()` ও `Object.seal()`-এর difference কী?

Freeze existing value modify, property add, delete—সব বন্ধ করে। Seal add ও delete বন্ধ করে, কিন্তু existing value modify করা যায়।

### 12. `in` এবং `Object.hasOwn()`-এর পার্থক্য কী?

`in` inherited property-ও detect করতে পারে। `Object.hasOwn()` শুধু object-এর own property detect করে।

## Destructuring

### 13. Object destructuring কী?

Object property value এক বা একাধিক variable-এ সংক্ষিপ্ত syntax-এ extract করার পদ্ধতি।

### 14. Destructuring default value কখন ব্যবহৃত হয়?

Property absent বা value `undefined` হলে।

### 15. Alias syntax কী?

```js
const { propertyName: newVariableName } = object;
```

### 16. Nested destructuring example দিন।

```js
const { address: { city } } = user;
```

### 17. Function parameter destructuring-এর সুবিধা কী?

Function কোন property ব্যবহার করছে তা স্পষ্ট হয় এবং repeated long property chain কমে।

## Optional Chaining

### 18. Optional chaining কী?

`?.` operator intermediate value `null` বা `undefined` হলে error না দিয়ে `undefined` return করে।

### 19. Optional chaining কখন ব্যবহার না করাই ভালো?

যখন property required এবং missing হলে error detect/report করা জরুরি।

# 40. Final Summary

এই lesson-এর মূল শিক্ষা হলো JavaScript-এর object শুধু key-value storage নয়; এটি real-world complex data modelling-এর কেন্দ্রীয় structure।

প্রথমে object literal syntax দিয়ে object তৈরি করা হয়েছে। এরপর dot notation ও bracket notation দিয়ে property access শেখানো হয়েছে। Special character বা dynamic key-এর জন্য bracket notation অপরিহার্য। Object তৈরি হওয়ার পরে property add, update এবং delete করা যায়।

Object creation-এর কয়েকটি pattern শেখানো হয়েছে:

- Object literal
- Constructor function
- Built-in Object constructor
- Factory function

Constructor function blueprint-এর মতো কাজ করে এবং `new` দিয়ে instance তৈরি করে। Factory function regular function হিসেবে নতুন object return করে।

Object property-এর value function হলে সেটি method; object হলে nested object। Property existence check-এর জন্য falsy check-এর চেয়ে `in` operator বা `Object.hasOwn()` বেশি reliable। Object-এর key iterate করতে `for...in`, key array পেতে `Object.keys()` ব্যবহার করা যায়।

Object reference type হওয়ায় দুইটি same-value object equality comparison-এ false হতে পারে। কারণ equality reference compare করে। `Object.assign()` source property target-এ copy করে, কিন্তু nested structure-এর ক্ষেত্রে shallow copy তৈরি করে। Deep copy-এর জন্য `structuredClone()` ব্যবহার করা হয়।

Object conversion-এর জন্য:

- `Object.entries()` → Object থেকে array
- `Object.fromEntries()` → key-value array থেকে object

Immutability নিয়ন্ত্রণে:

- `Object.freeze()` → add/update/delete বন্ধ
- `Object.seal()` → add/delete বন্ধ, update allowed

Object destructuring code ছোট ও clean করে। এর মাধ্যমে:

- Property extract
- Default value
- Dynamic default value
- Alias
- Nested property extract
- Function parameter destructuring
- Function return destructuring
- Loop destructuring

সব করা যায়।

সবশেষে optional chaining `?.` nested property access-এর সময় `null` বা `undefined` থাকলে runtime error এড়াতে সাহায্য করে। তবে required data missing হলে error detect করার প্রয়োজন থাকলে optional chaining অন্ধভাবে ব্যবহার করা উচিত নয়।

## 42.1 Object Destructuring-এর Complete Practice

```javascript
console.log("Learn Object Destructuring....");

const student = {
  name: "John Williamson",
  age: 9,
  std: 3,
  subjects: ["Maths", "English", "EVS"],
  parents: {
    father: "Brown Williamson",
    mother: "Sophia",
    email: "john-parents@abcde.com",
  },
  address: {
    street: "65/2, brooklyn road",
    city: "Carterton",
    country: "New Zealand",
    zip: 5791,
  },
};
```

এই `student` object-এ primitive এবং non-primitive—দুই ধরনের value আছে।

| Property | Value-এর ধরন |
|---|---|
| `name` | string বা primitive |
| `age` | number বা primitive |
| `std` | number বা primitive |
| `subjects` | array বা non-primitive |
| `parents` | nested object বা non-primitive |
| `address` | nested object বা non-primitive |

### Basic destructuring এবং default value

```javascript
const { name, age, meal = "bread" } = student;
console.log(name, age, meal);
```

Output:

```text
John Williamson 9 bread
```

`name` এবং `age` object-এ আছে, তাই সেখানকার value পাওয়া যায়। কিন্তু `meal` property নেই। তাই destructuring-এর default value হিসেবে `"bread"` ব্যবহৃত হয়।

> Default value কেবল property-এর value `undefined` হলে ব্যবহৃত হয়। Property-এর value `null`, `0`, `false` বা empty string হলে default value কার্যকর হয় না।

```javascript
const example = {
  meal: null,
};

const { meal = "bread" } = example;
console.log(meal); // null
```

### অন্য destructured variable ব্যবহার করে computed default value

```javascript
const {
  subjects,
  numberOfSubjects = subjects.length,
} = student;

console.log(numberOfSubjects); // 3
```

এখানে প্রথমে `subjects` variable তৈরি হয়। এরপর `numberOfSubjects` নামে কোনো property না থাকায় default expression `subjects.length` evaluate হয়। `subjects` array-তে তিনটি element আছে, তাই output `3`।

**Common mistake:** property order উল্টে দেওয়া।

```javascript
// ReferenceError হতে পারে, কারণ subjects এখনো initialize হয়নি।
const {
  numberOfSubjects = subjects.length,
  subjects,
} = student;
```

### Alias ব্যবহার

```javascript
const { std: standard } = student;
console.log(standard); // 3
```

এখানে:

- Object property-এর নাম: `std`
- নতুন local variable-এর নাম: `standard`

Alias ব্যবহার করার পর `std` নামে local variable তৈরি হয় না।

```javascript
console.log(standard); // 3
// console.log(std);   // ReferenceError
```

### Nested object destructuring

```javascript
const {
  address: { zip },
} = student;

console.log(zip); // 5791
```

এখানে প্রথমে `address` property-এর object-এ প্রবেশ করা হয়েছে, এরপর সেখান থেকে `zip` বের করা হয়েছে।

**গুরুত্বপূর্ণ:** এই syntax-এ `address` নামে আলাদা variable তৈরি হয় না; শুধু `zip` তৈরি হয়।

```javascript
// console.log(address); // ReferenceError
console.log(zip);        // 5791
```

`address` variable-ও দরকার হলে আলাদাভাবে destructure করতে হবে:

```javascript
const { address } = student;
const { zip: postalCode } = address;
```

### Function parameter-এ nested destructuring

```javascript
function sendEmail({ parents: { email } }) {
  console.log(`Sent an email to ${email}`);
}

```

Output:

```text

```

Function-এর পুরো body-তে বারবার নিচের expression লিখতে হয়নি:

Function parameter গ্রহণ করার সময়ই প্রয়োজনীয় `email` value বের করা হয়েছে।

**Common mistake:** argument-এর structure expected shape-এর মতো না হলে error হবে।

```javascript

// TypeError: Cannot read properties of undefined
```

Safer version:

```javascript

    console.log("Email address পাওয়া যায়নি");
```

### Function return value destructuring

```javascript
const getStudent = () => {
  return {
    name: "John Williamson",
    age: 9,
```


এখানে function-এর returned object আগে কোনো temporary variable-এ রাখা হয়নি। Return হওয়ার সঙ্গে সঙ্গে destructuring করা হয়েছে। Alias ব্যবহার করা হয়েছে, কারণ একই scope-এ `name` ও `subjects` আগে declare করা আছে।

### Loop-এর মধ্যে object destructuring

```javascript
const students = [

    name: "William",

    grade: "A+",
  },
  {

    grade: "B",
```

`students` একটি array এবং এর প্রতিটি element একটি object। `for...of` প্রতিবার একটি object নেয় এবং loop variable অংশেই object-টিকে destructure করে।

## 42.3 Constructor Function

```javascript
function Car(name, model) {
  this.name = name;
  this.model = model;
}

const bmwCar = new Car("BMW", "X1");
const audiCar = new Car("Audi", "A8");

console.log(bmwCar);
console.log(audiCar);
console.log(bmwCar instanceof Car); // true
```

Constructor function-এর convention:

1. নাম capital letter দিয়ে শুরু হয়।
2. Object তৈরির সময় `new` ব্যবহার করা হয়।
3. `this` নতুন object-কে নির্দেশ করে।

Conceptually `new Car("BMW", "X1")` করলে JavaScript:

1. একটি নতুন empty object তৈরি করে।
2. `this`-কে সেই object-এর সঙ্গে bind করে।
3. object-টিকে `Car.prototype`-এর সঙ্গে link করে।
4. constructor body execute করে।
5. object-টি return করে।

**Common mistake:** `new` বাদ দেওয়া। Non-strict mode-এ global object mutate হতে পারে; strict mode-এ `this` হবে `undefined` এবং error হতে পারে।

## 42.7 `for...in`, `Object.keys()` ও Property Existence

### `for...in`

```javascript
for (const key in profile) {
  console.log(key);
  console.log(profile[key]);
}
```

`key` dynamically পাওয়া যায়, তাই value access করতে bracket notation ব্যবহার করা হয়েছে।

> `for...in` enumerable property names iterate করে। Inherited enumerable properties-ও আসতে পারে। শুধু object-এর own property চাইলে `Object.hasOwn()` দিয়ে guard ব্যবহার করা নিরাপদ।

```javascript
for (const key in profile) {
  if (Object.hasOwn(profile, key)) {
    console.log(key, profile[key]);
  }
}
```

### `Object.keys()`

```javascript
console.log(Object.keys(profile));
```

Possible output:

```javascript
["name", "company", "message", "address"]
```

### Missing property access

```javascript
console.log(profile.salary); // undefined
```

### `in` operator

```javascript
console.log("salary" in profile); // false
```

`in` property existence পরীক্ষা করে; property-এর value truthy/falsy কিনা পরীক্ষা করে না।

### ভুল existence check

```javascript
if (!profile.salary) {
  console.log("The salary property doesn't exist");
}
```

এটি reliable নয়। কারণ property থাকলেও value `0`, `false`, `""`, `null` বা `undefined` হলে condition true হবে।

```javascript
profile.salary = 0;

console.log("salary" in profile); // true
console.log(!profile.salary);     // true
```

সঠিক পদ্ধতি:

```javascript
if (!("salary" in profile)) {
  console.log("The salary property doesn't exist");
}
```

শুধু own property পরীক্ষা করতে:

```javascript
if (!Object.hasOwn(profile, "salary")) {
  console.log("The salary property is not an own property");
}
```

## 42.8 Object Reference ও Comparison

```javascript
let fruit = { name: "mango" };          // ধরা যাক XA01
const oneMoreFruit = { name: "mango" }; // ধরা যাক YB02

console.log(fruit == oneMoreFruit);  // false
console.log(fruit === oneMoreFruit); // false
```

দুটির content একই হলেও object দুটো আলাদা memory reference নির্দেশ করে। Object equality-তে property-value-by-property-value comparison হয় না; reference comparison হয়।

```javascript
fruit = oneMoreFruit;

console.log(fruit == oneMoreFruit);  // true
console.log(fruit === oneMoreFruit); // true
```

Assignment-এর পর দুই variable একই object reference নির্দেশ করছে।

### মনে রাখার নিয়ম

```text
Same structure ≠ same object
Same reference = same object
```

## 42.9 `Object.assign()`

```javascript
const target = { p: 1, a: 2 };

const returnedObj = Object.assign(target, source);

```

Output:

`source.a` target-এর existing `a` value overwrite করেছে।

**গুরুত্বপূর্ণ:** `Object.assign()` target object-টিকে mutate করে।

```javascript
console.log(target);
// { p: 1, a: 3, b: 5 }

console.log(returnedObj === target);
// true
```

Original code-এর `retrunedObj` spelling-টি typo; এখানে `returnedObj` করা হয়েছে।

### নতুন target ব্যবহার করে clone

```javascript
const obj = { name: "tapaScript" };
const obj2 = Object.assign({}, obj);

console.log(obj2);
console.log(obj === obj2); // false
```

Top-level object নতুন reference পায়। তবে এটি shallow copy।

## 42.10 Shallow Copy বনাম Deep Copy

```javascript
const obj3 = {
  a: 1,
  b: {
    c: 2,
  },
};

const obj4 = Object.assign({}, obj3);
```

Top-level references:

```javascript
console.log(obj3 === obj4); // false
```

Nested references:

```javascript
console.log(obj3.b === obj4.b); // true
```

তাই:

```javascript
obj4.a = 100;
obj4.b.c = 3;

console.log(obj4.a);   // 100
console.log(obj3.a);   // 1

console.log(obj4.b.c); // 3
console.log(obj3.b.c); // 3
```

Top-level primitive `a` স্বাধীনভাবে copy হয়েছে। কিন্তু nested object `b`-এর reference share হচ্ছে।

### `structuredClone()` দিয়ে deep copy

```javascript
const obj5 = structuredClone(obj3);

obj5.a = 300;
obj5.b.c = 30;

console.log(obj5.a);   // 300
console.log(obj3.a);   // 1

console.log(obj5.b.c); // 30
console.log(obj3.b.c); // 3 বা clone করার সময় obj3.b.c-এর যে value ছিল
```

Fresh object দিয়ে isolated example:

```javascript
const original = {
  a: 1,
  b: { c: 2 },
};

const deepCopy = structuredClone(original);
deepCopy.b.c = 30;

console.log(deepCopy.b.c); // 30
console.log(original.b.c); // 2
```

> `structuredClone()` function, DOM node এবং কিছু unsupported value clone করতে পারে না। তবে সাধারণ object, array, Map, Set, Date এবং circular reference-এর মতো অনেক structure handle করতে পারে।

## 42.11 `Object.entries()` ও `Object.fromEntries()`

### Object থেকে array

```javascript
const myObj = {
  a: "tapas",
  b: 32,
};

const myArr = Object.entries(myObj);
console.log(myArr);
```

Output:

```javascript
[
  ["a", "tapas"],
  ["b", 32],
]
```

প্রতিটি property একটি দুই-element array হয়:

```text
[key, value]
```

### Entries/Map থেকে object

```javascript
const entries = new Map([
  ["foo", "bar"],
  ["baz", 42],
]);

const objEntries = Object.fromEntries(entries);
console.log(objEntries);
```

Output:

```javascript
{
  foo: "bar",
  baz: 42
}
```

Practical transformation:

```javascript
const prices = {
  book: 100,
  pen: 20,
};

const doubledPrices = Object.fromEntries(
  Object.entries(prices).map(([key, value]) => [key, value * 2])
);

console.log(doubledPrices);
// { book: 200, pen: 40 }
```

## 42.12 `Object.freeze()`

```javascript
const emp = {
  sal: 100,
};

Object.freeze(emp);

emp.sal = 200;
emp.name = "Alex";
delete emp.sal;

console.log(emp);
console.log(Object.isFrozen(emp)); // true
```

`freeze()` করার পর top-level object-এ:

- existing property modify করা যায় না
- নতুন property add করা যায় না
- property delete করা যায় না

Non-strict mode-এ invalid operation silently fail করতে পারে। Strict mode-এ `TypeError` হতে পারে।

```javascript
"use strict";

const frozen = Object.freeze({ value: 1 });
// frozen.value = 2; // TypeError
```

### `freeze()` shallow

```javascript
const frozenUser = Object.freeze({
  profile: {
    age: 20,
  },
});

frozenUser.profile.age = 21;
console.log(frozenUser.profile.age); // 21
```

Nested object-কে আলাদাভাবে freeze না করলে সেটি mutable থাকতে পারে।

## 42.13 `Object.seal()`

```javascript
const dept = {
  name: "finance",

Object.seal(dept);

dept.address = "Bangalore";
delete dept.name;
dept.name = "HR";
```


`seal()` করার পর:

| Operation | Allowed? |
|---|---:|
| Existing property update | হ্যাঁ |
| New property add | না |
| Existing property delete | না |

### `freeze()` বনাম `seal()`

| বিষয় | `Object.freeze()` | `Object.seal()` |
|---|---:|---:|
| Existing value update | না | হ্যাঁ |
| New property add | না | না |
| Property delete | না | না |
| Shallow behavior | হ্যাঁ | হ্যাঁ |

## 42.14 `Object.hasOwn()`

```javascript
console.log(Object.hasOwn(dept, "address")); // false
console.log(Object.hasOwn(dept, "name"));    // true
```

`Object.hasOwn(object, key)` শুধু object-এর নিজস্ব বা own property পরীক্ষা করে। Prototype chain থেকে পাওয়া inherited property হলে `false` দেয়।

```javascript
console.log("toString" in dept);              // true
console.log(Object.hasOwn(dept, "toString")); // false
```

এটাই `in` operator এবং `Object.hasOwn()`-এর গুরুত্বপূর্ণ পার্থক্য।

## 42.15 Optional Chaining

```javascript
console.log("Optional Chaining...");

const employee = {
  salary: {
    bonus: 300,
  },
};

console.log(employee.department); // undefined
```

নিচের expression runtime error দেবে:

```javascript
// console.log(employee.department.name);
```

কারণ `employee.department` হলো `undefined`; `undefined`-এর ওপর `.name` পড়া যায় না।

### পুরোনো guard pattern

```javascript
const oldDepartmentName =
  employee.department && employee.department.name;
```

### Optional chaining

```javascript
const departmentName = employee.department?.name;
console.log(departmentName); // undefined
```

`?.`-এর বাম পাশের value `null` বা `undefined` হলে chain থেমে যায় এবং expression `undefined` return করে। Program crash করে না।

### গুরুত্বপূর্ণ variable-name correction

একই script-এর শুরুতে নিচের declaration আছে:

```javascript
const { name, age, meal = "bread" } = student;
```

তাই শেষে আবার এটি লেখা যাবে না:

```javascript
// const name = employee.department?.name;
```

একই scope-এ দুটি `const name` declaration থাকলে parse stage-এই error হবে:

```text
SyntaxError: Identifier 'name' has already been declared
```

তাই corrected variable:

```javascript
const departmentName = employee.department?.name;
```

### Optional chaining-এর আরও ব্যবহার

```javascript
console.log(employee.salary?.bonus);       // 300
console.log(employee.department?.name);    // undefined
console.log(employee.department?.manager?.name); // undefined
```

Method call:

```javascript
const service = {};
service.start?.(); // method না থাকলেও error নয়
```

Dynamic property:

```javascript
const key = "department";
console.log(employee?.[key]); // undefined
```

### Optional chaining assignment-এর বাম পাশে ব্যবহার করা যায় না

```javascript
// employee.department?.name = "Finance";
// SyntaxError: Invalid left-hand side in assignment
```

# 44. Code Review থেকে পাওয়া গুরুত্বপূর্ণ Common Mistakes

1. **একই scope-এ একই `const` variable পুনরায় declare করা**

   ```javascript
   const { name } = student;
   const name = employee.department?.name; // SyntaxError
   ```

2. **Dynamic property access-এ dot notation ব্যবহার করা**

   ```javascript
   user.someKey;  // literal key "someKey"
   user[someKey]; // variable-এর value অনুযায়ী key
   ```

3. **Falsy check-কে property existence check হিসেবে ব্যবহার করা**

   ```javascript
   if (!profile.salary) {
     // salary = 0 হলেও এখানে ঢুকবে
   }
   ```

4. **`Object.assign()`-কে deep clone মনে করা**

   Nested object-এর reference share হয়।

5. **`Object.freeze()` বা `Object.seal()` deep operation মনে করা**

   দুটিই defaultভাবে shallow।

6. **Constructor function-এ `new` বাদ দেওয়া**

7. **Special-character key dot notation দিয়ে access করা**

8. **Nested destructuring-এ parent object variable-ও তৈরি হবে ধরে নেওয়া**

   ```javascript
   const { address: { zip } } = student;
   // এখানে zip আছে; address local variable নেই।
   ```

9. **Optional chaining দিয়ে সব error লুকিয়ে ফেলা**

   Required data অনুপস্থিত থাকলে কখনো error বা explicit validation দরকার হতে পারে। `?.` ব্যবহার করার আগে data optional কি না বুঝতে হবে।

# 45. Additional Interview Questions Based on the Practice Code

### 1. Object destructuring-এর default value কখন ব্যবহৃত হয়?

Property missing হলে বা value `undefined` হলে। `null`, `0`, `false`, এবং `""` হলে নয়।

### 2. `const { std: standard } = student`-এ `std` variable তৈরি হয় কি?

না। `standard` variable তৈরি হয়; `std` হলো source property name।

### 3. `for...in` এবং `for...of`-এর পার্থক্য কী?

- `for...in` property key iterate করে।
- `for...of` iterable-এর value iterate করে।
- Plain object defaultভাবে iterable নয়; array iterable।

### 4. `Object.assign(target, source)` কোন object return করে?

Modified `target` object-টিই return করে।

### 5. `Object.assign({}, source)` deep clone করে কি?

না। এটি shallow copy করে।

### 6. Object content একই হলেও `===` কেন false হতে পারে?

কারণ object equality reference compare করে, content নয়।

### 7. `in` এবং `Object.hasOwn()`-এর পার্থক্য কী?

`in` prototype chain-সহ property খোঁজে; `Object.hasOwn()` শুধু own property পরীক্ষা করে।

### 8. `Object.freeze()` করার পর nested object পরিবর্তন করা যায় কি?

হ্যাঁ, nested object আলাদাভাবে freeze না করলে পরিবর্তন করা যায়।

### 9. Optional chaining কখন short-circuit করে?

`?.`-এর ঠিক বাম পাশের value `null` বা `undefined` হলে।

### 10. Function parameter destructuring-এর সুবিধা কী?

Function কোন data ব্যবহার করছে তা signature-এই স্পষ্ট হয়, repeated property access কমে এবং code concise হয়।
