## 1. Node.js Installation Overview

### ধারণা কী?

Node.js ব্যবহার করতে হলে machine-এ Node.js runtime install থাকতে হবে। Node.js install থাকলে terminal থেকে JavaScript file run করা যায়।

Example:

```bash
node index.js
```

### কেন দরকার?

Node.js install না থাকলে:

* JavaScript file terminal থেকে run করা যাবে না
* npm command পাওয়া যাবে না
* Node.js backend project চালানো যাবে না
* package install করা যাবে না

### কীভাবে কাজ করে?

Node.js install করলে সাধারণত machine-এ দুইটি command পাওয়া যায়:

```bash
node
npm
```

| Command | কাজ                        |
| ------- | -------------------------- |
| `node`  | JavaScript file run করে    |
| `npm`   | package install/manage করে |

Transcript-এ বলা হয়েছে, Node.js install করার আগে machine-এ already install আছে কিনা check করতে হবে, কারণ অনেকের machine-এ npm বা Node package manager ব্যবহারের কারণে Node.js আগেই থাকতে পারে।

### মনে রাখার মতো পয়েন্ট

* Node.js install করলে `node` command পাওয়া যায়।
* npm সাধারণত Node.js-এর সাথে install হয়।
* install করার আগে version check করা ভালো।

## 2. Machine-এ Node.js আছে কিনা Check করা

### ধারণা কী?

Node.js already install আছে কিনা check করতে terminal-এ version command চালানো হয়।

```bash
node -v
```

অথবা:

```bash
node --version
```

### Output example

```bash
v18.19.0
```

যদি version number দেখায়, তাহলে Node.js install আছে।

যদি error দেখায়, যেমন:

```txt
node: command not found
```

অথবা Windows-এ:

```txt
'node' is not recognized as an internal or external command
```

তাহলে Node.js install নেই অথবা PATH ঠিকভাবে set হয়নি।

Transcript-এ instructor বলেছেন, terminal-এ `node -v` লিখলে version number পেলে বোঝা যাবে Node.js already install করা আছে।

### কেন দরকার?

Install করার আগে version check করলে বোঝা যায়:

* Node.js install আছে কিনা
* কোন version চলছে
* update করার প্রয়োজন আছে কিনা

### সতর্কতা

`node -v` command কাজ না করলে সবসময় Node.js নেই এমন নয়। কখনো PATH configuration সমস্যা হতে পারে। তবে beginner হিসেবে সহজ সমাধান হলো official installer দিয়ে আবার install করা।

## 3. Node.js Download এবং Install করা

### ধারণা কী?

Node.js official website থেকে operating system অনুযায়ী installer download করে install করা যায়।

Website সাধারণত দুই ধরনের version দেখায়:

| Version Type | অর্থ                                      |
| ------------ | ----------------------------------------- |
| LTS          | Long Term Support, stable এবং recommended |
| Current      | latest features, কিন্তু change হতে পারে   |

Transcript-এ বলা হয়েছে, official website-এ গেলে দুইটি option থাকে: stable/LTS version এবং current version। Node.js সাধারণত stable version recommend করে।

### কোনটা install করা উচিত?

Beginner বা production-oriented কাজের জন্য সাধারণত **LTS version** install করা ভালো।

### Install process

Windows:

1. Node.js website থেকে Windows installer download করুন।
2. `.msi` file run করুন।
3. default settings রেখে install করুন।
4. terminal খুলে check করুন:

```bash
node -v
npm -v
```

macOS:

1. macOS installer download করুন।
2. `.pkg` file run করুন।
3. install complete হলে terminal খুলুন।
4. check করুন:

```bash
node -v
npm -v
```

Linux:

Linux-এ distro অনুযায়ী package manager, NodeSource বা version manager ব্যবহার করা যায়।

### মনে রাখার মতো পয়েন্ট

* Beginnerদের জন্য official installer easiest।
* Already install থাকলে update করা যায়।
* Node.js install হলে npm-ও সাধারণত install হয়।

Transcript-এ instructor বলেছেন, already তৈরি করা জিনিস আবার scratch থেকে বানানোর মানে নেই; এতে মূল application বানানোর focus পিছিয়ে যায়।

## 6. `node_modules`, `package.json`, এবং `package-lock.json`

### `node_modules` কী?

`node_modules` হলো সেই folder যেখানে npm installed packages রাখে।

Example:

```txt
node_modules/
└── lodash/
```

### `package.json` কী?

`package.json` project-এর metadata এবং dependencies track করে।

Example:

```json
{
  "dependencies": {
    "lodash": "^4.17.20"
  }
}
```

Transcript-এ বলা হয়েছে, project-এ যেসব external library/package ব্যবহার করা হবে, সেগুলো `package.json`-এর `dependencies` section-এ save থাকে।

### `^` sign কী বোঝায়?

Example:

```json
"lodash": "^4.17.20"
```

`^` সাধারণত compatible newer minor/patch version install করার সুযোগ দেয়। Transcript-এ বলা হয়েছে, version number-এর আগে থাকা sign-এর কারণে পরবর্তী সময়ে newer version install হতে পারে।

### `package-lock.json` কী?

`package-lock.json` exact installed version track করে।

কেন দরকার?

ধরি আজ install হলো:

```txt
lodash 4.17.20
```

ছয় মাস পর কেউ project নিয়ে `npm install` দিলে যদি lodash-এর newer version আসে, তাহলে behavior change হয়ে app break হতে পারে। তাই exact version lock রাখতে `package-lock.json` দরকার।

Transcript-এ বলা হয়েছে, exact কোন package কোন version-এ install হয়েছে সেটা track করার জন্য lock file তৈরি হয়।

### `node_modules` GitHub-এ push করা হয়?

সাধারণত না।

কারণ:

* অনেক বড় folder
* হাজার হাজার file থাকতে পারে
* package.json থাকলেই dependencies reinstall করা যায়

Transcript-এ বলা হয়েছে, project distribute বা GitHub-এ push করার সময় `node_modules` folder দেওয়া হয় না; শুধু `package.json` থাকলে অন্য developer `npm install` দিয়ে dependencies install করতে পারে।

### অন্য developer project চালাবে কীভাবে?

```bash
npm install
```

এই command `package.json` এবং lock file দেখে required packages install করবে।

এখানে `lodash` package import করা হয়েছে। Lodash-এর জন্য conventionally `_` variable name ব্যবহার করা হয়।

```js

```

একটি array নেওয়া হয়েছে।

```js

```

`_.last()` lodash-এর function। এটি array-এর শেষ element return করে।

### কেন গুরুত্বপূর্ণ?

এই example দেখায়:

* external package install করা যায়

## 8. npm-এর Security এবং Maintenance Concern

### ধারণা কী?

npm public registry। এখানে developerরা package publish করতে পারে। এতে সুবিধা যেমন আছে, ঝুঁকিও আছে।

Transcript-এ বলা হয়েছে, npm huge public registry হওয়ায় unknown package install করলে malicious code, bug বা security vulnerability থাকার risk থাকে।

### Security risk কোথায়?

Package install করার সময় package-এর code আপনার machine/project-এ আসে। যদি package malicious হয়, তাহলে:

* sensitive file read করতে পারে
* harmful script চালাতে পারে
* dependency chain দিয়ে vulnerability আনতে পারে
* production app insecure করতে পারে

### Package install করার আগে check করুন

| বিষয়          | কী দেখবেন                        |
| ------------- | -------------------------------- |
| Popularity    | downloads, stars                 |
| Maintenance   | recent update আছে কি না          |
| Issues        | অনেক unresolved issue আছে কি না  |

<a id="chapter-17-eslint-rules-customize-করা"></a>

## 17. ESLint Rules Customize করা

### ধারণা কী?

ESLint-এর প্রতিটি rule customize করা যায়।

Transcript-এ `no-var` rule-এর example দেখানো হয়েছে। `var` ব্যবহার করলে ESLint warning দেয়: `Unexpected var, use let or const instead`। এরপর চাইলে `.eslintrc` file-এ rule off করা যায়।

### Example: `var` warning

```js
var name = "Rahim";

console.log(name);
```

ESLint বলবে:

```txt
Unexpected var, use let or const instead.
```

### Fix

```js
const name = "Rahim";

console.log(name);
```

যদি variable reassign করতে হয়:

```js
let count = 1;

count = 2;

console.log(count);
```

### Rule off করা

`.eslintrc.json`:

```json
{
  "rules": {
    "no-var": "off"
  }
}
```

অথবা:

```json
{
  "rules": {
    "no-var": 0
  }
}
```

### সতর্কতা

Rule off করা যায়, কিন্তু সব rule off করলে linting-এর value কমে যায়। Beginnerদের default recommended rules follow করা ভালো।

<a id="common-mistakes"></a>

# Common Mistakes

## Mistake 1: Node.js install আছে কিনা check না করা

সবসময় আগে check করুন:

```bash
node -v
npm -v
```

---

## Mistake 2: `node_modules` GitHub-এ push করা

ভুল:

```txt
node_modules GitHub-এ upload করা
```

সঠিক:

```gitignore
node_modules
```

`package.json` থাকলে অন্য developer `npm install` দিয়ে dependencies install করতে পারবে।

---

## Mistake 3: `package-lock.json` বা `yarn.lock` ignore করা

Lock file exact dependency version track করে। Team project-এ lock file commit করা উচিত।

---

## Mistake 4: npm এবং Yarn mix করা

এক project-এ একসাথে দুই package manager ব্যবহার করলে conflict হতে পারে।

Avoid:

```txt
package-lock.json + yarn.lock দুটোই রাখা
```

Team decide করুক npm নাকি Yarn।

---

## Mistake 5: Unknown package install করা

ভুল:

```bash
npm install random-unknown-package
```

সঠিকভাবে আগে check করুন:

* package popular কি না
* documentation আছে কি না
* last update কবে
* security issue আছে কি না

---

## Mistake 6: ESLint/Prettier globally install করে project config না রাখা

ভুল:

```bash
npm install -g eslint prettier
```

Better:

```bash
npm install -D eslint prettier
```

---

## Mistake 7: Formatter ও linter conflict ignore করা

ESLint এবং Prettier একসাথে use করলে proper config দরকার। না হলে এক tool code format করবে, অন্য tool error দেখাবে।

---

## Mistake 8: `var` unnecessary ব্যবহার করা

ভুল:

```js
var count = 10;
```

ভালো:

```js
const count = 10;
```

Reassign দরকার হলে:

```js
let count = 10;
count = 20;
```

# Interview / Exam Style Questions

## 1. Node.js install আছে কিনা কীভাবে check করবেন?

Terminal-এ লিখব:

```bash
node -v
```

Version number দেখালে Node.js install আছে।

---

## 2. npm কী?

npm হলো Node Package Manager। এটি Node.js/JavaScript package install, update, remove এবং dependency manage করতে ব্যবহৃত হয়।

---

## 3. `npm install lodash` দিলে কী হয়?

এই command project-এ lodash package install করে। Package `node_modules` folder-এ যায় এবং `package.json`-এর `dependencies` section-এ lodash entry যোগ হয়।

---

## 4. `node_modules` folder কী?

এটি installed npm packages রাখার folder। সাধারণত এটি GitHub-এ push করা হয় না, কারণ এটি বড় এবং `package.json` থেকে regenerate করা যায়।

---

## 5. `package.json` কেন দরকার?

`package.json` project-এর metadata, scripts, dependencies এবং devDependencies track করে। অন্য developer project clone করার পর `npm install` দিয়ে dependencies install করতে পারে।

---

## 6. `package-lock.json` কেন দরকার?

এটি exact installed dependency version lock করে, যাতে future install-এ version mismatch বা unexpected breaking change না হয়।

---

## 7. npm এবং Yarn-এর পার্থক্য কী?

দুইটিই package manager। npm Node.js-এর default package manager। Yarn npm-এর alternative, যা package install এবং dependency management করে। Yarn `yarn.lock` file তৈরি করে, npm `package-lock.json` তৈরি করে।

---

## 8. `dependencies` এবং `devDependencies`-এর পার্থক্য কী?

`dependencies` app runtime-এ দরকার। `devDependencies` শুধু development/build/test/formatting/linting-এর সময় দরকার।

---

## 9. ESLint কী?

ESLint হলো JavaScript linter, যা code-এর potential error, bad practice, style violation এবং rule violation detect করে।

---

## 10. Prettier কী?

Prettier হলো code formatter, যা indentation, spacing, line break, quote style ইত্যাদি automatically format করে।

---

## 11. ESLint এবং Prettier একসাথে কেন ব্যবহার করা হয়?

ESLint code quality এবং rules enforce করে। Prettier code formatting করে। একসাথে ব্যবহার করলে code readable, consistent এবং less error-prone হয়।

---

## 12. Airbnb Style Guide কী?

Airbnb Style Guide হলো popular JavaScript coding convention, যা consistent এবং professional JavaScript লেখার rule set দেয়।

---

## 13. ESLint/Prettier globally install না করে project dependency হিসেবে install করা ভালো কেন?

কারণ project dependency হিসেবে রাখলে team-এর সবাই একই version ব্যবহার করে, project-specific config কাজ করে এবং environment reproducible থাকে।

---

## 14. `npm install -D eslint`-এ `-D` মানে কী?

`-D` মানে package dev dependency হিসেবে install হবে। অর্থাৎ এটি `devDependencies` section-এ যাবে।

---

## 15. `.eslintrc.json` file-এর কাজ কী?

এই file ESLint configuration রাখে। এখানে style guide, environment, plugin এবং custom rules define করা হয়।
