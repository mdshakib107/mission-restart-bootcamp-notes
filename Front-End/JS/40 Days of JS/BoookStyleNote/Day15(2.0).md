# 1. `array` কী?

JavaScript-এ একজোড়া square bracket—`[]`—একটি array প্রকাশ করে। Square bracket-এর ভেতরে comma দিয়ে আলাদা করে element রাখা হয়।

```js
[1, 2, 3]
```

JavaScript array-এর একটি বিশেষ বৈশিষ্ট্য হলো—একই array-তে বিভিন্ন type-এর element রাখা যায়। যেমন `number`, `boolean`, `string`, `object`, এমনকি আরেকটি `array`-ও।

```js
const mixedArray = [100, true, "tapaScript", {}];
```

এখানে:

- `100` একটি `number`
- `true` একটি `boolean`
- `"tapaScript"` একটি `string`
- `{}` একটি `object`

## 1.1 `index` কী?

Array-এর প্রতিটি element একটি নির্দিষ্ট position-এ থাকে। সেই position-কে `index` বলা হয়।

**Definition:** একটি array-তে কোনো element-এর position-কে তার `index` বলা হয়।

সবচেয়ে গুরুত্বপূর্ণ rule:

```text
index শুরু হয় 0 থেকে
শেষ index = array.length - 1
```

উপরের `mixedArray`-তে:

- `100` → index `0`
- `true` → index `1`
- `"tapaScript"` → index `2`
- `{}` → index `3`

এখানে মোট element 4টি, তাই শেষ index হলো `4 - 1 = 3`।

JavaScript array fixed-length নয়। প্রয়োজন অনুসারে এর length বাড়ানো বা কমানো যায়। `length` নিয়ে পরে আলাদা করে গভীরভাবে আলোচনা করব।

# 2. Array তৈরি করার উপায়

## 2.1 Array literal

সবচেয়ে সরাসরি উপায় হলো array literal। চলো একটি salad array তৈরি করি। Salad-এ বিভিন্ন ফল ও সবজি থাকে, তাই এটি array দিয়ে প্রকাশ করার সুন্দর উদাহরণ।

```js
const salad = ["🍅", "🍄", "🥦", "🥒", "🌽", "🥕", "🥑"];
```

Emoji-গুলোর বদলে `"tomato"`, `"mushroom"` ইত্যাদি string-ও লেখা যেত। এখানে visual আকর্ষণের জন্য emoji ব্যবহার করা হয়েছে।

এই array-তে 7টি element আছে। `"🍅"` index `0`-এ এবং `"🥑"` index `6`-এ।

## 2.2 Constructor function-এর সংক্ষিপ্ত background

Array constructor বোঝার আগে constructor function স্মরণ করি। Constructor function দেখতে সাধারণ function-এর মতো, তবে convention অনুযায়ী নাম capital letter দিয়ে শুরু হয়। এটি `this` ব্যবহার করে runtime-এ object property তৈরি ও value assign করতে পারে।

```js
function Car(model) {
    this.model = model;
}

const bmwCar = new Car("BMW X1");
console.log(bmwCar);
```

Expected output:

```js
Car { model: "BMW X1" }
```

`Car` একটি blueprint-এর মতো কাজ করছে। `new` দিয়ে এর instance তৈরি করা হয়েছে।

## 2.3 `Array` constructor

একইভাবে `Array` constructor ব্যবহার করেও array তৈরি করা যায়।

```js
const anotherSalad = new Array(
    "🍅", "🍄", "🥦", "🥒", "🌽", "🥕", "🥑"
);

console.log("Salad", salad);
console.log("Another Salad", anotherSalad);
```

দুটি array দেখতে একই। কিন্তু strict equality করলে কী হবে?

```js
console.log(salad === anotherSalad);
```

Expected output:

```js
false
```

কারণ array হলো reference type। Value দেখতে একই হলেও দুটি array আলাদা memory location-এ তৈরি হয়েছে এবং তাদের reference আলাদা।

## Common Mistake: Constructor-এ একটি মাত্র number

এখন একটি tricky case:

```js
const two = new Array(2);
console.log(two);
```

অনেকে ভাবতে পারে output হবে:

```js
[2]
```

কিন্তু বাস্তবে output হবে length `2`-এর একটি sparse array:

```js
[empty × 2]
```

কারণ `new Array(2)`-এ একটি মাত্র numeric argument দিলে সেটি element হিসেবে নয়, array-এর length হিসেবে ধরা হয়।

অন্যদিকে একাধিক argument দিলে সেগুলো element হয়:

```js
const two = new Array(1, 2);
console.log(two); // [1, 2]
```

এই পার্থক্য interview-এও জিজ্ঞেস করা হতে পারে।

পরে আমরা `Array.of()`, `Array.from()` এবং spread operator দিয়েও array তৈরি করব।

# 3. Array থেকে element access করা

Array element access করার syntax:

```js
array[index]
```

Salad array থেকে কয়েকটি element বের করি:

```js
console.log(salad[0]); // '🍅'
console.log(salad[2]); // '🥦'
console.log(salad[5]); // '🥕'
```

Index hardcode করে element নেওয়া যায়, কিন্তু বড় array-তে প্রতিটি index মনে রাখা বাস্তবসম্মত নয়। তখন loop ব্যবহার করা হয়।

```js
for (let i = 0; i <= salad.length - 1; i++) {
    console.log(`Element at index ${i} is ${salad[i]}`);
}
```

Reasoning:

1. `i = 0`, কারণ array index `0` থেকে শুরু।
2. শেষ index হলো `salad.length - 1`।
3. প্রতিবার `i++` হওয়ায় পরের index-এ যাওয়া হয়।
4. `salad[i]` দিয়ে current element পাওয়া যায়।

Expected output ধারাবাহিকভাবে index `0` থেকে `6` পর্যন্ত tomato থেকে avocado print করবে।

# 4. Element যোগ করা: `push()` ও `unshift()`

## 4.1 `push()` — শেষে যোগ করা

`push()` array-এর শেষে element যোগ করে।

```js
const ret = salad.push("🥜");
console.log(ret); // 8
console.log(salad);
```

এখানে salad-এ আগে 7টি element ছিল। Peanut যোগ হওয়ার পর 8টি হয়েছে। তাই `push()` return করেছে `8`।

**খুব গুরুত্বপূর্ণ:** `push()` নতুন array return করে না। এটি updated length return করে এবং source array-কে mutate করে।

## 4.2 `unshift()` — শুরুতে যোগ করা

`unshift()` array-এর শুরুতে element যোগ করে।

```js
const unRet = salad.unshift("🥜");
console.log(unRet); // 9
console.log(salad);
```

এখন front-এ আরেকটি peanut যোগ হয়েছে। মোট element 9টি, তাই return value `9`। এটিও source array mutate করে।

# 5. Element সরানো: `pop()` ও `shift()`

## 5.1 `pop()` — শেষ থেকে সরানো

`pop()` array-এর শেষ element সরায়, removed element return করে এবং original array mutate করে।

```js
console.log(salad);
const popRet = salad.pop();
console.log(popRet); // '🥜'
console.log(salad);
```

শেষে থাকা peanut সরানো হয়েছে, তাই `popRet`-এ peanut এসেছে।

## 5.2 `shift()` — শুরু থেকে সরানো

`shift()` array-এর প্রথম element সরায়।

```js
console.log(salad);
const shiftRet = salad.shift();
console.log(shiftRet); // '🥜'
console.log(salad);
```

Front-এর peanut সরানো হয়েছে। এটিও removed element return করে এবং source array mutate করে।

## Recap: চারটি method

| Method | কাজ | Return | Source mutate করে? |
|---|---|---|---|
| `push()` | শেষে যোগ | নতুন length | হ্যাঁ |
| `unshift()` | শুরুতে যোগ | নতুন length | হ্যাঁ |
| `pop()` | শেষ থেকে সরায় | removed element | হ্যাঁ |
| `shift()` | শুরু থেকে সরায় | removed element | হ্যাঁ |

এই চারটি method source array বদলে দেয়—rule-টি লিখে রাখো।

# 6. Clone বা Copy: `slice()`

```js
const saladCopy = salad.slice();

console.log("Salad Before Copy", salad);
console.log("Salad After Copy", saladCopy);
console.log(salad === saladCopy); // false
```

`slice()` কোনো argument ছাড়া call করলে array-এর shallow copy তৈরি হয়। Source array mutate হয় না। Copy দেখতে একই হলেও reference আলাদা, তাই strict equality `false`।

এখান থেকেই immutability ধারণাটি সামনে আসে: source data সরাসরি পরিবর্তন না করে copy তৈরি করে কাজ করা। Spread operator দিয়েও পরে copy তৈরি করব।

# 7. কোনো value array কি না: `Array.isArray()`

```js
Array.isArray(["🍅", "🍄"]);   // true
Array.isArray("🍅");           // false
Array.isArray({ tomato: "🍅" }); // false
Array.isArray([]);              // true

const arr = [1, 2, 3, 4];
Array.isArray(arr);             // true
```

Empty array-তেও element না থাকলেও সেটি array, তাই result `true`।

# 8. Array Destructuring

`destructuring` হলো ES6-এ আসা একটি syntax, যার মাধ্যমে array থেকে এক বা একাধিক value extract করে এক ধাপে variable-এ assign করা যায়। এটি code ছোট করে এবং productivity বাড়ায়। তবে অতিরিক্ত জটিল destructuring readability কমাতে পারে; কোথায় ব্যবহার করা সুবিধাজনক, তা developer-কে বিচার করতে হবে।

Destructuring ছাড়া:

```js
const tomato = salad[0];
const mushroom = salad[1];
const carrot = salad[5];
```

Destructuring দিয়ে:

```js
const [tomato, mushroom, carrot] = ["🍅", "🍄", "🥕"];
console.log(tomato, mushroom, carrot);
```

Expected output:

```text
🍅 🍄 🥕
```

বাম পাশে square bracket-এর ভেতরে variable থাকে, ডান পাশে array। Position অনুযায়ী assignment হয়। `const`, `let` বা `var` declaration ভুলে গেলে চলবে না।

# 9. Destructuring-এ Default Value

Array-তে value না থাকলে destructured variable-এর value সাধারণত `undefined` হয়। তবে destructuring-এর সময় default value দেওয়া যায়।

```js
const [tomato, mushroom = "🍄"] = ["🍅"];

console.log(tomato);   // '🍅'
console.log(mushroom); // '🍄'
```

ডান পাশের array-তে mushroom নেই। তাই default value ব্যবহৃত হয়েছে।

---

# 10. Destructuring-এ Value Skip করা

ধরা যাক tomato ও carrot চাই, কিন্তু মাঝের mushroom চাই না।

```js
const [tomato, , carrot] = ["🍅", "🍄", "🥕"];

console.log(tomato); // '🍅'
console.log(carrot); // '🥕'
```

মাঝখানে variable না লিখে শুধু comma রেখে position skip করা হয়েছে।

**উত্তর:** প্রথম ও তৃতীয় variable-এর মাঝখানে একটি খালি slot রেখে দুইটি comma ব্যবহার করতে হবে।

# 11. Nested Array

একটি array-এর element হিসেবে আরেকটি array থাকলে ভেতরের array-কে nested array বলা হয়। Nesting যেকোনো depth পর্যন্ত যেতে পারে।

```js
[1, 2, [4, [6, 8, ["q"]]]]
```

খেয়াল করো, outer array-এর তৃতীয় element `4` নয়; তৃতীয় element হলো সম্পূর্ণ `[4, [6, 8, ["q"]]]` array।

## 11.1 Nested element access

```js
let fruits = ["🍈", "🍍", "🍌", "🍉", ["🍅", "🍄", "🥕"]];

const veg = fruits[4];
const carrot = veg[2];

fruits[4][2]; // '🥕'
```

প্রথমে outer array-এর index `4`, তারপর nested array-এর index `2`। এটি matrix-এর row-column access-এর মতো ভাবা যায়।

## 11.2 Nested destructuring

```js
let [, , , , [, , carrot]] = [
    "🍈", "🍍", "🍌", "🍉", ["🍅", "🍄", "🥕"]
];
```

এটি কাজ করে, কিন্তু syntax জটিল। Instructor-এর পরামর্শ—শুধু একটি গভীর nested value access করতে সরাসরি `fruits[4][2]` অনেক সময় বেশি readable। তবু destructuring syntax-টি জানা প্রয়োজন।

# 12. `rest parameter` ও `spread operator`

দুটিতেই তিনটি dot—`...`—ব্যবহার হয়। পার্থক্য কোথায় ব্যবহৃত হচ্ছে:

- Destructuring assignment-এর বাম পাশে থাকলে → `rest parameter`
- Value বা array-এর সামনে, ডান পাশে থাকলে → `spread operator`

## 12.1 Rest parameter

```js
const [tomato, mushroom, ...rest] = [
    "🍅", "🍄", "🥦", "🥒", "🌽", "🥕", "🥑"
];

console.log(rest);
```

Expected output:

```js
["🥦", "🥒", "🌽", "🥕", "🥑"]
```

প্রথম দুই element পৃথক variable-এ গেছে, বাকি সব element নতুন array হিসেবে `rest`-এ সংগ্রহ হয়েছে। Variable-এর নাম `rest` হওয়া বাধ্যতামূলক নয়।

## 12.2 Spread operator দিয়ে clone

```js
const mySalad = ["🍅", "🍄", "🥦", "🥒", "🌽", "🥕", "🥑"];
const mySaladCopy = [...mySalad];

console.log(mySaladCopy);
console.log(mySalad === mySaladCopy); // false
```

Spread operator array-এর element-গুলোকে “ছড়িয়ে” নতুন square bracket-এর মধ্যে বসায়। ফলে নতুন array instance তৈরি হয়।

এই জায়গায় আবার মনে রাখো: দ্রুত এগিও না। Logical break নাও, task করো, তারপর সামনে যাও। Quality learning-ই ভালো developer তৈরি করে।

# 13. Destructuring Use Case: Value Swap

দুটি variable-এর value swap করতে traditional পদ্ধতিতে temporary variable লাগে। Destructuring দিয়ে এক লাইনে করা যায়।

```js
let first = "😔";
let second = "🙂";

[first, second] = [second, first];

console.log(first);  // '🙂'
console.log(second); // '😔'
```

এখানে variable দুটিকে `let` দিয়ে তৈরি করা হয়েছে, কারণ পুনরায় assign করতে হবে। `const` হলে reassignment সম্ভব নয়।

```js
const emotions = ["🙂", "😔"];
```

# 15. Array `length` Property

`length` একটি property, method নয়। তাই parenthesis ব্যবহার করা হয় না।

```js
const arr1 = [11, 21, 73];
const arr2 = new Array(7);

console.log(arr1.length); // 3
console.log(arr2.length); // 7
```

JavaScript array-এর সর্বোচ্চ length হলো:

```js
2 ** 32 - 1 // 4294967295
```

## 15.1 Length কমালে element কেটে যায়

```js
const arr1 = [11, 21, 73];
arr1.length = 2;
console.log(arr1); // [11, 21]
```

Length `3` থেকে `2` করায় শেষ element `73` বাদ গেছে।

## 15.2 Invalid length

```js
arr1.length = 2 ** 32;
```

এতে `RangeError: Invalid array length` হবে। Negative length দিলেও একই ধরনের error হবে।

## 15.3 Array empty করার দ্রুত উপায়

```js
arr1.length = 0;
console.log(arr1); // []
```

## 15.4 Length বাড়ালে empty slot তৈরি হয়

```js
const arr1 = [11, 21, 73];
arr1.length = 9;
console.log(arr1);
```

Expected output ধারণাগতভাবে:

```js
[11, 21, 73, empty × 6]
```

প্রথম তিন element থাকে, বাকি ছয়টি empty slot হয়। Length কিন্তু `9`।

## বিরতি

এ পর্যন্ত break না নিলে এখন একটি break নেওয়া ভালো। এরপর আমরা array method-এর বড় অংশে প্রবেশ করছি—create, remove, add, update এবং access-সংক্রান্ত method।

# 16. `concat()` Method

`concat()` এক বা একাধিক array merge করে নতুন array return করে। Source array mutate হয় না।

```js
const first = [1, 2, 3];
const second = [4, 5, 6];
const third = [7, 8, 9];

const merged = first.concat(second, third);
console.log(merged);

console.log(first);  // [1, 2, 3]
console.log(second); // [4, 5, 6]
```

Expected merged output:

```js
[1, 2, 3, 4, 5, 6, 7, 8, 9]
```

দুইটির বেশি array-ও concat করা যায়।

# 17. `join()` Method

`join()` array-এর সব element একটি separator দিয়ে যুক্ত করে string return করে। Default separator হলো comma।

```js
const emotions = ["🙂", "😍", "🙄", "😟"];

const joined = emotions.join("<=>");
console.log(joined);
```

Expected output:

```text
🙂<=>😍<=>🙄<=>😟
```

Argument না দিলে comma separator ব্যবহৃত হয়। Empty array-এর ক্ষেত্রে:

```js
[].join(); // ""
```

কারণ join করার মতো element নেই। এটি interview-এর ছোট tricky question হতে পারে।

# 18. `fill()` Method

`fill()` array-এর সব বা নির্দিষ্ট range-এর element একটি static value দিয়ে replace করে। এটি source array mutate করে।

```js
const colors = ["red", "blue", "green"];
colors.fill("pink");
console.log(colors); // ["pink", "pink", "pink"]
```

Instructor একটি real-world application-এর কথা বলেন, যেখানে অনেক card-এর color ও shape একসঙ্গে user selection অনুযায়ী পরিবর্তন করা হয়েছিল। সেই ক্ষেত্রে প্রতিটি card-এর state-এ একই value বসাতে `fill()` কার্যকর ছিল।

## 18.1 Selective fill

```js
const colors = ["red", "blue", "green"];
colors.fill("pink", 1, 3);
console.log(colors); // ["red", "pink", "pink"]
```

Syntax:

```js
array.fill(value, start, end)
```

- `start` zero-based index
- `end` exclusive boundary; end index-এর element replace হয় না

Transcript-এ এটি “কত length পর্যন্ত যাবে” হিসেবে ব্যাখ্যা করা হয়েছে। ব্যবহারিকভাবে `1, 3` মানে index `1` থেকে শুরু করে index `3`-এর আগে পর্যন্ত—অর্থাৎ index `1` ও `2`।

# 19. `includes()` Method

```js
const names = ["tom", "alex", "bob", "john"];
```

```js
names.includes("Tom"); // false
```

# 20. `indexOf()` ও `lastIndexOf()`

```js
const names = ["tom", "alex", "bob", "tom"];
```

# 22. `sort()` Method

দুটি rule মনে রাখো:

1. Default `sort()` element-গুলোকে string-এ convert করে compare করে।
2. Default order ascending।

```js
const names = ["tom", "alex", "bob"];
console.log("After default sorting", names.sort());
```

Expected output:

```js
["alex", "bob", "tom"]
```

`sort()` original array mutate করে।

## 22.1 Artist name sort

```js
const artists = [
    "John White Abbott",
    "Leonardo da Vinci",
    "Charles Aubry",
    "Anna Atkins",
    "Barent Avercamp",
];

console.log("Default sorting of artists array", artists.sort());
```

Alphabetical ascending order পাওয়া যাবে।

## 22.2 Comparator function

Descending order বা non-string data সঠিকভাবে sort করতে comparator function লাগে। Comparator দুটি element নেয় এবং সাধারণত negative, zero বা positive value return করে।

```js
artists.sort(function (a, b) {
    return a === b ? 0 : a > b ? -1 : 1;
});

console.log("Sort the artist names(Descending)", artists);
```

- `a === b` হলে position বদলানোর দরকার নেই → `0`
- Descending order-এ `a > b` হলে `a` আগে থাকবে → negative value
- অন্যথায় positive value

## 22.3 Number sort-এর common trap

```js
let ages = [2, 1000, 10, 3, 23, 12, 30, 21];
console.log(ages.sort());
```

অনেকে numeric ascending আশা করে, কিন্তু default sort string comparison করে। তাই `10`, `1000`, `12`, `2` ইত্যাদি lexicographic order-এ আসতে পারে।

Numeric ascending comparator:

```js
ages.sort(function (a, b) {
    return a === b ? 0 : a > b ? 1 : -1;
});
```

আরও সংক্ষিপ্তভাবে সাধারণত লেখা হয়:

```js
ages.sort((a, b) => a - b);
```

Numeric descending:

```js
ages.sort((a, b) => b - a);
```

# 23. `splice()` Method

`splice()` প্রথমে confusing লাগতে পারে, কারণ একই method delete, add এবং replace—তিনটিই করতে পারে। Syntax:

```js
array.splice(start, deleteCount, item1, item2, ...)
```

- `start`: কোন zero-based index থেকে change শুরু হবে
- `deleteCount`: কতটি element delete হবে
- অতিরিক্ত `item`: start position থেকে কোন element যোগ হবে

`deleteCount` `0` বা negative হলে কোনো element delete হয় না।

**Return value:** deleted element-গুলোর একটি array। কিছু delete না হলে empty array।

**Mutation:** `splice()` original array mutate করে।

## 23.1 Delete

```js
const names = ["tom", "alex", "bob"];
console.log(names.splice(0, 1));
```

Expected return:

```js
["tom"]
```

Original `names` হয়ে যায়:

```js
["alex", "bob"]
```

## 23.2 Replace

```js
const names = ["tom", "alex", "bob"];
const removed = names.splice(0, 1, "john");

console.log(removed); // ["tom"]
console.log(names);   // ["john", "alex", "bob"]
```

Tom delete হয়েছে, একই start index-এ John যোগ হয়েছে। তাই এটি replacement-এর মতো কাজ করেছে।

## 23.3 Delete না করে insert

```js
const names = ["tom", "alex", "bob"];
names.splice(1, 0, "zack");
console.log(names);
```

Expected output:

```js
["tom", "zack", "alex", "bob"]
```

কিছু delete হয়নি, তাই `splice()`-এর return value হবে `[]`।

## 23.4 আরেকটি replacement

```js
const names = ["tom", "alex", "bob"];
names.splice(2, 1, "zack");
console.log(names); // ["tom", "alex", "zack"]
```

## Slice বনাম Splice

- `slice()` → copy করে; source mutate করে না
- `splice()` → remove, replace বা add করে; source mutate করে

এই mnemonic মনে রাখো।

# 24. `at()` Method

Square bracket access positive index-এর জন্য ভালো, কিন্তু negative index ব্যবহার করে শেষ দিক থেকে element নেওয়া যায় না। `at()` positive ও negative—দুই ধরনের index support করে।

```js
const junkFoodILove = ["🥖", "🍔", "🍟", "🍕", "🌭", "🥪", "🌮", "🍿"];

junkFoodILove.at(0);  // '🥖'
junkFoodILove.at(3);  // '🍕'
junkFoodILove.at(-1); // '🍿'
junkFoodILove.at(-5); // '🍕'
junkFoodILove.at(-8); // '🥖'
junkFoodILove.at(10); // undefined
```

Negative index-এ counting ডান দিক থেকে শুরু হয়: `-1` শেষ element, `-2` শেষের আগের element। Range-এর বাইরে গেলে `undefined`।

# 25. `copyWithin()` Method

এটি খুব বেশি ব্যবহৃত না হলেও code review-তে দেখা যেতে পারে। এটি একই array-এর একটি অংশকে একই array-এর অন্য position-এ copy করে। অন্য array তৈরি করে না এবং source mutate করে।

Syntax:

```js
array.copyWithin(target, start, end)
```

- `target`: কোথা থেকে copied sequence বসবে
- `start`: কোথা থেকে copy নেওয়া শুরু হবে
- `end`: কোথায় copy থামবে; optional এবং exclusive

```js
const array = [1, 2, 3, 4, 5, 6, 7];
array.copyWithin(0, 3, 6);
console.log(array);
```

`start = 3` থেকে `end = 6`-এর আগে পর্যন্ত element হলো `[4, 5, 6]`। এগুলো target index `0` থেকে বসে।

Expected output:

```js
[4, 5, 6, 4, 5, 6, 7]
```

আরেকটি উদাহরণ:

```js
const array1 = [1, 2, 3, 4, 5, 6, 7];
array1.copyWithin(0, 4);
console.log(array1);
```

End না থাকায় index `4` থেকে শেষ পর্যন্ত `[5, 6, 7]` copy হয়ে index `0` থেকে বসবে:

```js
[5, 6, 7, 4, 5, 6, 7]
```

# 26. `flat()` Method

Nested array-কে নির্দিষ্ট depth পর্যন্ত flatten করতে `flat()` ব্যবহার করা হয়।

```js
const arr1 = [0, 1, 2, [3, 4]];
console.log(arr1.flat());
```

Expected output:

```js
[0, 1, 2, 3, 4]
```

Default depth `1`।

```js
const arr2 = [0, 1, [2, [3, [4, 5]]]];

console.log(arr2.flat());
// [0, 1, 2, [3, [4, 5]]]

console.log(arr2.flat(2));
// [0, 1, 2, 3, [4, 5]]

console.log(arr2.flat(Infinity));
// [0, 1, 2, 3, 4, 5]
```

Requirement অনুযায়ী depth নির্বাচন করো। সব nesting flatten করতে `Infinity` ব্যবহার করা যায়।

# 27. Data Grouping: `Object.groupBy()`

যদিও data array-তে আছে, grouping-এর methodটি `Object`-এর static method:

```js
Object.groupBy(iterable, callback)
```

এটি iterable-এর element-গুলো callback থেকে পাওয়া key অনুযায়ী group করে। Instructor উল্লেখ করেন, methodটি তুলনামূলক নতুন; পুরোনো environment-এ polyfill লাগতে পারে।

Employee data:

```js
const employees = [
    { name: "Bob", dept: "Engineering", salary: 5000 },
    { name: "Alex", dept: "HR", salary: 3000 },
    { name: "Ravi", dept: "Engineering", salary: 7000 },
    { name: "John", dept: "Engineering", salary: 1000 },
    { name: "Tom", dept: "Sales", salary: 6000 },
];
```

## 27.1 Department অনুযায়ী group

```js
const groupedByDept = Object.groupBy(employees, ({ dept }) => dept);
console.log(groupedByDept);
```

Result-এ `Engineering`, `HR`, `Sales` key থাকবে। Engineering-এ Bob, Ravi ও John; HR-এ Alex; Sales-এ Tom। UI-তে accordion বা grouped list বানাতে এই result সরাসরি কাজে লাগতে পারে।

## 27.2 Business condition অনুযায়ী group

```js
const groupedByMoreThan5000 = Object.groupBy(
    employees,
    ({ salary }) => {
        return salary >= 5000 ? "More than 5k" : "Less than 5k";
    }
);

console.log(groupedByMoreThan5000);
```

# 28. Immutability

Programming-এ source data-কে সরাসরি পরিবর্তন না করে copy তৈরি করে নতুন state বানানোর পদ্ধতিকে immutable approach বলা হয়। Data application-এর source of truth। যদি বিভিন্ন function বিভিন্ন সময়ে একই source data সরাসরি mutate করে, পরে কে, কখন, কেন পরিবর্তন করেছে তা trace করা কঠিন হয়। Debugging ও predictability কমে যায়।

Immutable approach-এ প্রতিটি পরিবর্তনের জন্য নতুন copy বা snapshot তৈরি হয়। ফলে previous state, current state এবং future state compare করা সহজ। Modern application development-এ এটি অত্যন্ত গুরুত্বপূর্ণ।

`reverse()`, `sort()` ও `splice()` mutable। এগুলোর modern immutable alternative হলো:

- `toReversed()`
- `toSorted()`
- `toSpliced()`

আর index-এর value immutableভাবে বদলাতে আছে `with()`।

---

# 29. `toReversed()` Method

```js
const items = [1, 2, 3];
const reversedItems = items.toReversed();

console.log(reversedItems); // [3, 2, 1]
console.log(items);         // [1, 2, 3]
```

`reverse()` source বদলাত, `toReversed()` নতুন array return করে।

---

# 30. `toSorted()` Method

```js
const months = ["Mar", "Jan", "Feb", "Dec"];
const sortedMonths = months.toSorted();

console.log(sortedMonths); // ["Dec", "Feb", "Jan", "Mar"]
console.log(months);       // ["Mar", "Jan", "Feb", "Dec"]
```

এটি alphabetical sort করেছে, কিন্তু source array অপরিবর্তিত। Custom comparator-ও দেওয়া যায়।

---

# 31. `toSpliced()` Method

`splice()`-এর মতো add, delete ও replace করে, কিন্তু original array mutate করে না; পরিবর্তিত নতুন array return করে।

```js
const months = ["Jan", "Mar", "Apr", "May"];
const months2 = months.toSpliced(1, 0, "Feb");

console.log("Original Array", months);
console.log("Spliced Array", months2);
```

Expected output:

```js
Original Array: ["Jan", "Mar", "Apr", "May"]
Spliced Array:  ["Jan", "Feb", "Mar", "Apr", "May"]
```

Advice: সম্ভব হলে mutable method-এর বদলে immutable alternative ব্যবহার করা ভালো।

# 32. `with()` Method

Index অনুযায়ী element replace করার immutable উপায় `with(index, value)`।

সরাসরি assignment:

```js
numbers[2] = 6;
```

এটি source array mutate করে। Immutable version:

```js
const numbers = [1, 2, 3, 4, 5];
const newArray = numbers.with(2, 6);

console.log(numbers);  // [1, 2, 3, 4, 5]
console.log(newArray); // [1, 2, 6, 4, 5]
```

Negative index-ও support করে:

```js
const anotherArray = numbers.with(-2, 8);
console.log(anotherArray); // [1, 2, 3, 8, 5]
```

সরাসরি `numbers[-2] = 8` array-এর second-last element বদলায় না; বরং `"-2"` নামে object property তৈরি করতে পারে। `with(-2, 8)` সত্যিকারের negative indexing ব্যবহার করে।

# 33. Array-like Object

Array-like object দেখতে array-এর মতো:

1. Numeric index-সদৃশ key থাকে
2. `length` property থাকে

কিন্তু এটি প্রকৃত array নয়, তাই array method সরাসরি ব্যবহার করা যায় না।

```js
const arrLike = {
    0: "I",
    1: "am",
    2: "array-like",
    length: 3,
};

console.log(arrLike[2]);     // 'array-like'
console.log(arrLike.length); // 3

console.log(Array.isArray(arrLike));      // false
console.log(arrLike instanceof Object);  // true
```

`arrLike.pop()` বা `arrLike.forEach()` করলে `TypeError` হতে পারে, কারণ method-গুলো এর prototype chain-এ নেই।

```js

}

checkArgs(1, 45);
```

`arguments[0]` হলো `1`, `arguments[1]` হলো `45`, এবং `length` আছে। তবু এটি array নয়।

## 33.2 HTMLCollection

```js
document.getElementsByTagName("li")
```

# 34. Array-like থেকে Array

এখন `argArr` প্রকৃত array, তাই `forEach()` ব্যবহার করা যাচ্ছে।

# 35. `Array.fromAsync()` Static Method

```js
const collectionPromise = Array.fromAsync(

console.log("Converted Array", collectionPromise);
collectionPromise.then((value) => console.log(value));
```

এটি সরাসরি array নয়; একটি `Promise` return করে। Resolved value পেতে `then()` বা `await` লাগবে।

```js
const ret = Array.fromAsync({

    2: Promise.resolve("Apple"),
```


`ret` নিজে promise chain-এর result।

# 38. `filter()` Method

```js
const seniorCustomers = customers.filter((customer) => {
```

# 39. `map()` Method

`map()` প্রতিটি element transform করে সমান length-এর একটি নতুন array return করে।

Use case: gender ও marital status দেখে title নির্ধারণ করে full name যোগ করা।

```js
const customersWithFullName = customers.map((customer) => {
    let title = "";

    if (customer.gender === "M") {
        title = "Mr.";
    } else if (customer.gender === "F" && customer.married) {
        title = "Mrs.";
    } else {
        title = "Miss";
    }

    customer["full_name"] =
        `${title} ${customer.f_name} ${customer.l_name}`;

    return customer;
});

console.log(
    "Customer after adding fullname",
    customersWithFullName
);
```

Reasoning:

- Male → `Mr.`
- Female এবং married → `Mrs.`
- অন্যথায় → `Miss`

## Technical Note

উপরের transcript code-এ `customer` object সরাসরি mutate করা হয়েছে। `map()` নিজে নতুন array return করলেও ভেতরের object reference একই হতে পারে। কঠোর immutable style চাইলে নতুন object return করা যায়:

```js
return {
    ...customer,
    full_name: `${title} ${customer.f_name} ${customer.l_name}`,
};
```

এটি মূল lecture-এর বাইরে নতুন lesson নয়; শুধু mutation আচরণটি পরিষ্কার করার সতর্কতা।

# 40. `reduce()` Method

`reduce()` array-কে ধাপে ধাপে একটি single result-এ reduce করে। সাধারণ structure:

```js
array.reduce(
    function reducer(accumulator, currentValue, index, array) {
        // accumulator ও currentValue দিয়ে result তৈরি
        // result return করতে হবে
    },
    initialValue
);
```

`reducer` callback প্রতিটি element-এর জন্য চলে। প্রতিবার return করা value পরবর্তী iteration-এর `accumulator` হয়।

## 40.1 Basic sum

```js
const arr = [1, 2, 3, 4, 5];

const result = arr.reduce((accumulator, currentValue) => {
    return accumulator + currentValue;
}, 0);

console.log(result); // 15
```

Step-by-step:

- শুরু accumulator `0`
- `0 + 1 = 1`
- `1 + 2 = 3`
- `3 + 3 = 6`
- `6 + 4 = 10`
- `10 + 5 = 15`

## 40.2 Book কেনা customer-দের average age

```js
let count = 0;

const total = customers.reduce((accumulator, customer) => {
    if (customer.purchased.includes("Book")) {
        accumulator = accumulator + customer.age;
        count = count + 1;
    }

    return accumulator;
}, 0);

console.log(
    "Customer Avg age Purchased Book:",
    Math.floor(total / count)
);
```

Book কিনেছে Abby (`32`), Dianna (`22`) এবং Dev (`82`)।

```text
Total age = 32 + 22 + 82 = 136
Count = 3
Average = floor(136 / 3) = 45
```

Expected output:

```text
Customer Avg age Purchased Book: 45
```

# 41. `reduceRight()` Method

`reduceRight()` ডান দিক থেকে বাম দিকে reduce করে। Non-commutative operation-এ result `reduce()` থেকে আলাদা হতে পারে।

```js
let number = [100, 40, 15];

const subsResult = number.reduceRight((accumulator, current) => {
    return accumulator - current;
});

console.log("Subs", subsResult);
```

ডান দিক থেকে:

```text
15 - 40 = -25
-25 - 100 = -125
```

Expected output:

```text
Subs -125
```

তুলনায় `reduce()` বাম দিক থেকে করলে:

```text
100 - 40 = 60
60 - 15 = 45
```

# 46. Array Method Chaining

এক method-এর output-এর ওপর পরবর্তী method call করাকে chaining বলা হয়। Use case: married customer-দের মোট expense বের করা।

ধাপগুলো:

1. `filter()` দিয়ে married customer নেওয়া
2. `map()` দিয়ে শুধু expense নেওয়া
3. `reduce()` দিয়ে expense যোগ করা

```js
const totalExpense = customers
    .filter((customer) => {
        return customer.married;
    })
    .map((marriedCustomer) => {
        return marriedCustomer.expense;
    })
    .reduce((accum, expense) => {
        return accum + expense;
    }, 0);

console.log(
    "Total Expense of Married Customers in INR: ",
    totalExpense
);
```

Married customers-এর expense:

```text
500 + 100 + 1500 + 90 = 2190
```

Expected output:

```text
Total Expense of Married Customers in INR: 2190
```

Chaining শক্তিশালী, তবে readability বজায় রাখতে logical formatting জরুরি।

# 47. `forEach()` Method

`forEach()` প্রতিটি element-এর জন্য callback চালায়। এটি সাধারণত side effect-এর জন্য ব্যবহৃত হয় এবং নতুন array return করে না।

```js
const arr = [1, 2, 3, 4, 5];
```

# 50. `flatMap()` Method

`flatMap()` প্রথমে `map()` করে, তারপর result-কে এক level `flat()` করে।

## 50.1 Simple transformation

```js
const arr1 = [1, 2, 3, 4];

console.log(
    "simple map",
    arr1.map((item) => item * 2)
);

console.log(
    "simple flatmap",
    arr1.flatMap((item) => item * 2)
);
```

দুটির output-ই:

```js
[2, 4, 6, 8]
```

কারণ map result nested নয়, flatten করার কিছু নেই।

## 50.2 Nested result

```js
console.log(
    "complex map",
    arr1.map((item) => [item * 2])
);

console.log(
    "complex flat map",
    arr1.flatMap((item) => [item * 2])
);
```

`map()` output:

```js
[[2], [4], [6], [8]]
```

`flatMap()` output:

```js
[2, 4, 6, 8]
```

## Quiz

```js
arr1.map((item) => [[item * 2]]);
arr1.flatMap((item) => [[item * 2]]);
```

`map()` output হবে:

```js
[[[2]], [[4]], [[6]], [[8]]]
```

`flatMap()` শুধু এক level flatten করে, তাই output:

```js
[[2], [4], [6], [8]]
```

কেননা ভিতরে দুই level nesting তৈরি হয়েছিল; এক level মাত্র সরেছে।

# Common Mistakes ও Tricky Cases

## 1. `new Array(2)`-কে `[2]` ভাবা

এটি length `2`-এর sparse array তৈরি করে। Element `2` চাইলে `Array.of(2)` বা `[2]` ব্যবহার করো।

## 2. একই content-এর দুই array-কে strict equal ভাবা

```js
[1, 2] === [1, 2] // false
```

Reference আলাদা।

## 3. `push()` নতুন array return করে ভাবা

`push()` updated length return করে এবং source mutate করে।

## 4. `pop()`/`shift()` length return করে ভাবা

এগুলো removed element return করে।

## 5. `slice()` ও `splice()` গুলিয়ে ফেলা

`slice()` copy; `splice()` change।

## 6. Default `sort()`-কে numeric sort ভাবা

Default comparison string-based। Number-এর জন্য comparator দাও।

## 7. `includes()` case-insensitive ভাবা

এটি case-sensitive।

## 8. Array-like object-এ array method call করা

প্রথমে spread বা `Array.from()` দিয়ে convert করো।

## 9. `map()` ব্যবহার করেও nested object mutate করা

`map()` নতুন outer array দেয়, কিন্তু callback-এ original object বদলালে nested reference mutate হতে পারে।

## 10. `flatMap()` সব depth flatten করে ভাবা

এটি এক level flatten করে।

# Interview-oriented Questions

## প্রশ্ন 1

`new Array(3)` এবং `Array.of(3)`-এর পার্থক্য কী?

**উত্তর:** প্রথমটি length `3`-এর sparse array; দ্বিতীয়টি element `3`-সহ `[3]`।

## প্রশ্ন 2

`push()` ও `unshift()` কী return করে?

**উত্তর:** Operation শেষে array-এর নতুন length।

## প্রশ্ন 3

`pop()` ও `shift()` কী return করে?

**উত্তর:** Removed element।

## প্রশ্ন 4

```js
const [first, , third] = array;
```

## প্রশ্ন 5

`indexOf()` element না পেলে কী return করে?

**উত্তর:** `-1`।

## প্রশ্ন 6

`find()` ও `filter()`-এর পার্থক্য কী?

**উত্তর:** `find()` প্রথম matching element; `filter()` সব matching element-এর নতুন array।

## প্রশ্ন 7

`some()` ও `every()`-এর পার্থক্য কী?

Mutable `reverse`, `sort`, `splice`-এর immutable alternative কী?

## প্রশ্ন 9

`flatMap()` কত level flatten করে?

**উত্তর:** এক level।
