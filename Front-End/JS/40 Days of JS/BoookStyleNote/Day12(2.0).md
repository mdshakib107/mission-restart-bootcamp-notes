# 2. JavaScript Object কী এবং কেন দরকার

Application-এ data দরকার। আগের lesson-গুলোতে data সাধারণ variable-এ রাখা হয়েছে। যেমন:

```js
let age = 40;
let isActive = true;
let name = "Tapas";
```

এগুলো `primitive` data type:

- `number`
- `string`
- `boolean`
- ইত্যাদি

কিন্তু real-world application-এ complex data আসে। যেমন:

- একজন user-এর data
- একটি company-এর data
- একজন employee-এর data
- একটি product-এর data
- একটি student-এর profile

এ ধরনের data একটি মাত্র primitive variable-এ ভালোভাবে রাখা যায় না। তখন structured data দরকার। JavaScript-এ এমন একটি বহুল ব্যবহৃত data structure হলো `object`।

## Instructor-এর Definition

> **Objects are nothing but key-value pairs.**

আরও বলা হয়েছে:

> Object is a **keyed collection**—অর্থাৎ unique key ব্যবহার করে value সংরক্ষণ এবং পরে retrieve করা যায়।

উদাহরণ:

```js
const user = {
  name: "Tapas",
  age: 40
};
```

এখানে:

| Key | Value |
|---|---|
| `name` | `"Tapas"` |
| `age` | `40` |

### সহজ explanation

Object-কে একটি labelled storage box হিসেবে ভাবা যায়। Box-এর ভেতরে প্রতিটি তথ্যের একটি label বা key থাকে। সেই key ব্যবহার করে তথ্যটি পাওয়া যায়।

### মনে রাখার নিয়ম

> Array সাধারণত position/index-ভিত্তিক; object key-ভিত্তিক।

---

# 3. Object Literal Syntax দিয়ে Object তৈরি

Object তৈরির সবচেয়ে straightforward এবং common উপায় হলো `object literal syntax`।

```js
const user = {};
```

এক জোড়া curly braces `{}` নিজেই একটি object। এর ভেতরে কোনো তথ্য না থাকলে একে `empty object` বলা হয়।

```js
let user = {};
```

এখানে `user` variable একটি object value ধরে রেখেছে।

## Empty Object বনাম Data-সহ Object

```js
const emptyUser = {};

const user = {
  name: "Tapas",
  age: 40
};
```

### Common mistake

```js
let user = "{}";
```

এটি object নয়, string। কারণ braces quotation-এর ভেতরে আছে।

### মনে রাখার নিয়ম

> `{}` → object value, কিন্তু `"{}"` → string value।

---

# 4. Object Property: Key–Value Pair

Object-এর ভেতরে প্রতিটি তথ্যকে `property` বলা হয়। প্রতিটি property সাধারণত `key: value` আকারে লেখা হয়।

```js
const user = {
  name: "Tapas",
  age: 40
};
```

এখানে:

- `name: "Tapas"` একটি property
- `age: 40` আরেকটি property

Property-গুলোর মাঝে comma ব্যবহার করা হয়।

```js
const product = {
  title: "Keyboard",
  price: 2500,
  inStock: true
};
```

### Syntax breakdown

```js
const objectName = {
  key1: value1,
  key2: value2
};
```

### Common mistakes

ভুল:

```js
const user = {
  name = "Tapas",
  age = 40
};
```

সঠিক:

```js
const user = {
  name: "Tapas",
  age: 40
};
```

Object property-তে key ও value আলাদা করতে `:` লাগে, `=` নয়।

---

# 5. Object Property Access করা

Object থেকে property value retrieve করার সবচেয়ে পরিচিত উপায় হলো `dot notation`।

```js
const user = {
  name: "Tapas",
  age: 40
};

console.log(user.name); // Tapas
console.log(user.age);  // 40
```

## Instructor-এর ব্যাখ্যা

Dot হলো এমন operator, যার সাহায্যে object-এর key-এর value query করা যায়।

```js
objectName.propertyName
```

### Example

```js
const book = {
  title: "JavaScript Basics",
  pages: 250
};

console.log(book.title);
console.log(book.pages);
```

### Common mistake

```js
console.log(user."name");
```

এটি invalid syntax। Dot notation-এ key quotation-এর মধ্যে লেখা হয় না।

---

# 6. নতুন Property যোগ করা

Object তৈরি হয়ে যাওয়ার পরও নতুন property যোগ করা যায়।

```js
const user = {
  name: "Tapas",
  age: 40
};

user.isSeniorCitizen = false;

console.log(user);
```

Output conceptually:

```js
{
  name: "Tapas",
  age: 40,
  isSeniorCitizen: false
}
```

### গুরুত্বপূর্ণ বিষয়: `const` object-এ property যোগ করা যায় কেন?

`const` variable-কে নতুন object দিয়ে reassign করা যায় না, কিন্তু object-এর ভেতরের property পরিবর্তন করা যায়।

Allowed:

```js
const user = { name: "Tapas" };
user.age = 40;
```

Not allowed:

```js
const user = { name: "Tapas" };
user = { name: "Bob" }; // TypeError
```

### মনে রাখার নিয়ম

> `const` reference-কে স্থির রাখে; object-কে automatically immutable করে না।

# 7. Special Character-যুক্ত Key

Key-তে space বা অন্য special character থাকলে key-কে quotation-এর মধ্যে লিখতে হয়।

```js
const user = {
  name: "Tapas",
  "is admin": true
};
```

সাধারণ key-গুলো quotation ছাড়াও লেখা যায়, কারণ JavaScript internally এগুলোকে string হিসেবে treat করে।

```js
const user = {
  name: "Tapas",
  age: 40
};
```

কিন্তু space-যুক্ত key এভাবে লেখা যাবে না:

```js
const user = {
  is admin: true // SyntaxError
};
```

## Special key access করার নিয়ম

Dot notation কাজ করবে না:

```js
console.log(user.is admin); // SyntaxError
```

Bracket/subscript notation ব্যবহার করতে হবে:

```js
console.log(user["is admin"]); // true
```

## পরে special key যোগ করা

```js
user["movie lover"] = true;
```

### Common mistake

```js
user.movie lover = true;
```

এটি invalid।

### মনে রাখার নিয়ম

> Key-তে space/special character থাকলে: **quotation + bracket notation**।

# 8. Existing Property Modify করা

Existing property-র value নতুন value assign করে update করা যায়।

```js
const user = {
  name: "Tapas",
  age: 40
};

user.age = 34;
console.log(user.age); // 34
```

Special key modify করতে bracket notation ব্যবহার করতে হবে:

```js
user["movie lover"] = false;
```

### Common mistake

অনেকে ভাবেন object literal-এ একবার value দিলে আর বদলানো যাবে না। বাস্তবে normal object mutable।

### মনে রাখার নিয়ম

> Existing property-তে নতুন assignment দিলে value overwrite হয়।

---

# 9. Property Delete করা

Property delete করার জন্য JavaScript-এর `delete` keyword ব্যবহার করা হয়।

```js
const user = {
  name: "Tapas",
  age: 40,
  "movie lover": true
};

delete user["movie lover"];
delete user.age;

console.log(user);
```

### Syntax

```js
delete objectName.propertyName;
```

অথবা:

```js
delete objectName["special key"];
```

### Common mistake

```js
delete user;
```

এভাবে variable binding delete করা যায় না। `delete` মূলত object property-এর জন্য ব্যবহৃত হয়।

# 10. Dynamic Key Access

Real-world programming-এ সব key আগে থেকে hardcoded থাকে না। Key আসতে পারে:

- user input থেকে
- API response থেকে
- computation থেকে
- অন্য variable থেকে

উদাহরণ:

```js
const user = {
  name: "Tapas",
  age: 34
};

const someKey = "age";
```

ভুল:

```js
console.log(user.someKey);
```

এখানে JavaScript `someKey` variable-এর value দেখবে না; বরং object-এ literal `someKey` নামে property খুঁজবে।

সঠিক:

```js
console.log(user[someKey]); // 34
```

Runtime-এ `someKey`-এর value `"age"` বসে যায়। Conceptually:

```js
user[someKey]
// becomes
user["age"]
```

### মনে রাখার নিয়ম

> Dynamic key হলে dot নয়, bracket notation ব্যবহার করতে হবে।

# 11. Computed/Dynamic Property দিয়ে Object তৈরি

Object থেকে dynamic key access করা যায়; একইভাবে object তৈরির সময় key-ও dynamic হতে পারে।

Instructor prompt-এর উদাহরণ দেন:

```js
const car = prompt("Which is your favorite car?");

const favoriteCars = {
  [car]: 5
};

console.log(favoriteCars);
```

User যদি `BMW` লেখে:

```js
{ BMW: 5 }
```

User যদি `Audi` লেখে:

```js
{ Audi: 5 }
```

এখানে `[car]` হলো computed property name।

### Example

```js
const fieldName = "email";

const formData = {
  [fieldName]: "student@example.com"
};

console.log(formData.email);
```

### Common mistake

```js
const formData = {
  fieldName: "student@example.com"
};
```

এখানে key হবে literal `fieldName`, variable-এর value `email` নয়।

# 12. Object Creation Patterns

Instructor object তৈরির একাধিক pattern দেখিয়েছেন। Prototype এবং OOP-তে এগুলো পরে আরও গুরুত্বপূর্ণ হবে।

## 12.1 Object Literal

সবচেয়ে common উপায়:

```js
const user = {
  name: "Tapas",
  age: 40
};
```

### সুবিধা

- সহজ
- concise
- ছোট বা one-off object-এর জন্য ভালো

### সীমাবদ্ধতা

একই structure-এর অনেক object তৈরি করতে বারবার code লিখতে হতে পারে।

---

## 12.2 Constructor Function

Constructor function দেখতে regular function-এর মতো, কিন্তু convention অনুযায়ী এর নাম capital letter দিয়ে শুরু হয়।

```js
function Car(name, model) {
  this.name = name;
  this.model = model;
}
```

এখানে:

- `name` ও `model` হলো parameters
- `this.name` ও `this.model` object-এর properties
- `this` keyword পরের lesson-এ বিস্তারিত আসবে

Object তৈরি করতে `new` keyword ব্যবহার করতে হবে:

```js
const bmwCar = new Car("BMW", "X1");
const audiCar = new Car("Audi", "A8");

console.log(bmwCar);
console.log(audiCar);
```

## Parameter বনাম Argument

Instructor আগের function lesson-এর সঙ্গে connection দেখান:

- **Parameter:** function define করার সময় placeholder
- **Argument:** function call করার সময় actual value

```js
function Car(name, model) { // parameters
  this.name = name;
  this.model = model;
}

new Car("BMW", "X1"); // arguments
```

## Constructor Function-এর বড় সুবিধা

একটি blueprint থেকে একাধিক instance তৈরি করা যায়।

```js
console.log(bmwCar instanceof Car); // true
```

এতে জানা যায় object কোন constructor/blueprint থেকে তৈরি।

### Common mistakes

1. Constructor function-এর নাম lowercase করা

```js
function car(name, model) {}
```

Syntax error নয়, কিন্তু convention ভাঙে এবং regular function-এর সঙ্গে confusion হয়।

2. `new` বাদ দেওয়া

```js
const bmwCar = Car("BMW", "X1");
```

এটি intended object creation করবে না।

### মনে রাখার নিয়ম

> Constructor Function → Capitalized name + `this` + `new`।

---

## 12.3 Built-in Object Constructor

JavaScript-এর built-in `Object` constructor থেকেও object তৈরি করা যায়।

```js
const person = new Object();

person.name = "Alpha";
person.age = 76;

console.log(person);
```

এটি valid, কিন্তু custom type দেয় না। Constructor function দিয়ে `Car` type তৈরি করা যায়; `new Object()` শুধু generic object দেয়।

### Practical note

সাধারণভাবে object literal বেশি readable:

```js
const person = {
  name: "Alpha",
  age: 76
};
```

---

## 12.4 Factory Function

Factory function হলো regular function, যা নতুন object return করে।

```js
function createUser(name, age) {
  return {
    name,
    age
  };
}

const user1 = createUser("Tapas", 39);
const user2 = createUser("Bob", 32);
```

এখানে `new` দরকার নেই।

## Instructor-এর Factory Function ধারণা

Factory function user input অনুযায়ী different object/functionality produce করতে পারে। এই lesson-এ function প্রতিবার নতুন object return করছে।

### Constructor বনাম Factory Function

| বিষয় | Constructor Function | Factory Function |
|---|---|---|
| Naming convention | Capital letter | সাধারণ camelCase |
| `new` লাগে? | হ্যাঁ | না |
| `this` ব্যবহার | সাধারণত হয় | প্রয়োজন নেই |
| Custom instance check | `instanceof` করা যায় | সাধারণত custom type থাকে না |
| Invocation | `new Car(...)` | `createUser(...)` |

### Common mistake

Factory function-কে `new` দিয়ে call করার প্রয়োজন নেই:

```js
const user = new createUser("Tapas", 39); // unnecessary/wrong style
```

## 12.5 Object Property Shorthand

যখন key name এবং variable/parameter name একই হয়, তখন shorthand ব্যবহার করা যায়।

Full form:

```js
function createUser(name, age) {
  return {
    name: name,
    age: age
  };
}
```

Shorthand:

```js
function createUser(name, age) {
  return {
    name,
    age
  };
}
```

JavaScript internally conceptually এটিকে expand করে:

```js
{
  name: name,
  age: age
}
```

### কখন shorthand ব্যবহার করা যাবে?

যখন property key এবং variable/parameter-এর নাম **exactly same**।

```js
const title = "Book";
const product = { title };
```

কিন্তু ভিন্ন key চাইলে full syntax লাগবে:

```js
const title = "Book";
const product = {
  productTitle: title
};
```

# 13. Object Method

Object property-এর value primitive ছাড়াও non-primitive হতে পারে। Instructor বলেন non-primitive value-এর মধ্যে রয়েছে:

- object
- array
- function

Object-এর ভেতরে function থাকলে সেই function-কে `method` বলা হয়।

## Instructor-এর Definition

> Whenever you have a function inside an object, you call that function a method.

উদাহরণ:

```js
const user = {
  name: "Tapas",
  greet: function () {
    console.log(this.name);
  }
};

user.greet();
```

Method call করতে parentheses দিতে হবে:

```js
user.greet();
```

শুধু reference পেতে:

```js
user.greet;
```

## আরও একটি example

```js
const profile = {
  name: "Tapas",
  company: "CreoWis",
  message: function () {
    console.log(`${this.name} works at ${this.company}`);
  }
};

profile.message();
```

Instructor এখানে template literal সংক্ষেপে ব্যাখ্যা করেন:

```js
`${this.name} works at ${this.company}`
```

`${...}` runtime-এ dynamic value বসায়।

### Common mistake

```js
profile.message;
```

এতে method execute হবে না; শুধু function reference পাওয়া যাবে। Execute করতে `()` দরকার।

# 14. Nested Object

একটি object-এর property value আরেকটি object হলে তাকে `nested object` বলা হয়।

## Instructor-এর Definition

> An object can have a property whose value can be another object.

উদাহরণ:

```js
const profile = {
  name: "Tapas",
  company: "CreoWis",
  address: {
    city: "Bangalore",
    zip: 560032,
    state: "Karnataka",
    country: "India"
  }
};
```

এখানে `address` হলো outer object `profile`-এর nested object।

## Nested property access

```js
console.log(profile.address.country); // India
```

Access chain:

1. `profile`
2. `profile.address`
3. `profile.address.country`

Nested object-এর মধ্যে method-ও থাকতে পারে:

```js
const profile = {
  address: {
    country: "India",
    greeting: function () {
      console.log("Welcome to India");
    }
  }
};

profile.address.greeting();
```

### Instructor-এর গুরুত্বপূর্ণ কথা

Object nesting যেকোনো depth পর্যন্ত যেতে পারে। এর মাধ্যমে complex data structure তৈরি করা যায়।

### Common mistake

```js
console.log(profile.country);
```

`country` top-level property নয়; এটি `address`-এর ভেতরে।

সঠিক:

```js
console.log(profile.address.country);
```

# 15. `in` Operator দিয়ে Property Existence Check

কোনো property object-এ সত্যিই আছে কি না, তা check করতে `in` operator ব্যবহার করা যায়।

প্রথমে Instructor একটি flawed approach দেখান:

```js
if (!profile.salary) {
  console.log("The salary property doesn't exist");
}
```

এটি সবসময় reliable নয়। কারণ property থাকতে পারে, কিন্তু value হতে পারে:

- `undefined`
- `0`
- `false`
- `""`
- `null`

উদাহরণ:

```js
const profile = {
  salary: undefined
};

if (!profile.salary) {
  console.log("The salary property doesn't exist");
}
```

এখানে message ভুল; property আছে, শুধু value `undefined`।

## সঠিক উপায়

```js
console.log("salary" in profile);
```

Property থাকলে `true`, না থাকলে `false`।

```js
const profile = {
  salary: undefined
};

console.log("salary" in profile); // true
console.log("age" in profile);    // false
```

### মনে রাখার নিয়ম

> Value আছে কি না এবং property আছে কি না—দুটি ভিন্ন প্রশ্ন। Property existence check-এ `in` ব্যবহার করো।

# 16. `for...in` Loop দিয়ে Object Traverse

Object-এর সব key iterate করতে `for...in` loop ব্যবহার করা যায়।

```js
const profile = {
  name: "Tapas",
  company: "CreoWis",
  age: 40
};

for (let key in profile) {
  console.log(key);
}
```

Output:

```text
name
company
age
```

Value পেতে dynamic bracket access:

```js
for (let key in profile) {
  console.log(key, profile[key]);
}
```

Dot notation কাজ করবে না:

```js
profile.key
```

এতে literal `key` property খোঁজা হবে।

### কেন bracket notation দরকার?

Loop-এর প্রতিটি iteration-এ `key` variable-এর value বদলায়। তাই:

```js
profile[key]
```

ব্যবহার করতে হবে।

### `for...in` বনাম `for...of`

| Loop | সাধারণত কিসের জন্য | কী দেয় |
|---|---|---|
| `for...in` | object properties | key |
| `for...of` | iterable যেমন array | element/value |

# 18. Object Reference ও Comparison

Object হলো non-primitive/reference type। Instructor execution context এবং memory diagram-এর আগের lesson-এর সঙ্গে connection করেন।

## Instructor-এর বক্তব্য

> Object is stored and passed by reference instead of value.

উদাহরণ:

```js
const fruit = {
  name: "Mango"
};

const anotherFruit = {
  name: "Mango"
};

console.log(fruit === anotherFruit); // false
console.log(fruit == anotherFruit);  // false
```

দুই object-এর content একই, কিন্তু memory-তে আলাদা location/reference আছে। তাই comparison false।

ধারণাগতভাবে:

```text
fruit        → memory address XA01
anotherFruit → memory address YB02
```

Values একই হলেও references আলাদা।

## একই reference assign করলে

```js
let fruit = {
  name: "Mango"
};

let anotherFruit = {
  name: "Mango"
};

fruit = anotherFruit;

console.log(fruit === anotherFruit); // true
```

এখন উভয় variable একই object/reference point করছে।

## `const` কেন বদলাতে হলো?

Reassignment করতে হলে `let` দরকার:

```js
const fruit = { name: "Mango" };
fruit = anotherFruit; // error
```

### গুরুত্বপূর্ণ rule

> Object equality values দেখে নয়, reference দেখে নির্ধারিত হয়।

### Common mistake

```js
{ a: 1 } === { a: 1 }
```

এটি `false`। দুইটি আলাদা object instance।

# 19. `Object.assign()`

## Instructor-এর Definition

> `Object.assign()` copies all of an object's own properties from a source object to a target object.

Syntax:

```js
Object.assign(target, source);
```

উদাহরণ:

```js
const target = {
  p: 1,
  q: 2
};

const source = {
  a: 3,
  b: 5
};

const returnedObject = Object.assign(target, source);

console.log(returnedObject);
```

Output:

```js
{
  p: 1,
  q: 2,
  a: 3,
  b: 5
}
```

## Key conflict হলে overriding

```js
const target = {
  p: 1,
  a: 2
};

const source = {
  a: 3,
  b: 5
};

const result = Object.assign(target, source);
```

Final result:

```js
{
  p: 1,
  a: 3,
  b: 5
}
```

Source-এর `a: 3`, target-এর `a: 2` overwrite করে।

## Clone করার pattern

```js
const obj = {
  name: "Tapascript"
};

const obj2 = Object.assign({}, obj);

console.log(obj2);
console.log(obj === obj2); // false
```

দুই object-এর content একই, reference আলাদা।

### Common mistake

অনেকে ভাবেন `Object.assign()` সব level deep copy করে। বাস্তবে এটি shallow copy।

---

# 20. Shallow Copy বনাম Deep Copy

## Shallow Copy

Shallow copy top-level property copy করে, কিন্তু nested object-এর জন্য নতুন nested reference তৈরি করে না; nested reference share হয়।

উদাহরণ:

```js
const obj3 = {
  a: 1,
  b: {
    c: 2
  }
};

const obj4 = Object.assign({}, obj3);

obj4.b.c = 3;

console.log(obj3.b.c); // 3
console.log(obj4.b.c); // 3
```

কেন?

- `obj3` ও `obj4` top-level-এ আলাদা object
- কিন্তু `b` nested object-এর reference একই

ধারণাগতভাবে:

```text
obj3.b ──┐
         ├──> same nested object { c: 2 }
obj4.b ──┘
```

তাই এক জায়গায় `c` বদলালে অন্য জায়গাতেও দেখা যায়।

## Primitive top-level property-এর আচরণ

```js
obj4.a = 100;

console.log(obj4.a); // 100
console.log(obj3.a); // 1
```

Top-level primitive value আলাদাভাবে copied হয়েছে।

## Instructor-এর মূল সতর্কতা

> Nested object clone করার সময় `Object.assign()` ব্যবহার করলে reference copy হয়; independent deep clone হয় না।

---

# 21. `structuredClone()`

Deep cloning-এর জন্য Instructor `structuredClone()` ব্যবহারের কথা বলেন।

```js
const obj3 = {
  a: 1,
  b: {
    c: 2
  }
};

const obj5 = structuredClone(obj3);

obj5.a = 300;
obj5.b.c = 30;

console.log(obj5.a);   // 300
console.log(obj3.a);   // 1
console.log(obj5.b.c); // 30
console.log(obj3.b.c); // 2
```

এখানে nested object-সহ আলাদা references তৈরি হয়েছে। তাই clone-এর nested property পরিবর্তন করলে source object বদলায় না।

## Comparison table

| বিষয় | `Object.assign({}, obj)` | `structuredClone(obj)` |
|---|---|---|
| Copy type | Shallow | Deep |
| Top-level object নতুন? | হ্যাঁ | হ্যাঁ |
| Nested reference নতুন? | না | হ্যাঁ |
| Nested change source-এ impact করে? | হ্যাঁ | না |

### Common mistake

সব JavaScript value `structuredClone()` দিয়ে clone করা যায় না। যেমন function clone করা যায় না। তবে transcript-এর মূল focus ছিল nested plain object-এর deep copy।

### মনে রাখার নিয়ম

> Flat object → shallow copy যথেষ্ট হতে পারে। Nested independent clone → `structuredClone()`।

# 22. `Object.entries()`

`Object.entries()` object-কে array-like structure-এ convert করে। প্রতিটি property `[key, value]` pair হিসেবে একটি nested array হয়।

```js
const obj = {
  a: "Tapas",
  b: 32
};

const myArray = Object.entries(obj);
console.log(myArray);
```

Output:

```js
[
  ["a", "Tapas"],

]
```

### Real-world use case

Instructor বলেন API object দিতে পারে, কিন্তু UI এমনভাবে তৈরি যে data loop করার জন্য array দরকার। তখন `Object.entries()` useful।

```js
for (const [key, value] of Object.entries(obj)) {
  console.log(key, value);
}
```

### মনে রাখার নিয়ম

> `Object.entries()` → object থেকে `[key, value]` pair-এর array।

# 23. `Object.fromEntries()`

`Object.fromEntries()` হলো `Object.entries()`-এর বিপরীত কাজের method। এটি key-value pair array বা map-like entries থেকে object তৈরি করে।

```js
const entries = [
  ["foo", "bar"],
  ["baz", 42]
];

const obj = Object.fromEntries(entries);

console.log(obj);
```

Output:

```js
{
  foo: "bar",
  baz: 42
}
```

### Conversion direction

```text
Object --Object.entries()--> Array of [key, value]
Array of [key, value] --Object.fromEntries()--> Object
```

### Common mistake

Input pair structure ঠিক না হলে expected object পাওয়া যাবে না। প্রতিটি entry ideally দুই element-এর হওয়া উচিত:

```js
[key, value]
```

# 24. Immutability: `Object.freeze()`

## Instructor-এর Definition

> Immutable means something that you cannot change. Mutable means something that you can change.

`Object.freeze()` object-কে এমনভাবে freeze করে, যাতে:

- existing property value modify করা যায় না
- new property add করা যায় না
- existing property delete করা যায় না

```js
const employee = {
  salary: 100
};

Object.freeze(employee);

employee.salary = 200;
employee.name = "Alex";
delete employee.salary;

console.log(employee);
```

Object unchanged থাকবে:

```js
{
  salary: 100
}
```

## Frozen কি না check

```js
console.log(Object.isFrozen(employee)); // true
```

### Important note

Non-strict mode-এ mutation silently fail করতে পারে; strict mode-এ error আসতে পারে। Transcript-এ output unchanged দেখানো হয়েছে।

### Common mistake

`Object.freeze()` deep freeze নয়। Nested object থাকলে nested object আলাদাভাবে freeze না করলে সেটি mutable থাকতে পারে। Transcript-এ এই deep-freeze limitation আলোচনা করা হয়নি, কিন্তু বাস্তবে মনে রাখা ভালো।

---

# 25. Immutability: `Object.seal()`

`Object.seal()`-ও object structure lock করে, কিন্তু `freeze()`-এর সঙ্গে গুরুত্বপূর্ণ পার্থক্য আছে।

Seal করলে:

- নতুন property add করা যায় না
- existing property delete করা যায় না
- existing property value modify করা যায়

```js
const department = {
  name: "Finance"
};

Object.seal(department);

department.address = "Bangalore"; // add হবে না
delete department.name;           // delete হবে না
department.name = "HR";           // modify হবে

console.log(department);
```

Output:

```js
{
  name: "HR"
}
```

## Freeze বনাম Seal

| Operation | `Object.freeze()` | `Object.seal()` |
|---|---:|---:|
| Existing value modify | না | হ্যাঁ |
| New property add | না | না |
| Property delete | না | না |

### মনে রাখার analogy

- `freeze()` → ভেতরের value-সহ সম্পূর্ণ জমে গেছে
- `seal()` → box sealed; item add/remove করা যাবে না, কিন্তু existing label-এর value বদলানো যাবে

# 27. Object Destructuring কী

Instructor বলেন `destructure` মানে কোনো structure ভেঙে তার অংশ আলাদা করে নেওয়া। Object-এর ক্ষেত্রে property values বের করে variable-এ রাখা হয়।

ধরা যাক:

```js
const student = {
  name: "John Williamson",
  age: 9,
  standard: 3,
  subjects: ["Math", "English", "Science"],
  parents: {
    father: "Mr. Williamson",
    mother: "Mrs. Williamson",
    email: "john.parents@abcd.com"
  },
  address: {
    street: "Example Street",
    city: "John City",
    country: "Example Country",
    zip: 5791
  }
};
```

Traditional access:

```js
const name = student.name;
const city = student.address.city;
```

Destructuring:

```js
const { name, age } = student;

console.log(name);
console.log(age);
```

JavaScript automatically:

- `name` variable তৈরি করে
- `student.name` assign করে
- `age` variable তৈরি করে
- `student.age` assign করে

### কেন useful?

- কম code
- cleaner syntax
- repeated dot access কমে
- function parameter clean হয়
- API response handling সহজ হয়

### Basic syntax

```js
const { property1, property2 } = object;
```

# 28. Destructuring-এর গুরুত্বপূর্ণ Use Cases

Instructor প্রায় ছয়-সাতটি practical use case দেখিয়েছেন।

## 28.1 Basic Multiple Property Extraction

```js
const { name, age } = student;
```

Equivalent:

```js
const name = student.name;
const age = student.age;
```

---

## 28.2 Default Value দিয়ে নতুন Variable

Object-এ property না থাকলেও destructuring-এর সময় default value দেওয়া যায়।

```js
const { name, age, meal = "Bread" } = student;

console.log(meal); // Bread
```

Object-এ `meal` থাকলে object-এর value ব্যবহৃত হবে; না থাকলে default value।

Equivalent long form conceptually:

```js
const meal = student.meal ? student.meal : "Bread";
```

আরও accurate modern equivalent:

```js
const meal = student.meal === undefined ? "Bread" : student.meal;
```

### Important rule

Default value সাধারণত property value `undefined` হলে apply হয়।

---

## 28.3 Dynamic/Computed Default Value

Destructuring-এ আগে পাওয়া variable ব্যবহার করে আরেকটি default/computed variable তৈরি করা যায়।

```js
const {
  subjects,
  numberOfSubjects = subjects.length
} = student;

console.log(numberOfSubjects); // 3
```

এখানে `subjects` array destructure হয়েছে এবং তার `.length` থেকে `numberOfSubjects` তৈরি হয়েছে।

### Common mistake

Order গুরুত্বপূর্ণ। `subjects` আগে available থাকতে হবে।

---

## 28.4 Alias / Rename

Property-এর নাম এক, কিন্তু local variable-এর নাম অন্য করতে alias ব্যবহার করা যায়।

```js
const {
  standard: studentStandard
} = student;

console.log(studentStandard); // 3
```

এখানে:

- property name: `standard`
- local variable name: `studentStandard`

Instructor-এর example-এ `std` property-কে `standard` alias দেওয়া হয়েছিল।

```js
const { std: standard } = student;
```

এক্ষেত্রে `std` নামে variable তৈরি হয় না; variable হয় `standard`।

```js
console.log(standard); // works
console.log(std);      // ReferenceError
```

### কেন alias দরকার?

Instructor React, Angular, REST, GraphQL, webhook এবং async response-এর উদাহরণ দেন। একাধিক source থেকে একই নামের variable এলে conflict হতে পারে। যেমন তিন জায়গা থেকে `loading` এলে:

```js
const { loading: graphQLLoading } = graphQLResponse;
const { loading: restLoading } = restResponse;
const { loading: webhookLoading } = webhookResponse;
```

### মনে রাখার নিয়ম

> `propertyName: aliasName`

---

## 28.5 Nested Object Destructuring

Nested property সরাসরি extract করা যায়।

Traditional:

```js
const address = student.address;
const zip = address.zip;
```

Destructuring:

```js
const {
  address: { zip }
} = student;

console.log(zip); // 5791
```

Syntax বুঝুন:

```js
const {
  topLevelProperty: { nestedProperty }
} = object;
```

আরও গভীরে:

```js
const data = {
  user: {
    profile: {
      city: "Dhaka"
    }
  }
};

const {
  user: {
    profile: { city }
  }
} = data;
```

### Common mistake

উপরের destructuring-এ `address` variable automatically তৈরি হয় না; শুধু `zip` তৈরি হয়।

---

## 28.6 Function Parameter-এ Destructuring

Function-এর পুরো object parameter নিয়ে ভেতরে repeated query করার বদলে parameter level-এ destructure করা যায়।

Traditional:

```js
function sendEmail(student) {
  console.log(`Sent an email to ${student.parents.email}`);
}

sendEmail(student);
```

Destructured parameter:

```js
function sendEmail({ parents: { email } }) {
  console.log(`Sent an email to ${email}`);
}

sendEmail(student);
```

### সুবিধা

- Function কোন data ব্যবহার করছে তা signature দেখেই বোঝা যায়
- repeated `student.parents.email` লিখতে হয় না
- cleaner code

### Common mistake

Nested property missing হলে destructuring error দিতে পারে। Optional default object ব্যবহার করা যায়:

```js
function sendEmail({ parents: { email } = {} }) {
  console.log(email);
}
```

Transcript-এর মূল lesson-এ basic nested parameter destructuring দেখানো হয়েছে।

---

## 28.7 Function Return Value Destructure করা

Function object return করলে call-এর result সরাসরি destructure করা যায়।

```js
const getStudent = () => student;
```

Traditional:

```js
const anotherStudent = getStudent();
const anotherName = anotherStudent.name;
const anotherSubjects = anotherStudent.subjects;
```

Destructuring with aliases:

```js
const {
  name: anotherName,
  subjects: anotherSubjects
} = getStudent();

console.log(anotherName);
console.log(anotherSubjects);
```

এতে intermediate object variable এবং extra lines কমে যায়।

---

## 28.8 Loop-এর মধ্যে Destructuring

একটি array-তে multiple student object থাকলে `for...of` loop-এর variable অংশে destructuring করা যায়।

```js
const students = [
  { name: "William", grade: "A" },
  { name: "Tom", grade: "A+" },
  { name: "Bob", grade: "B" }
];

for (const { name, grade } of students) {
  console.log(name, grade);
}
```

Output:

```text
William A
Tom A+
Bob B
```

### Instructor-এর background note

Array নিয়ে future “array master course”-এ বিস্তারিত আসবে। আপাতত array-কে elements-এর collection হিসেবে ধরতে বলা হয়েছে। Element primitive বা non-primitive হতে পারে; এখানে প্রতিটি element object।

### `for...in` বনাম `for...of` এখানে

- `for...in` object-এর key iterate করেছে
- `for...of` array-এর object elements iterate করেছে

# 29. Optional Chaining

Optional chaining ES2020-এ introduced একটি syntax, যা nested property safely access করতে সাহায্য করে—বিশেষ করে intermediate value `null` বা `undefined` হতে পারে যখন।

ধরা যাক:

```js
const employee = {
  salary: 100
};
```

এটি safe:

```js
console.log(employee.salary); // 100
console.log(employee.department); // undefined
```

কিন্তু এটি error দেবে:

```js
console.log(employee.department.name);
```

কারণ `employee.department` হলো `undefined`, আর `undefined`-এর ওপর `.name` পড়া যায় না। Error conceptually:

```text
Cannot read properties of undefined (reading 'name')
```

## Traditional condition

```js
const name = employee.department
  ? employee.department.name
  : undefined;
```

## Optional chaining

```js
const name = employee.department?.name;
console.log(name); // undefined
```

`?.` হলো optional chaining operator।

### কীভাবে কাজ করে?

- `employee.department` যদি `null` বা `undefined` হয়, chain stop করে `undefined` return করে
- application runtime error দিয়ে crash করে না
- value থাকলে পরের property access করে

### Multiple levels

```js
const city = employee?.department?.address?.city;
```

### Instructor-এর গুরুত্বপূর্ণ caveat

সবসময় optional chaining ব্যবহার করতেই হবে এমন নয়। যদি কোনো value অবশ্যই থাকার কথা এবং না থাকলে error report হওয়া উচিত, তখন normal access ব্যবহার করা যেতে পারে।

অর্থাৎ:

- Missing data gracefully handle করতে চাইলে → `?.`
- Missing data programming error হিসেবে ধরতে চাইলে → normal access এবং proper error handling

### Common mistake

Optional chaining missing property create করে না; শুধু safe access করে।

```js
employee.department?.name = "HR";
```

Optional chain assignment target হিসেবে ব্যবহার করা যায় না।

### মনে রাখার নিয়ম

> `?.` মানে: “এই অংশটি থাকলে পরের property দেখো; না থাকলে error না দিয়ে `undefined` দাও।”

# 30. Common Mistakes

## Mistake 1: Dynamic key dot notation দিয়ে access

ভুল:

```js
const key = "age";
user.key;
```

সঠিক:

```js
user[key];
```

---

## Mistake 2: Space-যুক্ত key dot notation দিয়ে access

ভুল:

```js
user.is admin;
```

সঠিক:

```js
user["is admin"];
```

---

## Mistake 3: Constructor function-এ `new` বাদ দেওয়া

ভুল:

```js
const car = Car("BMW", "X1");
```

সঠিক:

```js
const car = new Car("BMW", "X1");
```

---

## Mistake 4: Object content একই বলে equality true ভাবা

```js
{ name: "Mango" } === { name: "Mango" } // false
```

কারণ references আলাদা।

---

## Mistake 5: `Object.assign()`-কে deep clone ভাবা

Nested object shared reference হতে পারে। Independent nested copy দরকার হলে:

```js
structuredClone(object)
```

---

## Mistake 6: Property existence value দিয়ে check করা

Problematic:

```js
if (!profile.salary) {}
```

Better:

```js
"salary" in profile
```

অথবা own property check:

```js
Object.hasOwn(profile, "salary")
```

---

## Mistake 7: Method call-এ parentheses বাদ দেওয়া

```js
profile.message;   // function reference
profile.message(); // function execution
```

---

## Mistake 8: Alias দেওয়ার পর original property name variable হিসেবে ব্যবহার

```js
const { std: standard } = student;

console.log(standard); // works
console.log(std);      // ReferenceError
```

---

## Mistake 9: `freeze()` এবং `seal()` একই ভাবা

- Freeze: modify/add/delete—কোনোটাই নয়
- Seal: existing modify করা যায়; add/delete নয়

---

## Mistake 10: Missing nested property normal chain দিয়ে access

ভুল:

```js
employee.department.name;
```

Safe:

```js
employee.department?.name;
```

# 31. Interview Questions

## Basic Questions

### 1. JavaScript object কী?

Object হলো key-value pair-এর keyed collection, যেখানে unique key ব্যবহার করে value store ও retrieve করা হয়।

### 2. Object literal syntax কী?

Curly braces ব্যবহার করে সরাসরি object তৈরি করাকে object literal syntax বলে।

```js
const user = { name: "Tapas" };
```

### 3. Dot notation এবং bracket notation-এর পার্থক্য কী?

- Dot notation সাধারণ, valid identifier key-এর জন্য
- Bracket notation special-character key এবং dynamic key-এর জন্য

### 4. Dynamic property access কীভাবে করা হয়?

```js
const key = "age";
user[key];
```

### 5. Object-এর ভেতরের function-কে কী বলা হয়?

Method।

### 6. Nested object কী?

যখন একটি object-এর property value আরেকটি object হয়।

### 7. `in` operator কী করে?

কোনো property object বা তার prototype chain-এ আছে কি না check করে।

### 8. `Object.keys()` কী return করে?

Object-এর enumerable own keys-এর array।

---

## Intermediate Questions

### 9. কেন দুইটি identical-looking object equal নয়?

কারণ object comparison value নয়, reference দিয়ে হয়।

```js
{} === {} // false
```

### 10. Constructor function কী?

এক ধরনের function যা blueprint হিসেবে কাজ করে এবং `new` ব্যবহার করে object instance তৈরি করে। Convention অনুযায়ী নাম capital letter দিয়ে শুরু হয়।

### 11. Factory function কী?

Regular function যা object return করে এবং call করতে `new` লাগে না।

### 12. Constructor এবং factory function-এর পার্থক্য কী?

Constructor সাধারণত `new` ও `this` ব্যবহার করে; factory function সরাসরি object return করে।

### 13. `Object.assign()` কী করে?

Source object-এর own properties target object-এ copy করে। এটি shallow copy করে।

### 14. Shallow copy কী?

Top-level properties copy হয়, কিন্তু nested object-এর references shared থাকতে পারে।

### 15. Deep copy কী?

Nested object-সহ independent copy তৈরি করা। Transcript-এ `structuredClone()` দেখানো হয়েছে।

### 16. `Object.entries()` এবং `Object.fromEntries()` কীভাবে বিপরীত?

- `Object.entries()` object → key-value pair array
- `Object.fromEntries()` key-value pair array → object

### 17. `Object.freeze()` ও `Object.seal()`-এর পার্থক্য কী?

Freeze existing value পরিবর্তনও আটকায়; seal existing value modify করতে দেয়, কিন্তু add/delete আটকায়।

### 18. `Object.hasOwn()` কী করে?

Object-এর own property আছে কি না boolean return করে।

---

## Destructuring Questions

### 19. Object destructuring কী?

Object property values concise syntax-এ আলাদা variable-এ extract করা।

```js
const { name, age } = user;
```

### 20. Destructuring-এ default value কীভাবে দিতে হয়?

```js
const { meal = "Bread" } = student;
```

### 21. Alias কীভাবে দিতে হয়?

```js
const { standard: studentStandard } = student;
```

### 22. Nested destructuring কীভাবে হয়?

```js
const { address: { city } } = student;
```

### 23. Function parameter-এ destructuring কেন useful?

Function কোন properties ব্যবহার করছে তা পরিষ্কার হয় এবং repeated access কমে।

### 24. Function return value কীভাবে destructure করা যায়?

```js
const { name, subjects } = getStudent();
```

### 25. Loop-এ object destructuring-এর example দাও।

```js
for (const { name, grade } of students) {
  console.log(name, grade);

```

## Optional Chaining Questions

### 26. Optional chaining কী?

`?.` operator, যা `null`/`undefined` intermediate value থাকলে runtime error না দিয়ে `undefined` return করে।

### 27. Optional chaining কখন ব্যবহার করা উচিত?

Nested API data বা optional properties safely access করতে।

### 28. Optional chaining সব error hide করার জন্য ব্যবহার করা উচিত কি?

না। Value অবশ্যই থাকার কথা হলে missing value error report করা দরকার হতে পারে।
