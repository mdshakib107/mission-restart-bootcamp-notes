আমি একটি tutorial video-এর transcript upload করছি।

তুমি transcript-টিকে summary, short notes বা condensed explanation-এ রূপান্তর করবে না। বরং Instructor-এর পুরো বক্তব্যের ধারাবাহিকতা বজায় রেখে বইয়ের একটি পূর্ণাঙ্গ অধ্যায়ের মতো বাংলায় অনুবাদ ও সাজিয়ে দেবে।

নির্দেশনা:

1. Transcript-এর শুরু থেকে শেষ পর্যন্ত lecture flow একই রাখবে।
2. Instructor যে analogy, গল্প, উদাহরণ, background explanation, আগের lesson-এর reference, পরের lesson-এর reference, motivational কথা, warning, recap, interview question এবং task বলেছেন—সব অন্তর্ভুক্ত করবে।
3. কোনো গুরুত্বপূর্ণ বক্তব্য বাদ দেবে না এবং অপ্রয়োজনীয়ভাবে সংক্ষিপ্ত করবে না।
4. ভাষা হবে স্বাভাবিক, সহজ, পরিষ্কার ও বইয়ের মতো বাংলা।
5. Programming terms যেমন `variable`, `function`, `object`, `array`, `scope`, `closure`, `execution context`, `this`, `async`, `defer`, `DOM`, `primitive`, `non-primitive` ইত্যাদি English-এই রাখবে।
6. সব code হুবহু বা transcript অনুযায়ী proper fenced code block-এ লিখবে।
7. Code-এর আগে ও পরে Instructor যেভাবে ব্যাখ্যা করেছেন, সেই reasoning step-by-step বাংলায় তুলে ধরবে।
8. Instructor কোনো definition দিলে অর্থ পরিবর্তন না করে সেটি বাংলায় অনুবাদ করবে।
9. Instructor একই rule বারবার বুঝিয়ে বললে তা সম্পূর্ণ বাদ দেবে না; বইয়ের ভাষায় পুনরাবৃত্তিটি স্বাভাবিকভাবে রাখবে, কারণ এটি teaching flow-এর অংশ।
10. Transcript-এ কোনো analogy থাকলে একই analogy বজায় রাখবে; নিজের নতুন analogy, গল্প বা উদাহরণ যোগ করবে না।
11. Instructor কোনো প্রশ্ন করে পরে উত্তর দিলে সেই question-answer flow বজায় রাখবে।
12. যেখানে output predict করা হয়েছে, সেখানে expected output এবং কেন সেই output হবে—দুটিই লিখবে।
13. Common mistake, tricky case এবং interview problem আলাদা subsection-এ পরিষ্কার করবে, তবে transcript-এর বাইরে নতুন concept যোগ করবে না।
14. Lecture-এর recap অংশ আলাদাভাবে রাখবে।
15. Transcript-এ assignment বা task থাকলে শেষে `Assignment / Task` section বানাবে।
16. শেষে transcript অনুযায়ী একটি concise `Final Recap` যোগ করবে, তবে মূল content সংক্ষিপ্ত করবে না।
17. Output হবে Markdown `.md` format-এ।
18. Structured heading ও subheading ব্যবহার করবে, যেন এটি tutorial transcript নয়, প্রকাশযোগ্য বাংলা programming book chapter মনে হয়।
19. Timestamp বাদ দিতে পারো, কিন্তু timestamp অনুযায়ী বক্তব্যের sequence পরিবর্তন করবে না।
20. Instructor-এর বক্তব্যকে third-person summary হিসেবে লিখবে না। যথাসম্ভব direct teaching voice বজায় রাখবে, যেমন:

* “এখন আমরা দেখব…”
* “চলো example দিয়ে বুঝি…”
* “এই rule-টি মনে রাখো…”

21. অসম্পূর্ণ transcript sentence থাকলে context অনুযায়ী কেবল প্রয়োজনীয় grammar ঠিক করবে; নতুন তথ্য তৈরি করবে না।
22. কোনো factual বা technical inconsistency চোখে পড়লেও মূল translation-এ Instructor-এর বক্তব্য বজায় রাখবে। প্রয়োজন হলে আলাদা `Technical Note`-এ সংক্ষেপে সতর্কতা জানাবে।
23. Output beginner-friendly হবে, যেন video না দেখেও পুরো lesson বোঝা যায়।
24. পুরো কাজ শেষে একটি downloadable `.md` file তৈরি করে দেবে।

সবচেয়ে গুরুত্বপূর্ণ নির্দেশনা:

**এটি summary বা study notes নয়। এটি transcript-এর বক্তব্য, teaching flow, examples, analogies, code walkthrough, recap, interview questions এবং tasks বজায় রেখে বইয়ের অধ্যায়ের মতো পূর্ণ বাংলা অনুবাদ।**
