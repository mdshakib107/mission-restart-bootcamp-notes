এখানে দেখানো হয়েছে JavaScript কীভাবে **Call Stack** দিয়ে কোড চালায়।

এই কোডটা ভাবুন:

```js
function one() {
  two();
}

function two() {
  three();
}

function three() {
  console.log("Hi");
}

one();
```

প্রথমে JavaScript **Global Execution Context** তৈরি করে। এরপর `one()` call হলে `one()` stack-এর উপরে যায়। `one()` এর ভিতরে `two()` call হয়, তাই `two()` উপরে যায়। তারপর `two()` এর ভিতরে `three()` call হয়, তাই `three()` সবার উপরে যায়।

Stack-এর নিয়ম হলো **LIFO — Last In, First Out**। অর্থাৎ যে function সবার শেষে ঢোকে, সে আগে বের হয়।

তাই execution শেষ হলে:

`three()` আগে বের হবে → তারপর `two()` → তারপর `one()` → শেষে শুধু Global Context থাকবে।

সহজভাবে:

```text
Push order:
Global → one() → two() → three()

Pop order:
three() → two() → one()
```

মানে JavaScript একসাথে সব function চালায় না; সে একটার ভিতরে আরেকটা function call হলে সেগুলো stack-এ জমা রাখে, আর কাজ শেষ হলে উল্টো ক্রমে সরিয়ে দেয়।
