# 2. What is Control Flow?

JavaScript normally line by line execute হয়।

Example:

```text
Line 1
Line 2
Line 3
Line 4
```

কিন্তু programming এ অনেক সময় condition অনুযায়ী:

- কিছু line skip করতে হয়
- অন্য জায়গায় jump করতে হয়
- different execution path নিতে হয়

এই execution control করাকে বলে:

```text
Control Flow
```

# 19. Grading System Problem

Rules:

| Score     | Grade |
| --------- | ----- |
| >= 90     | A     |
| >= 80     | B     |
| >= 70     | C     |
| otherwise | Fail  |

---

# 20. Incorrect Way Using Independent if

```js
if (score >= 90) {
  console.log("Grade A");
}

if (score >= 80) {
  console.log("Grade B");
}

if (score >= 70) {
  console.log("Grade C");
}
```

এটি কিছু ক্ষেত্রে problem করতে পারে।

---

# 21. Correct Way Using else if

```js
if (score >= 90) {
  console.log("Grade A");
} else if (score >= 80) {
  console.log("Grade B");
} else if (score >= 70) {
  console.log("Grade C");
} else {
  console.log("Fail");
}
```

---

provide করে।

# 30. Main Learning

| Independent if           | else if               |
| ------------------------ | --------------------- |
| সব condition check হয়    | First match এ stop হয় |
| Multiple output possible | Single path execution |
| No early exit            | Early exit            |

# 37. switch Syntax


# 44. Day Detection Program

Problem:

| Number | Day     |
| ------ | ------- |
| 1      | Monday  |
| 2      | Tuesday |
| ...    | ...     |
| 7      | Sunday  |

---

# 45. switch-case Day Program

```js
let day = 5;

switch (day) {
  case 1:
    console.log("Monday");
    break;

  case 2:
    console.log("Tuesday");
    break;

  case 3:
    console.log("Wednesday");
    break;

  case 4:
    console.log("Thursday");
    break;

  case 5:
    console.log("Friday");
    break;

  default:
    console.log("Invalid day");
}
```

# 62. Main Learning Summary

| Topic        | Key Idea               |
| ------------ | ---------------------- |
| Control Flow | Execution path control |
| if           | Single condition       |
| else         | Alternative path       |
| else if      | Multiple conditions    |
| Nested if    | if inside if           |
| switch       | Fixed value branching  |
| case         | Match block            |
| break        | Stop execution         |
| default      | Fallback               |
| fall-through | No break behavior      |
| ternary      | Short if-else          |

---

# 63. Best Practices

✅ Always use braces

✅ Use:

```js
else if
```

for exclusive conditions

✅ Use switch-case for fixed values

✅ Use if-else for complex logic

❌ Avoid deep nested ternary

❌ Avoid missing break accidentally
