## Learning Mindset: JavaScript শেখার সঠিক পদ্ধতি

Programming language শেখা অনেকটা spoken language শেখার মতো। প্রথম দিনেই সব concept মুখস্থ করার চেষ্টা করলে মাথা overload হয়ে যায়। বরং বারবার code দেখা, নিজে type করা, error পড়া, এবং practice করা—এই process-টাই সবচেয়ে গুরুত্বপূর্ণ।

### সহজভাবে বোঝো

যেমন একজন মানুষ নিজের native language grammar বই পড়ে নয়, বরং শুনে-শুনে, বলে-বলে শেখে; programming language-ও তেমন। তুমি যত বেশি code দেখবে, লিখবে, ভুল করবে, error পড়বে, তত বেশি flow তৈরি হবে।

### মনে রাখার নিয়ম

> JavaScript শেখা মানে শুধু syntax মুখস্থ করা নয়; code কীভাবে ভাবতে হয় সেটা শেখা।

### Common mistake

অনেক beginner ভাবে:  
“আমি `if`, `loop`, `function` লিখতে পারি, কিন্তু ভিতরের logic লিখতে পারি না।”

এর কারণ হলো concept বুঝে practice না করা। তাই প্রতিটি concept-এর পর ছোট ছোট example লিখে practice করতে হবে।

## Prerequisite Setup: HTML + JavaScript file

JavaScript browser-এ চালানোর জন্য সাধারণত একটি HTML file এবং একটি JavaScript file লাগে।

### Basic project structure

```text
project-folder/
│
├── index.html
└── script.js
```

### Step-by-step setup

1. একটি folder তৈরি করো।
2. folder-টি VS Code-এ open করো।
3. `index.html` file তৈরি করো।
4. `script.js` file তৈরি করো।
5. `index.html` file-এর মধ্যে HTML boilerplate বসাও।
6. `script.js` file-কে HTML-এর সাথে connect করো।

### HTML boilerplate example

```html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8" />
  <meta name="viewport" content="width=device-width, initial-scale=1.0" />
  <title>JavaScript Practice</title>
</head>
<body>
  <h1>JavaScript Learning</h1>

  <script src="script.js"></script>
</body>
</html>
```

### `script` tag কী করে?

```html
<script src="script.js"></script>
```

এই line browser-কে বলে:

> এই HTML page-এর সাথে `script.js` file যুক্ত করো এবং JavaScript code চালাও।

### Live Server ব্যবহার

VS Code-এ Live Server extension থাকলে `index.html` file-এ right click করে **Open with Live Server** করলে browser-এ page open হবে। JavaScript-এর output সাধারণত browser console-এ দেখা যায়।

Console open করার shortcut:

| Browser | Shortcut |
|---|---|
| Chrome/Edge Windows | `Ctrl + Shift + J` |
| Chrome/Edge Mac | `Cmd + Option + J` |
| General DevTools | `F12` |

### Common mistake

অনেকে `script.js` বানায়, কিন্তু HTML-এ connect করতে ভুলে যায়। ফলে JavaScript code লিখলেও browser-এ কিছু কাজ করে না।

### মনে রাখার নিয়ম

> JavaScript file লিখলেই হবে না; HTML-এর সাথে connect করাও জরুরি।

## JavaScript-এর Background/History

JavaScript-এর history বুঝলে language design-এর কিছু weird behavior বুঝতে সহজ হয়।

### মূল story

- Brendan Eich Netscape browser-এর জন্য একটি scripting language বানিয়েছিলেন।
- প্রথম নাম ছিল **Mocha**।
- পরে Java language popular হওয়ায় marketing কারণে language-এর নাম JavaScript রাখা হয়।
- JavaScript এবং Java এক জিনিস নয়।
- JavaScript Java-এর advanced version নয়।
- পরে browser vendors-দের মধ্যে standardization-এর প্রয়োজন হয়।
- ECMAScript standard JavaScript language-এর specification maintain করে।
- `var` পুরোনো JavaScript/ES5 style-এর অংশ।
- `let` এবং `const` ES6-এর modern features।

### Java vs JavaScript

| বিষয় | Java | JavaScript |
|---|---|---|
| Nature | General-purpose programming language | Web scripting/programming language |
| Syntax similarity | কিছুটা C-style | কিছুটা C-style |
| Relationship | আলাদা language | Java-এর advanced version নয় |
| Main use | Backend, Android, enterprise apps | Web, frontend, backend via Node.js |

### ES5 vs ES6

| Version | সাধারণ ধারণা |
|---|---|
| ES5 | পুরোনো JavaScript style; `var` বেশি ব্যবহৃত |
| ES6 | modern JavaScript; `let`, `const`, arrow function ইত্যাদি এসেছে |

### মনে রাখার নিয়ম

> JavaScript-এর নামের মধ্যে Java থাকলেও JavaScript আর Java আলাদা language।

## Words vs Keywords

JavaScript-এ সব text একরকম নয়। কিছু text সাধারণ word, আর কিছু text keyword।

### Word কী?

যে word-এর JavaScript language-এর কাছে predefined কোনো special meaning নেই, সেটি সাধারণ word।

Example:

```js
harsh
shyam
hello
myName
```

এগুলো নিজেরা কোনো কাজ করে না, যতক্ষণ না তুমি এগুলোকে variable/function name হিসেবে ব্যবহার করছ।

### Keyword কী?

Keyword হলো এমন special word যার JavaScript-এ আগে থেকেই নির্দিষ্ট meaning আছে।

Example:

```js
var
let
const
if
else
function
for
```

JavaScript এই words দেখলে বুঝে যায় কী করতে হবে।

### Table: Word vs Keyword

| বিষয় | Word | Keyword |
|---|---|---|
| Meaning | JavaScript-এর কাছে নিজে নিজে special meaning নেই | JavaScript-এর কাছে predefined meaning আছে |
| Example | `harsh`, `name`, `score` | `let`, `const`, `if`, `function` |
| Use | identifier হিসেবে ব্যবহার হতে পারে | language syntax-এর অংশ |
| নিজের মতো বানানো যায়? | হ্যাঁ | না, fixed |

### Important example

```js
let score = 10;
```

এখানে:

| অংশ | কী |
|---|---|
| `let` | keyword |
| `score` | variable name / identifier |
| `10` | value |

### Common mistake

Beginner-রা অনেক সময় keyword-কে variable name হিসেবে ব্যবহার করতে চায়।

Wrong:

```js
let let = 10;
```

Correct:

```js
let score = 10;
```

### মনে রাখার নিয়ম

> JavaScript যেসব word দিয়ে কিছু কাজ করায়, সেগুলো keyword।

## Variables: কেন দরকার?

Variable হলো data store করার container।

Program চলাকালীন কিছু data মনে রাখতে হয়। সেই data রাখার জন্য variable ব্যবহার করা হয়।

### Real-life analogy: Game score

Game-এ character-এর score প্রথমে `0` হতে পারে। পরে score বাড়তে পারে:

```js
let score = 0;

score = 10;
score = 50;
```

এখানে `score` variable data store করছে এবং পরে change হচ্ছে।

### Real-life analogy: ATM

ATM machine-এ card insert করলে machine-কে মনে রাখতে হয়:

- user কে?
- কোন account active?
- কত টাকা withdraw করতে চায়?

Program যদি active user-কে মনে না রাখে, তাহলে সঠিক account থেকে টাকা withdraw করা সম্ভব নয়।

Programming-এ এই “মনে রাখা” কাজটি variable দিয়ে করা হয়।

### Variable কেন useful?

ধরো একটি complex calculation আছে:

```js
let result = (23 * 12 - 89) % 45 * 122;
```

এই calculation-এর output পরে code-এর অন্য জায়গায় দরকার হতে পারে। বারবার calculation না লিখে `result` variable ব্যবহার করা যায়।

```js
console.log(result);
```

### Important concept

> Variable program-এর life cycle-এর মধ্যে data ধরে রাখে, যাতে future line-এ সেই data reuse করা যায়।

### Common mistake

অনেকে ভাবে variable শুধু simple value রাখার জন্য। আসলে variable যেকোনো useful data store করার জন্য:

- name
- score
- price
- discount
- calculation result
- user status
- form input
- game state

### মনে রাখার নিয়ম

> Future-এ কোনো value দরকার হবে মনে হলে সেটিকে variable-এ রাখো।

## `var`, `let`, `const`: মূল ধারণা

JavaScript-এ variable তৈরি করার জন্য তিনটি keyword commonly দেখা যায়:

```js
var
let
const
```

### High-level rule

| Keyword | ব্যবহার |
|---|---|
| `var` | পুরোনো style; avoid করা ভালো |
| `let` | value change হবে এমন variable |
| `const` | value change হবে না এমন constant |

### `let` example

```js
let score = 0;

score = 10;
score = 20;
```

Score game চলাকালীন change হতে পারে, তাই `let` ভালো choice।

### `const` example

```js
const pi = 3.14;
```

`pi`-এর value normally change হওয়ার কথা নয়, তাই `const` ব্যবহার করা ভালো।

### `var` example

```js
var age = 20;
```

এটি কাজ করে, কিন্তু modern JavaScript-এ `var` avoid করা হয়, কারণ এর কিছু confusing behavior আছে।

### মনে রাখার নিয়ম

> Default choice: `const`। value পরে বদলাতে হলে `let`। `var` avoid।

## Variable Declaration এবং Initialization

এই section খুব important, কারণ পরের topic `reassignment`, `redeclaration`, এবং `TDZ` বুঝতে এগুলো লাগবে।

### Declaration কী?

Variable তৈরি করার কাজকে declaration বলা হয়।

```js
let age;
```

এখানে `age` নামে variable declare করা হলো, কিন্তু value দেওয়া হয়নি।

### Initialization কী?

Variable-কে প্রথম value দেওয়ার কাজকে initialization বলা হয়।

```js
let age = 20;
```

এখানে দুইটি কাজ একসাথে হয়েছে:

1. `age` variable declare হয়েছে।
2. `age`-এর প্রথম value `20` দেওয়া হয়েছে।

### Declaration vs Initialization table

| Code | Declaration হয়েছে? | Initialization হয়েছে? |
|---|---:|---:|
| `let age;` | হ্যাঁ | না |
| `let age = 20;` | হ্যাঁ | হ্যাঁ |
| `age = 21;` | না | না; এটি reassignment |

### Important distinction

`initialization` মানে শুধু value দেওয়া নয়।  
`initialization` মানে variable-কে **প্রথম value** দেওয়া।

পরবর্তীতে value change করলে সেটি initialization নয়, সেটি reassignment।

### Example

```js
let score;       // declaration
score = 0;       // first value দেওয়া হলো; practical sense-এ initialization
score = 10;      // reassignment
```

আরেকভাবে:

```js
let score = 0;   // declaration + initialization
score = 10;      // reassignment
```

### Common mistake

অনেকে বলে:

> value দেওয়া মানেই initialization।

এটা সবসময় ঠিক নয়। প্রথম value দেওয়া initialization; পরে value change করা reassignment।

### মনে রাখার নিয়ম

> Declare = variable বানানো। Initialize = প্রথম value দেওয়া। Reassign = পরে নতুন value দেওয়া।

## Variable বানানোর বিভিন্ন পদ্ধতি

### `var` দিয়ে

```js
var a;
var b = 12;
```

### `let` দিয়ে

```js
let a;
let b = 12;
```

### `const` দিয়ে

```js
const pi = 3.14;
```

### `const` declare করে value না দেওয়া যায়?

না।

Wrong:

```js
const pi;
```

এতে error হবে, কারণ `const` variable বানানোর সময়ই value দিতে হয়।

Correct:

```js
const pi = 3.14;
```

### Keyword ছাড়া variable বানানো

JavaScript-এ technically এটা অনেক সময় কাজ করতে পারে:

```js
name = "Rahim";
```

কিন্তু এটি খুব bad practice। Modern code-এ এমন লেখা উচিত নয়।

Correct:

```js
let name = "Rahim";
```

### কেন keyword ছাড়া variable বানানো খারাপ?

কারণ এতে accidental global variable তৈরি হতে পারে এবং বড় project-এ bug খুঁজে পাওয়া কঠিন হয়ে যায়।

### মনে রাখার নিয়ম

> Variable বানালে অবশ্যই `let` বা `const` ব্যবহার করো।

## `var`-এর Problems

Transcript অনুযায়ী `var`-এর তিনটি major problem আলোচনা করা হয়েছে।

1. `var` variable `window` object-এ attach হতে পারে।
2. `var` function scoped।
3. `var` দিয়ে same name redeclare করলে error আসে না।

---

### Problem 1: `var` window object-এ add হয়

Browser environment-এ global `var` দিয়ে variable বানালে সেটি `window` object-এর property হিসেবে পাওয়া যেতে পারে।

Example:

```js
var a = 12;

console.log(window.a); // 12
```

কিন্তু `let` দিয়ে করলে এমন হয় না:

```js
let b = 20;

console.log(window.b); // undefined
```

### এটা problem কেন?

`window` browser-এর global object। এতে অনেক built-in property/method থাকে। যদি তুমি অযথা global `var` দিয়ে variable বানাও, global space polluted হতে পারে এবং name conflict হতে পারে।

### Common mistake

```js
var name = "Rahim";
```

`name` browser-এর existing global property-এর সাথে conflict করতে পারে।

### মনে রাখার নিয়ম

> Global `var` browser-এর `window` space নোংরা করতে পারে; তাই `let`/`const` ব্যবহার করো।

---

### Problem 2: `var` function scoped

`var` block scope respect করে না; এটি function scope follow করে।

Example:

```js
if (true) {
  var x = 10;
}

console.log(x); // 10
```

অনেক beginner আশা করে `x` শুধু `if` block-এর ভিতরে থাকবে। কিন্তু `var` block respect করে না, তাই বাইরে থেকেও access করা যায়।

### Function scope example

```js
function test() {
  var message = "Hello";

  console.log(message); // works
}

console.log(message); // error
```

এখানে `message` function-এর ভিতরে আছে, তাই function-এর বাইরে access করা যায় না।

### Important point

`var` nearest function-এর মধ্যে available থাকে। যদি কোনো function না থাকে, তাহলে global scope-এ চলে যায়।

### Common mistake

```js
if (true) {
  var count = 5;
}

console.log(count); // works, কিন্তু অনেকের expectation থাকে error হবে
```

### মনে রাখার নিয়ম

> `var` curly braces নয়, function boundary দেখে।

---

### Problem 3: `var` redeclaration allow করে

```js
var a = 12;
var a = 13;

console.log(a); // 13
```

JavaScript error দেয় না।

এটা dangerous, কারণ তুমি accidentally same variable name আবার declare করলে previous value replace হয়ে যেতে পারে।

### কেন error না আসা problem?

Error সবসময় খারাপ নয়। অনেক সময় early error developer-কে ভুল ধরিয়ে দেয়। `var` ভুলের জায়গায় error না দিয়ে code চালিয়ে যেতে দেয়, ফলে bug পরে বড় problem হয়ে দাঁড়ায়।

### `let` ব্যবহার করলে

```js
let a = 12;
let a = 13; // SyntaxError: Identifier 'a' has already been declared
```

এখানে JavaScript তোমাকে থামিয়ে দেয়।

### মনে রাখার নিয়ম

> Early error ভালো; silent bug খারাপ।

---

## `let`: কেন বেশি safe?

`let` modern JavaScript-এর recommended variable declaration keyword।

### `let`-এর features

| Feature | `let` behavior |
|---|---|
| Reassignment | allowed |
| Redeclaration in same scope | not allowed |
| Scope | block scoped |
| Window object-এ add | না |
| TDZ | আছে |

### Example: value change করা যায়

```js
let score = 0;

score = 10;
score = 20;

console.log(score); // 20
```

### Example: redeclare করা যায় না

```js
let score = 0;
let score = 10; // error
```

### Example: block scope respect করে

```js
if (true) {
  let message = "Inside block";
  console.log(message); // works
}

console.log(message); // error
```

### কখন `let` ব্যবহার করব?

যখন variable-এর value পরে change হতে পারে।

Examples:

```js
let score = 0;
let userInput = "";
let currentPage = 1;
let isLoggedIn = false;
```

### Common mistake

যেখানে value change হবে না, সেখানে `let` ব্যবহার করা।

```js
let pi = 3.14;
```

Better:

```js
const pi = 3.14;
```

### মনে রাখার নিয়ম

> Value বদলাবে? `let`। বদলাবে না? `const`।

---

## `const`: কখন ব্যবহার করব?

`const` ব্যবহার করা হয় এমন value-এর জন্য যা change করা উচিত নয়।

### Example

```js
const discount = 12;
const pi = 3.14;
```

### `const` value change করতে গেলে

```js
const pi = 3.14;

pi = 3.14159; // TypeError
```

JavaScript error দেবে, কারণ `const` reassignment allow করে না।

### `const` declare করার সময় value দিতে হয়

Wrong:

```js
const discount;
discount = 12;
```

Correct:

```js
const discount = 12;
```

### কখন `const` use করবে?

- mathematical constant
- fixed discount
- fixed configuration
- এমন value যা program চলাকালীন বদলানোর কথা নয়

### Common mistake

`const` মানে value কখনো কোনোভাবে change হবে না—এটা object/array-এর ক্ষেত্রে future lesson-এ nuanced। তবে এই transcript অংশে primitive fixed value হিসেবে `const`-এর idea দেওয়া হয়েছে: variable binding reassign করা যাবে না।

### মনে রাখার নিয়ম

> `const` মানে এই নামের variable-এ নতুন value assign করা যাবে না।

---

## Scope: Global, Block, Function

Scope মানে কোনো variable কোথা থেকে access করা যাবে।

সহজভাবে:

> Variable-এর দায়রা বা সীমা হলো scope।

---

### Global Scope

যে variable কোনো function বা block-এর ভিতরে নয়, সেটি global scope-এ থাকে।

```js
let username = "Rahim";

console.log(username); // works
```

এই variable পুরো file/code-এর যেকোনো জায়গা থেকে access করা যেতে পারে, যদি scope boundary না থাকে।

### Common mistake

সব variable global বানিয়ে ফেলা।

```js
let score = 0;
let username = "Rahim";
let discount = 10;
```

Small practice file-এ সমস্যা না হলেও বড় application-এ too many global variables conflict তৈরি করতে পারে।

### মনে রাখার নিয়ম

> Global variable পুরো code থেকে দেখা যায়, তাই অযথা global variable বানিও না।

---

### Block Scope

Curly braces `{}` দিয়ে তৈরি অংশকে block বলা হয়।

```js
{
  let a = 10;
  console.log(a); // works
}

console.log(a); // error
```

`let` এবং `const` block scope respect করে।

আরেকটি example:

```js
if (true) {
  let message = "Hello";
  console.log(message); // works
}

console.log(message); // error
```

### Block কী?

যেখানে `{}` আছে, সেটি block হতে পারে।

Example:

```js
if (true) {
  // this is a block
}

for (let i = 0; i < 5; i++) {
  // this is a block
}

{
  // standalone block
}
```

### মনে রাখার নিয়ম

> `let` এবং `const` যে `{}`-এর ভিতরে তৈরি হয়, সাধারণত সেই `{}`-এর ভিতরেই access করা যায়।

---

### Function Scope

Function-এর ভিতরে তৈরি variable function-এর ভিতরেই accessible থাকে।

```js
function greet() {
  let message = "Hello";
  console.log(message); // works
}

greet();

console.log(message); // error
```

`var` মূলত function scoped।

```js
function test() {
  if (true) {
    var x = 10;
  }

  console.log(x); // works, because var is function scoped
}

test();
```

এখানে `x` `if` block-এর ভিতরে declare হলেও function-এর মধ্যে access করা যায়।

### `var` বনাম `let` in block

```js
function test() {
  if (true) {
    var a = 10;
    let b = 20;
  }

  console.log(a); // works
  console.log(b); // error
}
```

### Scope summary table

| Scope type | Meaning | Example variable | কোথায় access করা যাবে |
|---|---|---|---|
| Global scope | কোনো block/function-এর বাইরে | `let name = "Rahim"` | পুরো code-এ |
| Block scope | `{}` braces-এর ভিতরে | `let x = 10` | সেই block-এর ভিতরে |
| Function scope | function-এর ভিতরে | `var y = 20` | সেই function-এর ভিতরে |

### Common mistake

`var` দিয়ে block-এর ভিতরে variable বানিয়ে ভাবা যে সেটি block-এর বাইরে পাওয়া যাবে না।

Wrong expectation:

```js
if (true) {
  var count = 10;
}

console.log(count); // works, because var ignores block scope
```

### মনে রাখার নিয়ম

> `var` function দেখে; `let`/`const` braces দেখে।

---

## Reassignment vs Redeclaration

এই দুইটি term beginner-দের জন্য confusing হতে পারে।

---

### Reassignment কী?

একটি already declared variable-কে নতুন value দেওয়া।

```js
let age = 20;

age = 21; // reassignment
```

এখানে `age` আবার declare করা হয়নি; শুধু value update হয়েছে।

### `let` দিয়ে reassignment allowed

```js
let score = 0;
score = 10;

console.log(score); // 10
```

### `var` দিয়ে reassignment allowed

```js
var score = 0;
score = 10;

console.log(score); // 10
```

### `const` দিয়ে reassignment not allowed

```js
const score = 0;
score = 10; // error
```

---

### Redeclaration কী?

একই scope-এ same variable name আবার declaration keyword দিয়ে declare করা।

```js
var a = 12;
var a = 13; // redeclaration
```

### `var` দিয়ে redeclaration allowed

```js
var a = 12;
var a = 13;

console.log(a); // 13
```

### `let` দিয়ে same scope-এ redeclaration not allowed

```js
let a = 12;
let a = 13; // error
```

### `const` দিয়েও redeclaration not allowed

```js
const a = 12;
const a = 13; // error
```

### Reassignment vs Redeclaration table

| Concept | Meaning | Example | Allowed with `let`? | Allowed with `const`? |
|---|---|---|---:|---:|
| Reassignment | old variable-এ new value দেওয়া | `a = 20` | হ্যাঁ | না |
| Redeclaration | same name আবার keyword দিয়ে declare | `let a = 20` | না, same scope-এ | না, same scope-এ |

### Common mistake

```js
let a = 10;
let a = 20;
```

অনেকে ভাবে এটা value update। আসলে এটা redeclaration, তাই error।

Correct:

```js
let a = 10;
a = 20;
```

### মনে রাখার নিয়ম

> Keyword আবার লিখলে declaration/redeclaration। Keyword ছাড়া value দিলে reassignment।

---

## TDZ: Temporal Dead Zone

TDZ বা **Temporal Dead Zone** JavaScript-এর একটি important interview topic।

### TDZ-এর basic idea

`let` বা `const` variable declare করার আগে সেই variable access করার চেষ্টা করলে error আসে।

Example:

```js
console.log(a);

let a = 12;
```

Error:

```text
ReferenceError: Cannot access 'a' before initialization
```

### কেন “not defined” error নয়?

ধরো তুমি এমন variable access করলে যেটি code-এ কোথাও নেই:

```js
console.log(a);
```

Error:

```text
ReferenceError: a is not defined
```

এখানে JavaScript বলছে: `a` নামে কিছুই নেই।

কিন্তু যদি পরে `let a = 12;` থাকে:

```js
console.log(a);

let a = 12;
```

তখন JavaScript বলে:

```text
Cannot access 'a' before initialization
```

মানে JavaScript জানে `a` আছে, কিন্তু initialization-এর আগে access করতে দিচ্ছে না।

### TDZ কী?

Variable declaration-এর আগে থেকে variable initialization পর্যন্ত যে zone-এ variable technically known কিন্তু access করা যায় না, সেটিই TDZ।

```js
// TDZ starts for a

console.log(a); // error

let a = 12; // TDZ ends here
```

### Visual explanation

```js
// line 1: TDZ starts
// line 2: still TDZ
// line 3: still TDZ

console.log(a); // cannot access

let a = 12;     // initialization; TDZ ends

console.log(a); // works
```

### `let` TDZ example

```js
{
  console.log(name); // error

  let name = "Rahim";
}
```

### `const` TDZ example

```js
{
  console.log(pi); // error

  const pi = 3.14;
}
```

### TDZ-এর জন্য মনে রাখার analogy

শিশু জন্মানোর আগেই তার official identity ব্যবহার করতে চাইলে সমস্যা হবে। Variable declare/initialize হওয়ার আগে value চাইলে JavaScript value দিতে পারবে না।

### Common mistake

```js
console.log(score);
let score = 0;
```

Correct:

```js
let score = 0;
console.log(score);
```

### মনে রাখার নিয়ম

> `let`/`const` variable declare করার আগে access করবে না।

## Important Differences

### `var` vs `let` vs `const`

| Feature | `var` | `let` | `const` |
|---|---|---|---|
| Modern use | Avoid | Use when value changes | Use when value should not change |
| Scope | Function scoped | Block scoped | Block scoped |
| Reassignment | Allowed | Allowed | Not allowed |
| Redeclaration in same scope | Allowed | Not allowed | Not allowed |
| Must initialize immediately? | না | না | হ্যাঁ |
| Global `window` attach | হ্যাঁ, browser global var হলে | না | না |
| TDZ behavior | আলাদা hoisting behavior আছে | TDZ আছে | TDZ আছে |

---

### Declaration, Initialization, Reassignment, Redeclaration

| Term | সহজ বাংলা অর্থ | Code example |
|---|---|---|
| Declaration | variable বানানো | `let age;` |
| Initialization | প্রথম value দেওয়া | `let age = 20;` |
| Assignment | value দেওয়া | `age = 21;` |
| Reassignment | পুরোনো variable-এ নতুন value দেওয়া | `age = 22;` |
| Redeclaration | same name আবার declare করা | `let age = 23;` |

---

### Scope differences

| Scope | Boundary | Keyword relation |
|---|---|---|
| Global scope | পুরো file/code | বাইরে declare করলে |
| Block scope | `{}` | `let`, `const` respect করে |
| Function scope | `function {}` | `var` function scope follow করে |

## Common Mistakes

### Mistake 1: `var` ব্যবহার করা

```js

```

Better:

```js
let score = 0;
```

```js

```

```js

```

Correct:

```js

```

---

### Mistake 3: `const` declare করে পরে value দেওয়া

Wrong:

```js

```

Correct:

```js

```

---

### Mistake 4: redeclaration আর reassignment গুলিয়ে ফেলা

Wrong:

```js
let age = 20;
let age = 21;
```

Correct:

```js
let age = 20;

```

---

### Mistake 5: block scope না বোঝা

Wrong expectation:

```js
if (true) {
  let message = "Hello";
}

```

Correct usage:

```js
if (true) {
  let message = "Hello";

}
```

---

### Mistake 6: TDZ ignore করা

Wrong:

```js

```

Correct:

```js
let a = 10;
console.log(a);
```

## Final Summary

এই lesson-এর সবচেয়ে important learning হলো JavaScript-এ variable শুধু syntax নয়; এটি program-এর data memory system। Program চলার সময় future-এ কোনো value দরকার হলে variable ব্যবহার করা হয়।

`var`, `let`, `const` তিনটি keyword থাকলেও modern JavaScript-এ সাধারণ rule হলো:

```text
value change না হলে → const
value change হলে → let
var → avoid
```

`var` পুরোনো এবং confusing, কারণ এটি function scoped, same name redeclare করতে দেয়, এবং browser global context-এ `window` object-এর সাথে attach হতে পারে।

`let` এবং `const` modern, safer, এবং block scoped। `let` value change করতে দেয়, কিন্তু same scope-এ redeclare করতে দেয় না। `const` declare করার সময়ই value দিতে হয় এবং পরে reassignment করতে দেয় না।

Scope বুঝতে হবে, কারণ variable কোথা থেকে access করা যাবে সেটা scope নির্ধারণ করে:

- Global scope: পুরো code থেকে access
- Block scope: `{}` braces-এর ভিতরে access
- Function scope: function-এর ভিতরে access

TDZ বুঝতে হবে, কারণ `let`/`const` variable declaration-এর আগে access করলে JavaScript বলে:

```text
Cannot access before initialization
```

মানে JavaScript variable-এর existence জানে, কিন্তু initialization-এর আগে value access করতে দেয় না।

## Quick Revision Cheatsheet

```js
// Declaration
let age;

// Declaration + Initialization
let score = 0;

// Reassignment
score = 10;

// const must be initialized
const pi = 3.14;

// const reassignment not allowed
// pi = 3.14159; // error

// let redeclaration not allowed
// let score = 20; // error

// var redeclaration allowed but avoid
var x = 1;
var x = 2;

// block scope
if (true) {
  let message = "Hello";
}

// console.log(message); // error

// TDZ
// console.log(name); // error
let name = "Rahim";
console.log(name);
```

### One-line memory rules

- `variable` = data রাখার container
- `keyword` = JavaScript-এর special meaning আছে এমন word
- `let` = value change হতে পারে
- `const` = value change করা উচিত নয়
- `var` = পুরোনো, avoid
- `declaration` = variable বানানো
- `initialization` = প্রথম value দেওয়া
- `reassignment` = পরে value change করা
- `redeclaration` = same name আবার declare করা
- `scope` = variable কোথা থেকে access করা যাবে
- `TDZ` = declaration-এর আগে `let`/`const` access করলে error zone
